Voice overview

Lyra integrates with voiceCLI — a separate project that runs TTS (text-to-speech) and STT (speech-to-text) daemons. When voice is enabled, Lyra can receive voice messages from Telegram and respond with audio.

Voice requires voiceCLI daemons to be running on the same machine. See ~/projects/voiceCLI and the supervisor pattern in lyra-stack/.

Setup

  1. Start voiceCLI daemons
    bash
    make tts    # start voicecli_tts
make stt    # start voicecli_stt
  2. Enable voice in config.toml
    toml
    [voice]
enabled = true
tts_url = "http://localhost:8020"
stt_url = "http://localhost:8021"
  3. Assign a voice-capable agent

    Ensure the agent has voice = true in its config or use lyra agent edit to enable it.

  4. Reload Lyra
    bash
    make lyra reload

Voice commands in chat

Command / ActionEffect
/voice onEnable voice responses for this session
/voice offDisable voice responses (text only)
Send voice messageLyra transcribes with STT, processes, and responds with TTS audio

Architecture

Telegram voice msg → STT daemon → text transcript → AgentBase → text response → TTS daemon → audio file
Known issue (codebase audit): audio_loop contains blocking I/O calls in an async context. This can cause latency spikes under concurrent load. A fix is tracked as a high-priority item.
