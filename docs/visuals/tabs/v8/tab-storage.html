File locations

~/.lyra/auth.db
Primary SQLite database. Contains all agents, bots, assignments, sessions.
~/projects/lyra/config.toml
Machine-specific config. Bot tokens, API keys, cwd. Gitignored.
~/projects/lyra/config.toml.example
Template committed to git. No secrets.
src/lyra/agents/*.toml
Seed files for agent defaults. Read only by lyra agent init.
src/lyra/personas/*.persona.toml
Voice / style overlays. Loaded at agent reload time.
~/.lyra/memory/
Persistent memory files. Written by the agent, read at session start.
~/projects/lyra-stack/conf.d/
Symlinks to supervisor program configs. Managed by make register.
~/projects/lyra/local/machines.md
Machine IPs, disk layouts, SSH commands. Gitignored, not versioned.

auth.db schema (key tables)

TableKey columnsPurpose

botsid, platform, token, agent_nameBot registrations and agent assignments.
sessionsscope_id, bot_id, created_at, last_activeActive conversation sessions.
messagessession_id, role, content, tsConversation history per session.
bashQuick DB inspection
sqlite3 ~/.lyra/auth.db ".tables"
sqlite3 ~/.lyra/auth.db "SELECT name, model, updated_at FROM agents"
sqlite3 ~/.lyra/auth.db "SELECT * FROM bots"

Backup strategy

auth.db is the single source of truth for all agent configs. Back it up before running lyra agent init --force or destructive DB operations.
bash
# Quick backup before risky operations
cp ~/.lyra/auth.db ~/.lyra/auth.db.bak.$(date +%Y%m%d_%H%M%S)

# Or use SQLite's online backup
sqlite3 ~/.lyra/auth.db ".backup ~/.lyra/auth.db.bak"
