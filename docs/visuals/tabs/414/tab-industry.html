How Others Do It
OpenAI Voice Agents
Two separate instruction layers
Agent instructions Speech API ├── identity ├── voice: "alloy" ├── demeanor ├── model: "gpt-4o-mini-tts" ├── tone └── instructions: ├── enthusiasm "accent, pacing, ├── formality emotion, delivery" ├── emotion level └── pacing
Two instructions fields — one for agent reasoning, one for Speech API delivery. Personality is duplicated across both manually.
LiveKit Agents
OOP class separation
Agent( AgentSession( instructions="..." vad=silero.VAD(), tools=[...] stt=STT("deepgram"), ) llm=LLM("gpt-4.1"), tts=TTS("cartesia", voice="9626c31c") )
Clean split: Agent = personality/logic, AgentSession = audio pipeline. Same agent reusable across different voice configs.
Vapi
Three-module orchestration
Transcriber ←→ Model ←→ Voice (STT config) (LLM + (TTS config) provider system provider language prompt voice ID model persona speed guardrails)
Personality lives entirely in LLM system prompt. Voice is a separate provider config. The orchestrator connects them but fields never cross.
ElevenLabs Agents
System prompt + voice sliders
System Prompt Voice Config ├── personality traits ├── voice ID ├── goals ├── stability (0.3–0.85) ├── guardrails ├── similarity └── role definition └── speed (0.9–1.1x)
Personality is narrative text in the prompt. Voice is purely technical sliders. No shared fields.
Retell AI
Sectioned dashboard config
Global Prompt Voice Speech ├── persona ├── voice ID ├── responsiveness ├── identity ├── speed ├── interruption ├── guardrails └── dynamic ├── backchanneling └── behavior speed └── pronunciation
Three distinct sections. Persona is prompt-only. Voice and Speech are separate technical configs.
SillyTavern
Complete store separation
Character Card TTS Extension ├── name ├── voice mapping ├── description (per character) ├── personality summary └── provider settings ├── scenario (in settings.json, └── example dialogues NOT in card)
Personality is purely narrative in the character card. TTS voice mapping lives in a completely separate store.
