Quick Start
Get Lyra running in 4 steps — ~10 minutes from zero to first message.
1
📋 Prerequisites
Install these before cloning.
🐍 Python 3.12+
python --version
📦 uv
pip install uv
🤖 Claude Code CLI
claude --version
# claude.ai/download
💡
Claude CLI is the default LLM backend. Lyra spawns claude --input-format stream-json as a subprocess. You need a Claude subscription. For API key instead, set backend = "anthropic-sdk" + ANTHROPIC_API_KEY.
2
📥 Clone & Install
bash
git clone https://github.com/roxabi/lyra
cd lyra
uv sync

# Put lyra CLI on your PATH
source .venv/bin/activate
# Or permanently in ~/.bashrc:
# export PATH="$HOME/projects/lyra/.venv/bin:$PATH"

# Verify
lyra --version
3
🔑 Configure .env & config.toml

.env — secrets (non-token)

bash.env
# Bot tokens are stored encrypted in auth.db
# Use: lyra bot add --platform telegram --bot-id lyra

# Voice STT (optional — omit to disable)
STT_MODEL_SIZE=small

# Health / config secrets (optional)
LYRA_HEALTH_SECRET=...
LYRA_CONFIG_SECRET=...
🔐
Bot tokens are encrypted in auth.db — not in .env or config.toml. Run lyra bot add once after install.

config.toml — auth (gitignored)

bash
cp config.toml.example config.toml
nano config.toml
tomlconfig.toml
# Minimal single-bot auth
[auth.telegram]
default = "blocked"
owner_users = [YOUR_TG_ID]

[auth.discord]
default = "blocked"
owner_users = [YOUR_DC_ID]
👤
Get Telegram ID from @userinfobot. Get Discord ID: Settings → Advanced → Developer Mode → right-click username.
4
▶️ Start Lyra
bash
lyra start
output
INFO lyra.bootstrap.agent_factory: Agent loaded: name=lyra_default model=claude-sonnet-4-6 backend=claude-cli
INFO lyra.bootstrap.multibot: Lyra started — adapters: telegram, discord, health on :8443.
INFO lyra.adapters.discord: Discord bot ready: YourBot#1234 (id=123456789)
🧪 Run tests
uv run pytest -v
No tokens needed — adapters are mocked.
✅ Validate config
lyra config validate
lyra config show
💬 Send first message
Telegram: DM your bot.
Discord: @YourBot hello!
What makes Lyra different
🎤 Full Voice Loop
Send a voice message → Whisper transcribes → Claude responds → TTS replies with audio. Full voice conversations on Telegram and Discord voice channels.
🔗 Paste a Link, Save Knowledge
Just paste a URL. Lyra auto-saves it to your personal knowledge vault with title and tags. Search later with /search. Build your second brain via chat.
💻 Code Assistant via Chat
Type /lyra what's the last commit? — Lyra switches Claude's working directory to your project and answers. Full code workspace access from Telegram or Discord.
Troubleshooting
🐍 Python version too old
python --version
# Need 3.12+. Install via pyenv or deadsnakes PPA.
Lyra uses modern Python features (StrEnum, tomllib). Older versions will fail at import.
🤖 Claude CLI not authenticated
claude --version
claude   # starts interactive — if it works, you're authenticated
Claude CLI needs an active subscription. Visit claude.ai/download to install and authenticate.
🔌 Port 8443 already in use
# Change the health port:
LYRA_HEALTH_PORT=9443 lyra start
# Or in supervisor config: environment=LYRA_HEALTH_PORT="9443"
Default health port is 8443. If another service uses it, override via env var.
🔑 Bot token invalid or missing
lyra bot list              # check stored credentials
lyra bot add --platform telegram --bot-id lyra  # re-add
Tokens are encrypted in auth.db. If the bot fails to start, re-add the token.
📄 config.toml errors
lyra config validate   # schema check
lyra config show       # dump resolved config
Common issues: missing owner_users, wrong TOML syntax. Validate before starting.
📦 uv sync fails
pip install --upgrade uv
uv sync --refresh
Ensure uv is up to date. Use --refresh to clear the cache if dependencies fail to resolve.
