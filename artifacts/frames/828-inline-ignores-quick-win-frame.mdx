## Problem

The codebase carries **394 inline ignores across 171 files** (47 unique `noqa:` / `type: ignore[...]` kinds). An Explore-agent audit classified the top 12 buckets by ROI. Five buckets — **60 ignores / ~15% of total** — are low-risk, mechanical refactors using stdlib primitives. Left alone, they erode lint signal and mask real regressions.

**Why now:** clusters are small (≤17 per bucket), edits are same-class and independent, and a cleaner baseline unblocks the structural-debt frame (`PLR0913`, `C901`, `attr-defined` — 147 ignores).

**Observable impact:**
- 60 inline ignores suppressing lint/type checks across tests **and** src
- Lint noise normalises ignores → future code accumulates more
- Cleaner baseline makes structural-debt scoping easier next

## Who

- **Primary:** developers maintaining lyra — clearer mocks, stricter fixtures, typed APIs
- **Secondary:** future contributors — fewer "why is this ignored?" questions during review

## Constraints

- **No behavior change:** pure refactor; public APIs untouched; green `make validate`
- **No coverage regressions:** pytest green, coverage ≥ pre-PR
- **No new runtime deps:** stdlib `unittest.mock` + existing `pytest` fixtures
- **Bucket-independent:** per-bucket commits inside one bundled PR; each bucket bisect-able
- **If a src ignore cannot be removed mechanically** → move it to `justified` list with a 1-line reason (do not silently keep)
- **Time budget:** ~10-12h total (revised up from 8h for src edits)
