Data & Storage
Where Lyra stores every byte — databases, logs, audio, sessions, cache, and config.
🗺️
Two root vaults: ~/.lyra/ for credentials & auth, ~/.roxabi-vault/ for memory & sessions. Everything else is in standard system locations.
📁 Directory Tree
~/.lyra/ ├── auth.db # bot credentials + agents (AgentStore) + grants + prefs ├── turns.db # raw conversation turns (L1 audit trail) ├── pairing.db # invite codes (auto-expire) ├── keyring.key # Fernet encryption key (chmod 600) ├── agents/ │ └── <name>.toml # user agent overrides (gitignored) └── logs/ ├── YYYYMMDD_HHMMSS_lyra.log # rolling (10 MB × 5) └── monitor.log # monitoring daemon (5 MB × 3) ~/.roxabi-vault/ └── [SQLite + FTS index] # memory vault (roxabi-vault manages) ~/.claude/projects/ └── <cwd-slug>/ └── <session_id>.jsonl # Claude session history (per pool) ~/projects/lyra/supervisor/logs/ ├── lyra_telegram.log # stdout (10 MB × 3) ├── lyra_telegram_error.log # stderr (5 MB × 3) ├── lyra_discord.log └── lyra_discord_error.log ~/projects/lyra-stack/logs/ ├── supervisord.log # process manager (10 MB × 3) ├── voicecli_stt.log └── voicecli_tts.log /tmp/ or $LYRA_AUDIO_TMP └── lyra_audio_XXXXXX.ogg # temp audio (deleted after send)

🎤 Voice & Audio
📥
STT Input
Incoming audio from Telegram / Discord
Temp path
$LYRA_AUDIO_TMP/lyra_audio_XXXXXX
falls back to system /tmp/
Formats
OGG/Opus, MP3, WAV
passed to voiceCLI STTService.transcribe()
Size limit
$LYRA_MAX_AUDIO_BYTES (default 5 MB)
RetentionTemporary — deleted after transcription
📤
TTS Output
Outgoing audio generated by Lyra
Storage
In-memory bytes only
OutboundAudio sent directly, never written to disk
Format
OGG/Opus 48 kHz mono
RetentionNone — ephemeral (memory only)

🔒 Security & Encryption
🗝️
keyring.key
~/.lyra/keyring.key
Algorithm
Fernet (AES-128-CBC + HMAC-SHA256)
Permissions
chmod 600 — owner read/write only
Used for
Encrypting bot tokens in auth.db
RetentionPermanent — losing this key = losing all credentials
🛡️
What is never stored
Data that stays ephemeral
TTS output
In-memory only, never written
Bot tokens
Never logged, only encrypted at rest
Message content
Persisted in turns.db (L1) + memory vault (L3)
Auth cache
In-memory dict (write-through to SQLite)

Why Lyra?
How Lyra compares to other approaches for AI on messaging platforms.
AlternativeLimitationLyra's Advantage
ChatGPT / Claude direct Cloud-only. No persistent memory across sessions. No voice on Telegram. No code workspace. Self-hosted. Persistent semantic memory. Full voice loop. Workspace access to your projects.
Botpress / Rasa Flow-builder paradigm. Requires training data. Weeks to deploy. AI-native — powered by Claude. Zero training data. Running in 10 minutes.
LangChain DIY Framework, not a product. You assemble everything yourself. Complete product. Multi-platform, voice, memory, routing — all built-in.
Poe / Character.ai bots Vendor-locked. No GPU control. Limited customization. Full control. Your GPU. Open source. Custom personas and plugins.
