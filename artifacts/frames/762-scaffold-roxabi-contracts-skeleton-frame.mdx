## Problem

ADR-049 mandates a second uv workspace subpackage `packages/roxabi-contracts/` alongside `packages/roxabi-nats/`. Until this scaffold exists, no per-domain contract (voice, image, memory, llm) has a home, and the epic (#761) chain is blocked at step 0.

**Why now:** #761 Phase 1 (voice-only) is ready to start. Every downstream child issue (#763 port voice, #764 test doubles, #765 CONTRACT_VERSION migration, etc.) imports from `roxabi_contracts.envelope` or depends on the package being discoverable by `uv sync`. Without the scaffold, nothing else in the epic can land.

**Observable impact:**
- Drift between lyra publishers and satellite subscribers cannot be caught at type-check time
- New domain contracts have no canonical landing place → ad-hoc per-repo re-implementation continues
- E2E tests remain blocked on full GPU + voiceCLI stack (test doubles require this package)

## Who

- **Primary:** Downstream child issues #763–#769 consuming `ContractEnvelope` as the base class for every domain model.
- **Secondary:** voiceCLI#69 — adopts `roxabi-contracts[testing]` once `v0.1.0` ships; Renovate pins satellite repos to the version.

## Constraints

- **ADR-049 §Layout is canonical** — directory structure, file names, and field set are fixed; no bikeshedding.
- **Pure Pydantic runtime** — no `roxabi-nats` or `nats-py` in runtime deps; transport-related extras only under `[testing]`.
- **`requires-python = ">=3.12"`** to match roxabi-nats floor.
- **`model_config = ConfigDict(extra="ignore")`** — forward-compat with future field additions; unknown fields silently dropped.
- **Root `pyproject.toml` touches three sections** — `[tool.uv.workspace].members`, `[tool.pyright].include`, `[tool.ruff].src`. Miss any and the package is invisible to tooling.
- **`uv sync` must succeed** after the edit — workspace resolver fails the whole build if the subpackage is malformed.

## Out of Scope

- **No voice models, subjects, or fixtures** — those land in #763.
- **No test doubles** (`FakeTtsWorker`, `FakeSttWorker`) — those land in #764.
- **No CONTRACT_VERSION migration from roxabi-nats** — that is #765 (will import from this scaffold once it exists).
- **No release-please config** — #767 wires it after the package has content.
- **No CI pytest integration for `packages/roxabi-contracts/tests`** — #768 extends root pytest config.
- **No Renovate rule documentation** — #769.

## Complexity

**Tier: F-lite** — 5 files touched (4 new, 1 modified), clear scope from ADR-049, no unknowns.

Signals observed:
- Epic #761 labels this as "S" (≤3 files). Actual file count is 5 (pyproject.toml, `__init__.py`, envelope.py, test_envelope.py, root pyproject.toml edits), which edges into F-lite territory.
- Root `pyproject.toml` edit is the delicate step — any typo breaks workspace resolution for every package. A spec + plan gate is cheap insurance.
- Pure mechanical scaffold otherwise — no architectural decisions remain open after ADR-049.

**Decision:** proceed with F-lite to get plan-gate verification on the root `pyproject.toml` changes. No `/analyze` step (skipped at F-lite).
