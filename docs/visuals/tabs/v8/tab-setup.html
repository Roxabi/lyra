Installation

  1. Clone and install dependencies
    bash
    git clone https://github.com/roxabi/lyra ~/projects/lyra
cd ~/projects/lyra
uv sync
  2. Create config.toml

    Copy the example and fill in your bot tokens and API keys.

    bash
    cp config.toml.example config.toml
$EDITOR config.toml
  3. Seed the agent database

    Imports TOML seed files from src/lyra/agents/ into ~/.lyra/auth.db. Required once on a new machine.

    bash
    lyra agent init
  4. Register with the supervisor stack
    bash
    cd ~/projects/lyra
make register    # symlinks supervisor configs into lyra-stack/conf.d/
  5. Start daemons
    bash
    make lyra        # starts lyra_telegram + lyra_discord
make lyra logs   # tail stdout to verify startup

config.toml reference

tomlconfig.toml (minimal)
[defaults]
cwd = "/home/mickael/projects"   # working directory injected as context
log_level = "INFO"

[telegram]
bots = [
  { token = "123:AAA...", agent = "lyra_default" }
]

[discord]
bots = [
  { token = "MTA...", agent = "lyra_default" }
]

[anthropic]
api_key = "sk-ant-..."
default_model = "claude-sonnet-4-6"

[voice]
enabled = false
tts_url = "http://localhost:8020"
stt_url = "http://localhost:8021"
KeyRequiredDescription
[defaults].cwdreqMachine-specific working directory. Never in agent TOML.
[telegram].bots[].tokenreqBot token from BotFather
[telegram].bots[].agentreqAgent name to handle messages on this bot
[anthropic].api_keyreqAnthropic API key for Claude models
[anthropic].default_modeloptFallback model when agent doesn't specify one
[voice].enabledoptEnable voice loop. Requires voiceCLI daemons.
config.toml is gitignored. It contains secrets. Never commit it. Use config.toml.example for documentation.

LYRA_LOG_LEVELOverride log level (DEBUG / INFO / WARNING)
