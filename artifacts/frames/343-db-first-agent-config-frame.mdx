## Problem

Agent configuration is split across three sources that drift independently: TOML seed files (`src/lyra/agents/*.toml`), persona files (`~/.roxabi-vault/personas/*.persona.toml`), and the SQLite DB (`~/.lyra/auth.db` agents table). The DB is the runtime source of truth at startup, but hot-reload still watches the TOML file's mtime — meaning `lyra agent edit` changes are silently overwritten on reload. The TOML and DB have already diverged (lyra_default has Bash+Skill tools in DB but not in TOML).

The naming taxonomy is also confused: "plugins" means slash-command handlers (echo, svc, pairing) but session commands (`/add`, `/explain`, `/summarize`) are a separate registration path that only works with `AnthropicAgent` — so `/add` returns "Unknown command" on both deployed bots because they use `SimpleAgent` (backend=claude-cli).

Voice configuration is scattered across three unrelated places: persona `[voice]` (decorative prose), `tts_json` (synthesis params), `stt_json` (transcription params), and `i18n_language` (also used as TTS/STT language fallback).

## Who

- **Primary:** Mickael — sole operator and user of Lyra on Telegram/Discord. Directly hit by `/add` not working, config drift between TOML and DB, and the cognitive overhead of understanding which file controls what.
- **Secondary:** Future agents/personas — the current indirection makes it harder to add new agents or modify existing ones reliably.

## Constraints

- Both machines (Machine 1 production, Machine 2 dev) have agents seeded in the DB already — migration must handle existing rows
- The DB schema change must be backward-compatible or include a migration step
- `lyra agent init` (TOML → DB seeder) and `lyra agent edit` (DB editor) must both work after the change
- `lyra agent create` has a TODO (#268) to write to DB directly — this change aligns with that
- Persona vault files (`~/.roxabi-vault/personas/`) exist on both machines and must be migrated to DB before deletion
- Tests use TOML fixtures extensively — test infrastructure must be updated

## Out of Scope

- Smart routing changes (stays as-is, just carried over)
- Workspace system changes
- New plugin development (only renaming/re-categorizing existing ones)
- Backend selection logic (claude-cli vs anthropic-sdk factory) — unchanged
- Memory system changes (namespace field stays, just clarified)
