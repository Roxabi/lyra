## Source

> "containerize the voicecli workers themselves — GPU passthrough, VRAM management, model cache volumes" — Issue #728, supersedes #609.

## Problem

voiceCLI STT and TTS workers run as bare-metal supervisord programs on M₁ (marked DEPRECATED in
conf files, `autostart=false`). The containerization artifacts exist in the voiceCLI repo
(Dockerfile, Quadlet units, publish CI, image published at `ghcr.io/roxabi/voicecli:staging`) but
have not been installed on M₁. Two implementation gaps in the Quadlet files block a safe cutover:

1. `NATS_NKEY_SEED_PATH` env var is not set — secret is mounted but the path is not exported to the process, so NATS auth will silently fail at startup
2. `ReadOnly=true` is missing — security regression vs ADR-053/054 pattern established in lyra

Additionally: Podman secrets are not yet provisioned on M₁, the model cache named volume is not
pre-seeded (~8GB cold download on first start), and NATS auth.conf voicecli identity presence is
unverified on M₁.

## Outcome

- `voicecli-stt` and `voicecli-tts` running as Podman Quadlet user services on M₁
- `make voice-smoke` green against containerized workers
- Model cache persists across container restarts via named volume (no re-download)
- NATS auth.conf voicecli identities verified on M₁
- Supervisord DEPRECATED stubs removed; supervisord no longer manages voiceCLI workers
- Docs updated (voiceCLI CLAUDE.md + lyra `docs/ops/`)

## Appetite

~3 days focused work. Most architecture is decided; the work is fix → provision → install → verify → cleanup. Risk: cold model download (~8GB on first volume init) and NATS_NKEY_SEED_PATH path confirmation may add half a day each if not straightforward.

## Shapes

Shapes here differ on **cutover strategy** — architecture is decided. Two hard prerequisites
apply to all shapes: (1) fix Quadlet gaps in voiceCLI PR, (2) confirm NATS_NKEY_SEED_PATH path
is correct for UID 1501 home layout. These are prerequisites, not shape-specific decisions.

### Shape A: Fix-then-cutover (recommended)

Fix the two Quadlet gaps in voiceCLI, merge to staging → publish CI updates
`ghcr.io/roxabi/voicecli:staging`. On M₁: provision secrets, pre-seed model volume, install
Quadlet units, activate, smoke-test, then remove supervisord stubs.

Since both `voicecli_tts` and `voicecli_stt` are `autostart=false`, there is no live traffic
to protect — cutover is a manual activation, not a hot swap.

**Rollback:** if smoke fails post-activation → `systemctl --user stop voicecli-stt voicecli-tts`
→ supervisord stubs still present and startable (`supervisorctl start voicecli_stt voicecli_tts`).
Stubs must not be removed until smoke is confirmed green.

**Trade-offs:**
- Pro: Single linear path; no parallel infrastructure to manage
- Pro: Fixes security regression before first prod activation
- Pro: `autostart=false` → no prod impact during transition window
- Con: ~8GB cold model download risk on first volume init; mitigate by pre-seeding

**Rough scope:** Contained — 2 files changed in voiceCLI repo + M₁ ops provisioning + lyra docs.

### Shape B: Validate-in-parallel, then flip

Install Quadlet units alongside stopped supervisord configs. Verify smoke against containers
before removing anything. Keep supervisord stubs as dormant rollback for 1+ week.

**Trade-offs:**
- Pro: Explicit rollback path documented; supervisord stubs retained longer
- Con: `autostart=false` means there's nothing running to roll back from — rollback value ≈ 0; supervisord stubs as parallel docs creates noise

**Rough scope:** Same as A; adds delayed cleanup.

### Shape C: lyra-repo-proxied Quadlet units

Copy `voicecli-*.container` into lyra's `deploy/quadlet/` so all M₁ units live in one repo.

**Trade-offs:**
- Pro: Single directory on M₁ for all Quadlet units (lower ops cognitive load); one `make quadlet-install`
- Con: Violates QUADLET-ADOPTION.md — each project owns its units; creates cross-repo coupling (voiceCLI updates need a lyra PR); breaks the pattern being replicated by imageCLI, llmCLI, 2ndBrain

**Rough scope:** Same as A; immediately creates technical debt.

## Fit Check

```
Shape C → eliminated: QUADLET-ADOPTION.md pattern violation; coupling not justified
Shape B → eliminated: rollback value ≈ 0 (autostart=false); parallel stubs add noise, not safety
Shape A → recommended: linear path, fixes gaps before first prod activation, correct ownership
```

### Implementation checklist (Shape A)

```
1. voiceCLI PR — fix Quadlet files:
   a. Confirm NATS_NKEY_SEED_PATH target path against UID 1501 home layout
      Secret mounts to target=/home/voicecli/.config/voicecli/nkeys/seed.seed
      Add: Environment=NATS_NKEY_SEED_PATH=/home/voicecli/.config/voicecli/nkeys/seed.seed
   b. Add ReadOnly=true to both container files
      Note: ReadOnly applies to root FS only — named volume mounts retain rw (Podman behavior)
   → Merge → CI publishes updated ghcr.io/roxabi/voicecli:staging

2. M₁ — NATS auth.conf verification:
   grep voicecli ~/projects/lyra/deploy/nats/auth.conf  # must show voice-stt, voice-tts identities
   If missing → add via lyra auth.conf update + make nats-setup

3. M₁ — secret provisioning:
   podman secret create voicecli-nats-stt ~/.voicecli/nkeys/voice-stt.seed
   podman secret create voicecli-nats-tts ~/.voicecli/nkeys/voice-tts.seed

4. M₁ — model volume pre-seeding (avoids 8GB cold download on first start):
   # If ~/.cache/huggingface exists from bare-metal workers:
   podman run --rm \
     -v ~/.cache/huggingface:/src/huggingface:ro \
     -v ~/.cache/voicecli:/src/voicecli:ro \
     -v voicecli-models:/dst \
     docker.io/library/alpine \
     sh -c "cp -r /src/huggingface /dst/ && cp -r /src/voicecli /dst/"

5. M₁ — install Quadlet units:
   cp ~/projects/voiceCLI/deploy/quadlet/*.{container,volume} ~/.config/containers/systemd/
   systemctl --user daemon-reload

6. M₁ — activate and verify:
   systemctl --user start voicecli-stt.service voicecli-tts.service
   systemctl --user status voicecli-stt voicecli-tts
   nvidia-smi  # confirm GPU visible inside containers
   make voice-smoke  # must be green

7. M₁ — cleanup (only after smoke green):
   rm ~/projects/conf.d/voicecli_{stt,tts}.conf
   supervisorctl reread && supervisorctl update
```

   `/home/voicecli/.config/voicecli/nkeys/seed.seed`. Verify this path is accessible at UID 1501
   inside the container and that roxabi-nats `_read_nkey_seed()` accepts it. If wrong, NATS auth
   silently fails on startup.
