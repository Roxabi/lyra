What is Lyra?

Lyra is a personal AI agent engine that runs as a persistent daemon on your home server. It connects to Telegram and Discord, routes incoming messages to the right AI agent, and streams responses back. Each agent is a full conversation partner: it has a system prompt, a set of tools, a model, and memory.

Unlike hosted chatbots, Lyra runs entirely on your own hardware — your GPU, your data, your rules. The agent configuration lives in a local SQLite database and is hot-reloaded on every message, so edits take effect instantly without a restart.

Core Concepts

ConceptOne-line definitionWhere it lives
AgentStateless AI persona: system prompt + tools + model + configagents table in auth.db
BotA Telegram/Discord bot token. One bot can host many agents (one at a time)bots table + config.toml
ScopeThe conversation context: a chat ID (group) or DM ID. One scope = one PoolIn-memory, per-platform
PoolA running agent session bound to a scope. Holds history + circuit stateIn-memory on Hub
PlatformA chat network adapter: Telegram or Discordsrc/lyra/adapters/
HubCentral router: receives all inbound messages, dispatches to Poolssrc/lyra/core/hub.py
AgentStoreWrite-through in-memory cache of all agent rows; source of truth for hot-reloadsrc/lyra/db/agent_store.py
Circuit BreakerPer-pool failure guard: opens after repeated errors, closes on recoveryCircuitRegistry

How a message flows

Telegram / Discord → Adapter → InboundBus → Hub → Pool (scope) → AgentBase → OutboundDispatcher → Telegram / Discord

Each step is async. The InboundBus uses a per-platform queue. The OutboundDispatcher has a per-bot retry queue with circuit breakers. Agent execution is isolated per Pool — one slow conversation doesn't block others.
