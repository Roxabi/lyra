## Problem

The `vault:vault-migrate` skill was built for the v1 vault schema. The vault has since moved to roxabi-memory v2 with fastembed ONNX embeddings, namespace support, and a unified `entries` table. The skill needs updating to use the v2 sync interface.

Meanwhile, 2ndBrain holds 465 entries in `memory.db` (twitter, github, articles, tools, etc.) plus **59 markdown files** (~511KB) across 6 directories. None of this data is in the Roxabi Vault yet. The vault currently has only 2 LinkedIn post entries.

**Scope decision:** Migrate everything — all 465 DB entries + all 59 disk files. Disk-only files get new vault entries created from their content. Broken DB file refs → migrate DB row, skip file copy, log warning.

The migration must be non-destructive (original `memory.db` untouched, existing vault entries preserved), idempotent (running twice produces the same result), and generate fresh fastembed embeddings at import time.
