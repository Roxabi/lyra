Tout est-il async via le hub / bus ?
  • OUI — entièrement asyncio. Aucun appel bloquant dans le chemin critique. Hub, pools, adapters, TTS : tout tourne en coroutines.
  • Texte + TTS concurrents. Hub.dispatch_response() envoie le texte immédiatement dans l'OutboundDispatcher, puis crée un asyncio.Task indépendant pour la synthèse TTS. Les deux s'exécutent en parallèle.
  • Cancel-in-flight. Pool._process_with_cancel() : si l'utilisateur envoie un nouveau message pendant que le LLM tourne, la tâche courante est annulée et le nouveau message est traité.
  • TTS failure isolé. Si voicecli échoue, erreur loggée. Le texte a déjà été envoyé. Pas de propagation à l'utilisateur.
  • Circuit breakers. OutboundDispatcher vérifie un CB par platform ("telegram", "discord") + CB "anthropic". Si le CB est ouvert, le message est droppé (pas de queue infinie).
  • Queues bornées. InboundBus: 100 msg/platform, _staging: 500. Backpressure automatique si le Hub est débordé.
  • voicecli CWD-sensitif.  voicecli cherche voicecli.toml dans le répertoire courant (Path("voicecli.toml")). Lyra doit invoquer voicecli depuis ~/projects/voiceCLI/ ou setter le CWD explicitement. Sinon : langue anglaise par défaut, chunk_size différent, fichiers ~169 KB au lieu de ~40 KB.
