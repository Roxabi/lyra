## Problem

Every user message hits the same model (Claude Sonnet) regardless of complexity. A greeting or yes/no question costs the same tokens and latency as a deep reasoning task. There is no optimization path — cost grows linearly with usage, and trivial queries cannot be routed to cheaper/faster models (including local models on Machine 2).

Both blocking dependencies are resolved: #76 (LlmProvider interface) is closed, and #104 (circuit breaker decorator) is closed. The decorator chain infrastructure is ready.

## Who

- **Primary:** Lyra end-users (Mickael) — faster responses for simple queries, lower cost
- **Secondary:** The agent system itself — enables future local model routing (Machine 2 SLMs)

## Constraints

- Must wrap any `LlmProvider` (decorator pattern, same as circuit breaker #104)
- Classifier must be zero-cost at baseline (heuristic-only, no LLM call for classification)
- Routing table defined in TOML config (`[agent.smart_routing]`)
- Disabled by default (`enabled = false`) — opt-in
- Must not break existing behavior when disabled

## Out of Scope

- SLM-backed classifier (future — requires Machine 2 inference endpoint)
- Local model routing (future — depends on voiceCLI/inference infra)
- Dynamic routing based on load/latency (future optimization)
- Cost tracking/budgeting per complexity tier
