## Problem

The Tts/Stt NATS contract between `lyra` and `voiceCLI` lives today only in ADR-044 prose and is re-implemented independently in two places: `lyra/src/lyra/nats/nats_tts_client.py` (+ `nats_stt_client.py`) and `voiceCLI/tts_adapter.py` (+ stt equivalent). There is no shared schema module, no single source of truth for subject names, and no structural test that both sides agree on the wire format. This is exactly the silent-drift scenario ADR-049 was written to eliminate by extracting shared Pydantic schemas into `packages/roxabi-contracts`.

With PR #771 (issue #762) now merged, the `roxabi-contracts` skeleton exists (`envelope.py`, `py.typed`, empty `__init__.py`). The voice domain is the first real contract to port and will validate the packaging/workspace setup end-to-end before `image`, `memory`, `llm` follow.

## Who

- **Primary:** contracts maintainer porting voice schemas into a typed, testable module; reviewers verifying no behavioral drift on the wire.
- **Secondary (downstream, post-port):** lyra hub voice code (`nats_tts_client.py`, `nats_stt_client.py`) and voiceCLI's `tts_adapter.py` / stt adapter — they will eventually consume `roxabi_contracts.voice`, but migration of those callers is **not** part of this issue.

## Constraints

- Pydantic v2; every model uses `model_config = ConfigDict(extra="ignore")` (forward-compat: unknown fields silently dropped).
- Package-root imports must stay pure Pydantic — **no NATS imports at `roxabi_contracts.voice.__init__`**. Any transport helpers live behind the `[testing]` extra per ADR-049.
- Fixtures are PURE DATA: `silence_wav_16khz` synthesized via `scipy.io.wavfile.write` at import/use time; no binary blobs committed, no real user audio.
- SUBJECTS namespace must match what `NatsTtsClient` and `nats_stt_client.py` publish/subscribe to today (this is the anti-drift check).
- Any reconciled field mismatch between lyra and voiceCLI must be called out in the PR description so the resolution is auditable.
- pyright clean on `packages/roxabi-contracts/src`.
- Blocked-by #762 is resolved (PR #771 merged).

## Out of Scope

- Migrating `NatsTtsClient`, `nats_stt_client.py`, or voiceCLI's adapters to consume `roxabi_contracts.voice`. That is a separate follow-up per ADR-049 Phase 1 steps 4+.
- Other domains: `image`, `memory`, `llm` — separate issues under Epic #761.
- Runtime transport helpers (NATS publish/subscribe wrappers). Envelope lives in `envelope.py`; voice ports pure schemas only.
- Tagging/publishing a new `roxabi-contracts` version — happens after all four domains land.
- Changing the wire contract itself. Port-only; reconciliation means **documenting and resolving drift**, not redesigning.

## Complexity

**Tier: F-lite** — Clear scope, single domain (backend schemas), no new architecture, ≤10 files (subjects.py, models.py, fixtures.py, voice/__init__.py, two test files, plus the package __init__ re-export). Label `size:F-lite` confirms.
