## Problem

The dep-graph generator at `scripts/dep-graph/` (promoted from a forge artifact to a git-tracked package earlier this week) handles a single repo via `meta.repo`. Cross-repo references are rendered only as free-text annotations in `cross_deps[]` — no live state, no card, no auto-sync with foreign issue titles or open/closed status.

This breaks as soon as an initiative spans repos. Concretely, epic #704 (unified vault + intel ingestion via NATS) already has a first-class foreign child:

- `Roxabi/roxabi-vault#24` — subscriber daemon (real issue)
- `Roxabi/roxabi-intel` — future optional co-subscriber (no issue filed yet; mentioned as an out-of-scope follow-up in #704)

Today these show up as opaque text under Lane I. As more epics cut across repos (voiceCLI ↔ lyra, vault ↔ intel, future roxabi-dashboard), the text-annotation pattern degrades fast: foreign state is stale, relationships are not navigable, and lane timelines lie about what's actually blocking what.

## Who

- **Primary:** Mickael — planning weekly across Lyra + vault + intel, using the dep-graph as the single "what's next" board.
- **Secondary:** future roxabi-dashboard consumers — the multi-repo schema is the foundation for per-project boards aggregated into one nav shell.

## Constraints

- **GitHub native cross-repo deps work** — fetcher must use `/dependencies/{blocked_by,blocking}` across all listed repos rather than inventing a custom format.
- **Breaking schema change** — every issue ref becomes `{repo, issue}` object; no bare-int shortcut, no `primary_repo` magic. Explicit > implicit.
- **Migration required** — existing `lyra-v2-dependency-graph.layout.json` has ~60+ bare ints + overrides keyed by int. One-shot migration script, diff-reviewable.
- **`extra_deps` semantic narrowing** — stays as closed-PR / informal-dep escape hatch only. Not a cross-repo crutch.
- **Backwards-compat at render level** — single-repo layouts must produce byte-identical output post-migration (golden test for lyra layout).
- **Tool location:** stays at `scripts/dep-graph/` in the `lyra` repo for now. Move to a dedicated `roxabi-dashboard` repo is Tier 3 / future.

## Out of Scope

- Dashboard aggregation / multi-board nav shell (future — `roxabi-dashboard` repo).
- Foreign-repo label search (listed repos contribute their labeled issues, but non-listed refs only appear if explicitly in `order[]`).
- GitHub Projects V2 integration.
- Reusable NATS subscriber scaffolding (tracked separately under #704 follow-ups).
- New visual features (filters, collapsible lanes, zoom) — this is schema + fetch + render plumbing only.
