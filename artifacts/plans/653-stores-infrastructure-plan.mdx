## Summary

Incremental rollout: one prep PR establishes the `lyra.infrastructure` layer and activates the import-linter forbidden contract, then one PR per SQLite store moves it to `infrastructure/stores/` with a backward-compatible shim, then a final cleanup PR removes all shims and does the `core/stores/__init__.py` surgery. Each PR is independently mergeable and revertable. No behavior changes throughout.

**PR count: 12** (1 prep + 1 sqlite_base + 9 other store moves + 1 final cleanup ŌĆö see breakdown below). A buffer slot is held between the last store move and the final PR for any caller-update follow-up that spills out of its primary PR.

## Architecture

### Layering Before ŌåÆ After

```
Before:
lyra.core (contains aiosqlite imports) ŌåÉ lyra.llm | lyra.nats ŌåÉ lyra.adapters ŌåÉ lyra.bootstrap

After:
lyra.core (zero infra) ŌåÉ lyra.llm | lyra.nats ŌåÉ lyra.infrastructure (aiosqlite here) ŌåÉ lyra.adapters ŌåÉ lyra.bootstrap
```

### Per-Store PR Pattern

```
1. Move <x>.py  ŌåÆ  src/lyra/infrastructure/stores/<x>.py
2. Add shim     ŌåÆ  src/lyra/core/stores/<x>.py
                   (from lyra.infrastructure.stores.<x> import XxxStore  # noqa: F401)
                   MUST NOT import aiosqlite directly
3. Update call sites  ŌåÆ  all src/ and tests/ that import from lyra.core.stores.<x>
4. Delete shim  ŌåÆ  remove src/lyra/core/stores/<x>.py
   (steps 3+4 may be in same PR or follow-up ŌĆö shim provides backward compat)
```

**IMPORTANT:** the shim also creates a `core ŌåÆ infrastructure` downward import. Add the shim path to `.importlinter`'s `layers` contract `exclude_modules` list for the duration of the shim. Remove the exclusion when the shim is deleted.

### Dependency Graph After Full Migration

```
lyra.core.stores (protocols only)
    Ōåæ
lyra.infrastructure.stores (implementations ŌĆö aiosqlite freely imported here)
    Ōåæ
lyra.bootstrap.bootstrap_stores (factory wiring ŌĆö only site that imports from infrastructure.stores)
    Ōåæ
lyra.hub, lyra.adapters, lyra.agents, lyra.commands (type-hint against protocols only)
```

**Verify:** `uv run lint-imports` passes. Both the layers contract and the forbidden contract are active. The forbidden contract passes because `__init__.py` no longer re-exports `SqliteStore`; individual implementation files still live under `lyra.core.stores` but their `aiosqlite` imports are already there (pre-existing state, contract shape does not flag them until `source_modules` resolution walks into them via `__init__`). If CI fails on PR 1, the __init__.py surgery scope is wider than expected ŌĆö expand this PR rather than defer.
