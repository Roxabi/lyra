## Summary

Slice 2 of #534: migrate both adapters to publish `InboundMessage(modality="voice")` directly on the unified NATS subject, then delete every piece of legacy infrastructure that Slice 1 kept alive. After this PR there is no `InboundAudio`, no `lyra.inbound.audio.*` subject tree, no compat shim, no `inbound_audio_bus`, and no `audio_pipeline.py`.

## Architecture — end state

```
Text:  adapter → InboundMessage(modality="text") ──┐
Voice: adapter → InboundMessage(modality="voice",  ├→ lyra.inbound.<platform>.<bot_id>
                                audio=AudioPayload) ┘    → NatsBus[InboundMessage] → hub → pipeline
```

All legacy paths (`lyra.inbound.audio.*`, `InboundAudioLegacyHandler`, `InboundAudio`, `InboundAudioBus`) are deleted. `NatsChannelProxy.normalize_audio` stub is removed. `core/audio_pipeline.py` becomes `core/tts_dispatch.py`.
