lyra CLI — Server Commands
bashlyra server
lyra start            # start the FastAPI server (uvicorn)
lyra stop             # stop the server
lyra status           # show server and adapter status
lyra reload           # reload config without full restart
lyra CLI — Agent Commands
CommandDescription
lyra agent initSeed DB from TOML (safe, skips existing rows)
lyra agent init --forceOverwrite existing DB rows from TOML
lyra agent create <name>Scaffold new agent TOML file

lyra agent edit <name>Interactive DB editor — bumps updated_at for hot-reload
lyra agent validate <name>Validate agent config schema

lyra agent unassign --platform <p> --bot <id>Remove agent assignment from bot
lyra agent delete <name>Delete agent (refuses if bot still assigned)
Make Targets — Local
bashcd ~/projects/lyra
make lyra             # start lyra_telegram + lyra_discord
make lyra reload      # restart both adapters
make lyra stop        # stop both adapters
make lyra status      # status of both adapters
make lyra logs        # tail lyra_telegram stdout
make lyra errors      # tail lyra_telegram stderr

make telegram         # telegram adapter only
make discord          # discord adapter only
make register         # one-time setup on new machine

Deploy to Production
bashMachine 1 deploy
make deploy           # git pull → pytest → supervisorctl restart
make remote status    # check production supervisor status
make remote logs      # tail production stdout
make remote reload    # restart lyra on production (no pull)
Deploy pipeline: make deploy runs pytest before restarting. If tests fail, the old version keeps running. Always check make remote logs after deploy to confirm clean startup.

Worktree testing: Always run [ -d .venv ] || uv sync before pytest in a git worktree — the venv is not shared from the main checkout.
