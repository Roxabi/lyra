## Problem

Code review identified 4 architectural questions requiring consensus:

1. STT/TTS result types location
2. Config loader functions location
3. sys.exit() calls in library code
4. Bare except Exception catches

## Panel

| Agent | Focus |
|-------|-------|
| architect | Architectural soundness, hexagonal pattern |
| backend-dev | Implementation complexity, testability |
| product-lead | Delivery speed, risk, pragmatism |

## Consensus

### Q1: Result Types

**ρ:** Keep `TranscriptionResult` and `SynthesisResult` in infrastructure (`lyra.stt`, `lyra.tts`) with TYPE_CHECKING guard in `core/ports/`.

**Rationale:** Result types are concrete DTOs with infrastructure-shaped fields (audio_bytes, waveform_b64). TYPE_CHECKING guard isolates domain from runtime imports. Moving them to domain adds indirection without functional benefit.

### Q2: Config Loaders

**ρ:** Move `load_telegram_config` and `load_discord_config` to `bootstrap/`. Keep config dataclasses in `core/config.py`.

**Rationale:** Loaders perform env var reads + raise SystemExit — CLI/bootstrap behavior, not domain. ADR-059 V6 already prescribes this.

### Q3: sys.exit() Calls

**ρ:** Convert `sys.exit()` → `raise SystemExit` in bootstrap helpers (wiring_helpers, lockfile). Leave standalone entry points as-is.

**Rationale:** `raise SystemExit` is catchable in tests; bootstrap helpers are library code, not entry points. Low priority (P3).

### Q4: Bare Exception Catches

**ρ:** Accept broad `except Exception` catches at external boundaries (adapters, I/O) with `log.exception()` + `# noqa: BLE001` comment explaining resilience intent.

**Rationale:** External APIs (discord.py, aiogram, nats-py) raise unpredictable exception types. Resilience at boundaries is correct. Requires logging — not silent swallowing.

## Dissent

- Architect preferred moving result types to domain (pure hexagonal pattern)
- Backend preferred keeping config loaders in core/config
- Both accepted majority position

## Implementation Notes

1. **No action for Q1** — status quo is correct
2. **Q2: Move loaders** to `bootstrap/config_loader.py` or `bootstrap/factory/config.py`
3. **Q3: Replace sys.exit()** in wiring_helpers.py, lockfile.py
4. **Q4: Add noqa comments** to existing broad catches, ensure all have logging
