## Problem

Four folders in `src/lyra/` exceed the 12-file cap defined in `.claude/stack.yml` quality gates:

| Folder | Files | Cap |
|---|---|---|
| `src/lyra/core/` | 53 | 12 |
| `src/lyra/adapters/` | 30 | 12 |
| `src/lyra/bootstrap/` | 25 | 12 |
| `src/lyra/core/hub/` | 21 | 12 |

This violates architectural hygiene and makes navigation cognitively expensive. The exemptions are tracked in `tools/folder_exemptions.txt` referencing this issue.

**Why now:** V1–V3 (file-size exemption elimination) merged in #772. V4 was deferred due to higher blast radius (~150+ import-path updates across `src/` and `tests/`). Now proceeding with the deferred work.

## Who

- **Primary:** Maintainers (Mickael) — navigating large folders slows development
- **Secondary:** Future contributors — flat folder structure obscures module boundaries

## Constraints

- **Approach:** Scripted, one-subfolder-at-a-time to minimize blast radius per commit
- **Tooling:** Python AST-based import rewriting (not sed/grep) to ensure correctness
- **Timebox:** ~2 days, 4–5 commits (one per major folder)
- **Dependencies:** Must run green on `pyright`, `pytest`, `lint-imports` after each slice

## Out of Scope

- V1–V3 file-size work (already merged in #772)
- Any functional changes — pure refactoring, behavior unchanged
- `tests/` folder decomposition (test files follow source structure implicitly)

## Complexity

**Tier: F-full** — High blast radius (~150+ import updates), multi-folder coordination, requires scripted tooling to avoid manual errors.

**Signals:**
- XL label on parent #760, L-equivalent scope deferred
