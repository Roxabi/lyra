## Problem

All agent-related tests require a live SQLite `auth.db`. There is no way to test agent
configuration changes without touching the real database, making local iteration slower
and introducing risk of corrupting state during development. CI environments must
provision a real database file, adding setup friction and coupling test runs to disk I/O.

## Who

- **Primary:** Contributors writing or running agent-related tests — affected on every
  test run that touches agent configuration logic.
- **Secondary:** CI pipelines that currently must bootstrap `auth.db` before tests can
  pass.

## Constraints

- `AgentStoreProtocol` already exists and defines the interface — implementation must
  satisfy it exactly.
- The JSON store path should default to `~/.lyra/agents_test.json` and be overridable
  via an environment variable (e.g. `LYRA_AGENT_STORE_PATH` or derived from `LYRA_DB`).
- Swap-in mechanism: `LYRA_DB=json` env var activates `JsonAgentStore` instead of
  `SqliteAgentStore` inside `agent_db_loader.py`.
- No changes to existing agent logic beyond `agent_db_loader.py` — all callers depend
  on the Protocol, so substitution is transparent.

## Out of Scope

- Persistence guarantees or production use of `JsonAgentStore` — this is a test stub only.
- Migration tooling between JSON and SQLite formats.
- Changes to the Telegram/Discord adapter startup paths beyond what `agent_db_loader.py`
  already controls.

## Complexity

**Tier: F-lite** — clear scope, single domain (agent store / testing infrastructure),
`AgentStoreProtocol` already defines the contract, no architectural unknowns.

- 3 files affected: 1 new (`json_agent_store.py`), 1 modified (`agent_db_loader.py`), 1
  fixture update (`tests/`)
- Single domain (agent store abstraction)
- No new patterns introduced — follows existing Protocol pattern
- P2-medium label
