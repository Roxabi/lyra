Setup
Start modes, bot tokens, supervisor configs, and config.toml schema.
Start modes
lyra start (direct)
lyra start
# or: python -m lyra --adapter all
Starts all configured adapters (Telegram + Discord) in the foreground. Good for development and quick testing. Ctrl+C to stop.
make lyra (supervisor)
make lyra          # start lyra_telegram + lyra_discord
make lyra reload   # restart both adapters
make lyra status   # status of both
make lyra logs     # tail lyra_telegram stdout
make lyra errors   # tail lyra_telegram stderr
Recommended for production. Delegates to lyra-stack supervisord. Adapters restart independently.
📌
Single-adapter mode: python -m lyra --adapter telegram or --adapter discord. Discord is skipped even if configured in the other case.
Bot token setup
Telegram — via @BotFather
Open Telegram → search @BotFather
Send /newbot → choose display name & username
Copy token → run lyra bot add --platform telegram --bot-id lyra
Your numeric ID → get from @userinfobot
Set owner_users = [YOUR_TG_ID] in config.toml
bash
lyra bot add --platform telegram --bot-id lyra
# Prompts for token → encrypted in auth.db
lyra bot list   # verify
Discord — via Developer Portal
discord.com/developers → New Application
Bot tab → Reset Token → run lyra bot add --platform discord --bot-id lyra
Enable Message Content Intent ← required!
OAuth2 → scope bot → Send Messages → invite URL
Your ID: Settings → Advanced → Developer Mode → right-click name
bash
lyra bot add --platform discord --bot-id lyra
# Prompts for token → encrypted in auth.db

Supervisor configuration
For production, adapters run as independent supervisor processes — each can restart without affecting the other. Configs live in the lyra repo; lyra-stack/conf.d/ holds only symlinks.
supervisor/lyra_telegram.conf
ini
[program:lyra_telegram]
command     = run.sh --adapter telegram
environment = LYRA_HEALTH_PORT="8443"
autostart   = true
autorestart = true
stdout_logfile = logs/lyra_telegram.log
stderr_logfile = logs/lyra_telegram_err.log
Health endpoint on port 8443.
supervisor/lyra_discord.conf
ini
[program:lyra_discord]
command     = run.sh --adapter discord
environment = LYRA_HEALTH_PORT="8444"
autostart   = true
autorestart = true
stdout_logfile = logs/lyra_discord.log
stderr_logfile = logs/lyra_discord_err.log
Health endpoint on port 8444.
🔗
run.sh sources .env before launching the Python process — secrets never live in supervisor conf files. Register symlinks with make register from the lyra repo.
bash
# One-time: clone lyra-stack alongside lyra, then register
git clone https://github.com/Roxabi/lyra-stack ~/projects/lyra-stack
cd ~/projects/lyra
make register         # creates symlinks in lyra-stack/conf.d/

cd ~/projects/lyra-stack
make start            # start supervisord (idempotent)
make ps               # status of all programs

# Deploy to production (Machine 1)
make deploy           # git pull → pytest → supervisorctl restart
make remote status    # check remote supervisor status
config.toml schema

Minimal single-bot config

tomlconfig.toml
[admin]
user_ids = []   # extra cross-platform admins

[[auth.telegram_bots]]
bot_id      = "lyra"
default     = "blocked"
owner_users = [YOUR_TG_ID]

[[auth.discord_bots]]
bot_id      = "lyra"
default     = "blocked"
owner_users = [YOUR_DC_ID]

[defaults]
cwd = "~/projects"
[defaults.workspaces]
lyra = "~/projects/lyra"

Multi-bot explicit wiring

tomlconfig.toml
# Tokens resolved from CredentialStore (auth.db)
[[telegram.bots]]
bot_id = "lyra"
agent  = "lyra_default"

[[discord.bots]]
bot_id      = "lyra"
auto_thread = true
agent       = "lyra_default"

[[auth.telegram_bots]]
bot_id      = "lyra"
default     = "blocked"
owner_users = [YOUR_TG_ID]

[[auth.discord_bots]]
bot_id      = "lyra"

Agent TOML search locations
User-level (higher precedence)
~/.lyra/agents/
Machine-specific overrides. Gitignored. Takes priority over bundled defaults during lyra agent init.
System defaults (lower precedence)
src/lyra/agents/
Bundled in the repo. Committed. Loaded only when no user override exists for that agent name.
Extensions & environment

[plugins] — extension config

tomlconfig.toml
[plugins]
enabled = ["echo", "search"]
# search enables /search + /add + /explain + /summarize
CommandLoader (#345) discovers both built-in commands and loaded extensions. /help shows both sections.
