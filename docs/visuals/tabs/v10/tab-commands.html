PluginLoader → CommandLoader rename (v10). src/lyra/plugins/ is now src/lyra/commands/. PluginLoader → CommandLoader, PluginReloadManager → CommandReloadManager. The TOML config key [plugins].enabled is intentionally kept unchanged — the [commands] section is used for command metadata and would collide. Plugin names are restricted to [a-zA-Z0-9_] (no hyphens).

Command types

Session commands

Built-in slash commands registered at startup. Work in every conversation regardless of agent.

Workspace commands

Per-agent slash commands from workspaces_json. Each key becomes /workspace <key>, overriding cwd for that message.

Extension commands

Loaded by CommandLoader from src/lyra/commands/. Available when agent has the Skill tool enabled and [plugins].enabled = true.

Extension commands (CommandLoader)

Extension commands are Python packages under src/lyra/commands/. Each command is a directory with a plugin.toml manifest and a handlers.py. Commands are loaded on agent startup and reloaded when updated_at changes via CommandReloadManager.

bashBuilt-in extension commands
src/lyra/commands/
  echo/         # echo command (debug)
  pairing/      # code pairing assistant
  search/       # search integration
  svc/          # service management
Plugin name restriction: command package names must match [a-zA-Z0-9_]. Hyphens are not allowed — they create non-identifier sys.modules keys that leak on reload. Rename my-command to my_command.

patterns_json — per-agent URL rewriting v10

The patterns_json DB column configures per-agent rewriting rules for the CommandRouter. The most common option is bare_url, which automatically rewrites bare URLs (no slash command prefix) into a processing command.

jsonpatterns_json example
{
  "bare_url": true
}
bare_url defaults to false (opt-in). An agent that should auto-process pasted URLs must have "bare_url": true in its patterns_json. Edit via lyra agent edit <name>.

Discord watch channels v10

Discord bots can be configured with watch_channels — a list of channel IDs where the bot auto-processes messages without requiring a @mention. Links posted in these channels are automatically forwarded to the agent.

jsonsettings_json on bot_agent_map
{
  "watch_channels": [1234567890, 9876543210],
  "auto_thread": true
}

Configure via AgentStore.set_bot_settings(platform, bot_id, settings) or directly in the DB. The settings_json column on bot_agent_map is loaded into the in-memory _bot_settings cache on startup.

If auto_thread=True and the channel isn't thread-based, the Discord adapter automatically creates a thread for each processed link. Thread creation failures are non-fatal — the message is still processed.

Workspace commands

Each key in the agent's workspaces_json becomes a /workspace <key> slash command that overrides the working directory for that message.

bashConfigure via DB edit
lyra agent edit my_agent
# Set workspaces_json to:
# {"lyra": "/home/mickael/projects/lyra", "plugins": "/home/mickael/projects/roxabi-plugins"}

Sending /workspace lyra list the recent commits sets cwd=/home/mickael/projects/lyra for that message only.

cwd is machine-specific and lives in config.toml [defaults]. Workspace paths are absolute and override cwd for the scope of a single message.
