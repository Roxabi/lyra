Notes & Risks

Bottleneck — #139 is the highest-leverage pick

Message Normalization (#139) unblocks 5 issues: #99 (command sessions), #80 (STT), #79 (TTS), and indirectly the entire voice pipeline. Should be picked immediately after any in-progress work completes.

Critical — Synchronous Embeddings

sentence-transformers blocks the asyncio event loop without run_in_executor. Phase 1b uses fastembed (ONNX, lighter). Evaluate before adding more embedding calls in Phase 2. Non-negotiable: never call synchronous I/O in the event loop.

Critical — Machine 2 Fallback Not Implemented

Timeout + circuit breaker if Machine 2 is offline is not implemented. Not relevant in Phase 1 (cloud LLM only). Required before #51 in Phase 2. #23 covers this.

VRAM Budget — Measure Before Phase 2 SLMs

Machine 1 VRAM: TTS ~5GB + embeddings ~0.5GB = 5.5GB / 10GB used. 4.5GB headroom. Measure with nvidia-smi under real load before committing to Phase 2 SLMs. Budget is tight.

Rolling Decisions — Still Open

Default LLM for Machine 2 → after benchmark #14 (Qwen vs Mistral).
LegalTech go/no-go → social signals #10 (min: 3 qualified DMs or 1 pricing request).
MedTech go/no-go → social signals #11.
NATS adoption → after Phase 1b tail shipped and stable in production.
YouTube automation → after 1 validated manual workflow.

ADR-010 — External Tool Pattern (Install → Wrap → Declare)

External CLIs (voiceCLI, imageCLI, Google Workspace) follow: Install binary on PATH, Wrap in a thin roxabi-plugins skill (SKILL.md only), Declare in agent TOML (Bash allowlist now, MCP later). No forking, no vendoring. Upstream maintains the CLI.

Security Contract — Never Derive Trust from Inbound Data

Adapters set trust="user" unconditionally. Only Hub internal code may set trust="system". Message.user_id is the authenticated sender identity (rate limiting, pairing). Message.extract_scope_id() is the conversation scope (pool routing). Never derive either from unverified channel data.
