## Problem

All six Python `nats.connect()` call sites (`hub_standalone.py`, `adapter_standalone.py`, `nats_bus.py` ×2, `nats/__init__.py`, test conftest) connect with no `nkeys_seed_file`, `tls`, or `tls_ca_file` argument. The NATS server config (`nats.conf`) already requires nkey auth — it includes `/etc/nats/nkeys/auth.conf` generated by `gen-nkeys.sh` — but production currently deploys `nats-local.conf` (no auth) because the Python side never provides credentials.

This leaves the NATS bus unauthenticated on a host running multiple services, and blocks switching production to the hardened `nats.conf`.

## Who

- **Primary:** Lyra deployment — hub, adapters, and any future NATS client must authenticate.
- **Secondary:** Developers running Lyra locally — must work seamlessly without nkeys (dev mode stays on `nats-local.conf`).

## Constraints

- Seed files live at `/etc/nats/nkeys/{hub,llm-worker,monitor}.seed` (mode 0600, root-owned). Python processes run as `mickael`, so file permissions or a read helper is needed.
- Local dev must remain zero-config: no nkey seed required when connecting to `nats-local.conf`.
- `nats.py` (nats-py) supports `nkeys_seed` (callback returning seed bytes) or `nkeys_seed_file` (path) — both are valid.
- Related: #461 (JWT/account auth) is the broader auth story. This issue wires only nkey auth — no accounts, no JWTs.

## Out of Scope

- TLS certificates (`tls_ca_file`, `cert_file`, `key_file`) — separate hardening step.
- JWT/account-based auth (#461).
- Multi-machine NATS clustering or leafnodes.
- Changing `gen-nkeys.sh` or the server-side config.

## Complexity

**Tier: F-lite** — Clear scope (6 call sites + config + docs), single domain (NATS infra), no architectural unknowns.

Signals:
- 6 `nats.connect()` call sites to update (hub, adapter, nats_bus ×2, nats/__init__)
- 1 shared connection helper likely needed to avoid repeating nkey logic 6 times
- Config: add `NATS_NKEY_SEED_PATH` env var, read in bootstrap
- Deploy docs update for seed file permissions
- Tests: conftest needs optional nkey support
