## Problem

`tools/check_file_length.sh` has 26 exempted files (up from 12 in ADR-027). The exemptions have grown organically without a coordinated reduction effort, undermining the 300-line policy and increasing cognitive load.

Additionally, no folder file count limit is enforced. ADR-025 recommends ≤20 files per directory, but `core/` has 53 files.

## Who

- **Primary:** Developers — maintainability suffers when files exceed cognitive limits
- **Secondary:** Code reviewers — large files increase review burden and miss rates

## Constraints

- Zero behavioral changes — extractions must be pure code moves with re-exports
- All `__init__.py` re-exports preserved — public API unchanged
- Each extraction runnable in parallel — minimize merge conflicts by separating file groups
- Pre-commit hook integration required for new folder limit tool

## Out of Scope

- Functional refactoring (only organizational changes)
- New features or bug fixes
- Documentation beyond code comments
- Performance optimizations

## Complexity

**Tier: F-lite** — Clear scope, single domain (code organization), well-defined task breakdown.

- Effort estimates provided (XS/S/M)
- No architectural unknowns
