CLI Reference
Server management, agent commands, Make targets, deploy pipeline, dev commands.
Server Commands
bashlyra server
lyra start            # start all configured adapters (telegram + discord)

lyra setup commands                    # register slash commands with Telegram (one-time)
lyra setup commands -c prod.toml       # use a specific config file
📌
Server lifecycle (stop, reload, status) is managed via make lyra / supervisord — not CLI subcommands. See Make Targets below.
Agent Commands
Command Description
lyra agent init Seed DB from bundled TOML files (safe — skips existing rows)
lyra agent init --force Overwrite existing DB rows from TOML files
lyra agent create <name> Scaffold a new agent TOML file and import into DB
lyra agent list List all agents with backend, model, and assignment info
lyra agent show <name> Full config for one agent (all DB fields)
lyra agent edit <name> Interactive DB editor — bumps updated_at for hot-reload on save
lyra agent validate <name> Validate agent config schema without starting the daemon
lyra agent assign <name> --platform <p> --bot <id> Assign agent to a bot (creates row in bot_agent_map)
lyra agent unassign --platform <p> --bot <id> Remove agent assignment from a bot
lyra agent delete <name> Delete agent — refuses if a bot is still assigned
💾
DB-first: lyra agent init imports TOML files into ~/.lyra/auth.db. Runtime reads DB only. TOML files are seed sources — edit them, then re-run init --force to propagate changes.
Make Targets — Local
bashcd ~/projects/lyra
make lyra             # start lyra_telegram + lyra_discord via supervisord
make lyra reload      # restart both adapters
make lyra stop        # stop both adapters
make lyra status      # status of both adapters
make lyra logs        # tail lyra_telegram stdout
make lyra errors      # tail lyra_telegram stderr

make telegram         # lyra_telegram only (start|reload|stop|logs|errors)
make discord          # lyra_discord only  (start|reload|stop|logs|errors)
make register         # one-time setup on a new machine

Deploy to Production
bashMachine 1 deploy — from ~/projects/lyra on Machine 2
make deploy           # git pull → pytest → supervisorctl restart on Machine 1
make remote status    # check production supervisor status
make remote logs      # tail production stdout
make remote reload    # restart lyra on production (no git pull)
⚠
Deploy pipeline: make deploy runs pytest before restarting. If tests fail, the old version keeps running — the deploy aborts cleanly. Always check make remote logs after deploy to confirm clean startup.

Dev Commands
bashdevelopment workflow
uv run pytest         # run full test suite
uv run ruff check .   # lint (checks only)
uv run ruff format .  # auto-format all Python files
uv run pyright        # type check (strict mode)
💡
Worktree testing: always run [ -d .venv ] || uv sync before pytest in a git worktree. The .venv is not shared from the main checkout — each worktree needs its own virtual environment.
bashworktree setup
# In a fresh worktree (e.g. from git worktree add)
[ -d .venv ] || uv sync   # create venv only if missing
uv run pytest              # now safe to run tests
