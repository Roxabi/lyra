## Problem

`src/lyra/core/` has grown to 71 files — 3.5× the recommended 20-file ceiling. Every agent
session that touches core (debugging, feature work, review) must load a mental model of a
flat directory with no discernible structure. Searching for a store, a pool primitive, or a
hub component means scanning 71 entries instead of navigating to a named subdirectory.

The cost is not hypothetical: Wave 1 analysis found that core/ was the #1 contributor to
agent context overhead and the first directory called out in the new CLAUDE.md files (PR #407).
The `ROADMAP.md` refactoring policy (PR #404) set the 20-file ceiling as the enforcement
threshold; core/ was already 3× over it before that commit landed.

There is no architectural ambiguity — the groupings are already self-evident from naming
conventions: `*_store.py`, `hub*.py`, `pool*.py`, `command*.py`. The work is a pure
structural reorganisation: create four subdirectories, move files, update imports.

## Who

- **Primary:** AI coding agents — every core/ session starts with a directory scan. Reducing
  71 entries to 4 subdirectories of ≤20 files each cuts initial context load significantly.
- **Secondary:** Human developers — faster `grep`, clearer `git diff`, easier PR reviews when
  changes are scoped to `core/stores/` vs the entire flat `core/`.

## Constraints

- **Import blast radius:** 44 files outside `core/` import from `lyra.core.*`; ~25 files
  inside `core/` import each other. Every moved file requires its import paths updated in
  all consumers.
- **No API surface change:** public symbols (`lyra.core.hub`, `lyra.core.agent_store`, etc.)
  must remain importable via `core/__init__.py` re-exports or the move is breaking.
- **Top-level `src/lyra/commands/` already exists** — contains plugin commands (echo, pairing,
  search, svc). `core/commands/` must be clearly scoped to internal routing infrastructure
  (loader, parser, registry, router) to avoid confusion.
- **`processors/` subdir already exists** — not in scope; preserve as-is.
- **`CLAUDE.md` in `core/`** (added PR #407) must be updated post-split to reflect the new
  layout and add sub-`CLAUDE.md` files per subdirectory.
- **Full test suite must pass green** — no regressions. CI is the acceptance gate.
- **Wave 2 dependency:** Wave 3 issues (#396 file-size hook, #397 pyright strict) depend on
  a stable `core/` structure. This must land cleanly before those start.

## Out of Scope

- Renaming any class, function, or public symbol — pure moves only.
- Splitting `adapters/`, `llm/`, `agents/`, `bootstrap/`, or any package outside `core/`.
- Moving `processors/` (already a subdirectory, already compliant).
- Changing test structure beyond updating import paths in existing test files.
- Adding new functionality — this is a structural refactor only.
- Updating `docs/ARCHITECTURE.md` diagrams (a follow-up doc-sync task).

## Complexity

**Tier: F-full** — Large-scale structural refactor touching ~70 files across 6 packages.
Import blast radius, risk of circular imports in new `__init__.py` re-exports, and the
Wave 3 dependency make careful analysis and a detailed plan mandatory.

Signals observed:
- 71 files in scope, ~70 import-update sites
- Cross-package fan-out: `adapters/`, `llm/`, `agents/`, `bootstrap/`, `commands/`, `cli_*.py`
- New `__init__.py` re-export pattern required (no prior art in this codebase)
- Wave 3 issues (#396, #397) are blocked on this landing cleanly
