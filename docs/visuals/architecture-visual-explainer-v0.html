02
Security & Routing Layer
AuthMiddleware → RoutingContext → CommandParser — four layers of trust and correctness
#151 — P0 · Size S · Ready
AuthMiddleware + TrustLevel
Auth at Adapter level — before the Bus. A BLOCKED message never reaches the queue or consumes LLM tokens.
OWNER
Full access, all commands, admin ops
TRUSTED
Normal access, standard features
PUBLIC
Limited access (if enabled)
BLOCKED
Dropped silently, never reaches Bus
#152 — P0 · Size M · Blocks #151
RoutingContext Verification
Every Response carries a RoutingContext populated at inbound normalization. Outbound adapter verifies channel + bot_id before sending.
FieldPurpose

thread_idForum thread or Discord thread
reply_to_message_idNative threading
session_idConversation continuity key

5-Level Memory System
From volatile working memory to persistent semantic knowledge — strict user_id isolation throughout
#83 — P1 · Extended with strict isolation + full metadata schema
Memory Isolation Rule — Non-Negotiable
Every L4 query must include WHERE user_id = :user_id. L3 path always contains {user_id}/. No global queries ever — even for stats, aggregate by user.
