## Problem

Two related issues in `core/hub/pool_manager.py:70-77`:

1. **Race condition** — iterating `self.pools.items()` while calling `pop()` is not atomic. Narrow window but real under reaper + new-message simultaneity.
2. **Unbounded growth** (architect S-01 / D-05) — TTL-only eviction; memory grows without cap under high concurrent-scope load until TTL fires.

Both fixes touch the same eviction code path — bundling avoids churn.
