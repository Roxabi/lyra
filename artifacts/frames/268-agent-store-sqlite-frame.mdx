## Problem

Agent configuration in Lyra is fully static: each agent lives in a TOML file
(`src/lyra/agents/<name>.toml`) and bot↔agent associations are hardcoded in
`config.toml`. This means:

- **No runtime CRUD** — adding, editing, or deleting an agent requires a file
  edit + restart.
- **No dynamic reassignment** — changing which bot uses which agent requires
  editing config.toml + restart.
- **No programmatic access** — the wizard (`lyra agent create`) writes TOML
  files that nothing can query without parsing files.
- **No runtime state** — agent liveness, active pool count, last activity are
  ephemeral and invisible.

The `lyra agent` CLI is equally limited: it only wraps TOML file I/O and has
no concept of DB state, initialization, assignment, or live update. Commands
like `edit`, `delete`, `assign`, `init` (seed DB from existing TOMLs) don't
exist yet.

The rest of Lyra's persistent state (auth grants, bot credentials, user prefs)
already lives in SQLite (`auth.db`). Agent management is the last holdout.

## Who

- **Primary:** Lyra runtime — Hub, Pool, CLI all need read/write access to
  agent definitions via `AgentStore`.
- **Secondary:** operators / admins who init, create, edit, delete, or reassign
  agents at runtime via `lyra agent` subcommands without restarting the hub.

## Constraints

- Must integrate with `auth.db` (shared WAL, same aiosqlite pattern as
  `AuthStore`, `CredentialStore`, `PrefsStore`) **or** use a dedicated
  `agents.db` if table count warrants separation — decision deferred to spec.
- TOML files are **not deleted** — they become seed documents (read once on
  first startup via `lyra agent init`, DB is authoritative thereafter).
- Pool/Hub must continue to call `get_agent(name)` synchronously from cache;
  DB reads are async and happen only on startup / mutation.
- No encryption required — agent config is not sensitive.
- `lyra agent create` wizard UX must be preserved (interactive prompts), but
  output goes to DB instead of a TOML file.

## CLI surface — current vs target

| Command | Current | Target |
|---------|---------|--------|
| `lyra agent create` | wizard → writes TOML file | wizard → inserts into DB |
| `lyra agent list` | reads all TOML files | reads from DB (+ source column: db/toml-only) |
| `lyra agent validate` | parses TOML + schema check | validates DB row + runtime constraints |
| `lyra agent init` | ✗ missing | seeds DB from existing TOML files (one-time migration) |
| `lyra agent show <name>` | ✗ missing | prints full agent config from DB |
| `lyra agent edit <name>` | ✗ missing | interactive update of a DB row |
| `lyra agent delete <name>` | ✗ missing | removes agent from DB (guard: refuse if bots assigned) |
| `lyra agent assign <name> --bot <id>` | ✗ missing | sets bot↔agent mapping in DB |
| `lyra agent unassign --bot <id>` | ✗ missing | removes a bot↔agent mapping |

## Out of Scope

- Encryption of agent fields.
- Remote/multi-node agent config sync.
- Plugin hot-reload triggered by DB change.
- Agent versioning / rollback history.

## Complexity

**Tier: F-full** — New DB schema across 4 tables, TOML seed migration,
write-through cache in Hub, full CLI CRUD overhaul (9 commands), and Pool/Hub
wiring.

Signals observed:
