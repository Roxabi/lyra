capture + index
Capture Service

Extracts memories from agent output via <memory_note> XML tags. Hybrid capture: durable notes go to both native markdown and registry; ephemeral notes go native-only.

Pipeline
Parse tags → Junk filter → Plausibility check → Exact dedupe (normalized hash) → Semantic dedupe (Jaccard similarity) → Quality gate → Upsert to memory_current
Dedupe Thresholds
Above 0.92 → auto-merge. Between 0.85–0.92 → review queue. Type-aware: USER_FACT/PREFERENCE use 0.78/0.62.
Remember Intent
Detects phrases like "remember this" / "merk dir". Writes native markdown + registry entry when durable. Queues review if model omits the <memory_note> tag.
Native Sync + Promotion

Indexes workspace markdown files into SQLite chunks, then promotes durable chunks to structured registry memories with provenance tracking.

Native Sync
Scans MEMORY.md, daily notes (YYYY-MM-DD.md), curated files. Parses bullets under headings into chunks. Stores in memory_native_chunks with source_kind (memory_md / daily_note / curated).
Native Promotion
Promotes chunks to memory_current with source_layer: promoted_native. Skips CONTEXT/EPISODE types and ephemeral content. Links chunk to registry via linked_memory_id. Min confidence: 0.72.
Person Service
Extracts proper names from all memories. Builds memory_entity_mentions table. Classifies roles: relationship, public_profile, ops_noise. Enables entity-aware recall with priority boosts.

Quality Policy Engine
Junk Filter
32 base patterns block system prompts, API keys, benchmark artifacts, XML wrappers. Configurable append/replace.
Durable Patterns
Personal preferences, relationships, agent identity, continuity facts get boosted retention. Always kept regardless of score.
Plausibility Heuristics
Broken phrase detection, entityless numeric facts, low-confidence generic extractions, token anomaly patterns.
Value Classification
9-feature weighted score → keep (≥0.78), archive (0.30–0.78), reject (≤0.18). Durable/relationship overrides bypass thresholds.
Semantic Similarity
Composite Jaccard: word overlap (35%) + char trigrams (25%) + numeric tokens (20%) + semantic anchors (20%). Type-aware thresholds.
Memory Types
USER_FACT, PREFERENCE, DECISION, ENTITY, EPISODE, AGENT_IDENTITY, CONTEXT. Each type has distinct base utility and dedupe thresholds.
Stack: Node.js ≥22 (node:sqlite experimental API), TypeScript entry point, JavaScript core services. No embedding model — all similarity is token-level Jaccard, no vectors. Recall uses LIKE queries, not vector search. LLM optional — Ollama/OpenAI for audit second opinion only. Core capture/recall is fully deterministic. Event sourcing: append-only event log + materialized projection (CQRS pattern).
