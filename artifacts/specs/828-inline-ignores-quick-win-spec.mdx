## Context

Promoted from [frame #828](../frames/828-inline-ignores-quick-win-frame.mdx). Audit identified **60 mechanical ignores** across 5 buckets in ~25 files (tests + src + 1 script). Scope was expanded from "test-only" after grep verification — src ignores share the same fix patterns and stay in scope.

**Why these 5 buckets, not the larger structural ones:** `PLR0913` (68), `C901` (37), `type: ignore[attr-defined]` (42) — 147 ignores — require architectural refactor (DI containers, mixin → Protocol, function extraction). They are deferred to a dedicated structural-debt frame. The 5 buckets in this spec are mechanical substitutions using stdlib primitives — no design decisions, no API changes, bounded risk.

## Goal

Drop **60 inline ignores** (5 kinds: `arg-type`, `ARG002`, `assignment`, `method-assign`, `PLC0415`) to **zero** across enumerated paths, using stdlib primitives (`unittest.mock`, `functools.partial`, `TYPE_CHECKING`). No behavior change. Green `make validate`.

## Users

- **Primary:** developers maintaining lyra — navigate/maintain test suite and src hotspots (bootstrap, hub, nats) with clearer types and no suppressed lint/type noise.
- **Secondary:** future contributors — fewer "why is this ignored?" questions during review; tighter precedent for new code.

## Expected Behavior

Before: 394 total ignores, 60 of the 5 targeted kinds.
After: 394 − 60 = 334 total (minus any secondary ignores surfaced). Test + lint + type pipelines remain green. No runtime behavior change — only annotations, test doubles, and import placement.

Any ignore that resists mechanical removal (genuinely justified, e.g. unavoidable third-party typing gap) → **moved to a `justified` bucket with an inline reason**, not silently retained. Target for silent-ignore count = 0.

## Ignore Taxonomy & Fix Patterns

### Per-bucket fix patterns

| Bucket | Detect | Fix pattern (priority order) | Escape hatch (if truly justified) |
|---|---|---|---|
| `arg-type` | `# type: ignore[arg-type]` | 1. Replace fake stub with `MagicMock(spec=Proto)` — 2. Narrow call-site annotation — 3. Widen Proto | `# justified: <reason>` |
| `ARG002` | `# noqa: ARG002` | 1. Delete param if unused and not protocol-required — 2. `_name` prefix if protocol requires the slot — 3. Wrap via `functools.partial` if the signature is pinned by a handler registry | `# justified: callback protocol requires slot` |
| `assignment` | `# type: ignore[assignment]` | 1. **Prefer** `mock.patch.object(target, "attr", value)` context manager / decorator — 2. `cast(T, value)` only if target is not patchable (module-level constants, frozen dataclasses) | `# justified: narrowing to Protocol T` |
| `method-assign` | `# type: ignore[method-assign]` | **Two sub-cases — distinguish before fixing:** (a) replacing with a mock → `mock.patch.object(obj, "method", new=AsyncMock(spec=obj.method))` with `autospec=True`; (b) replacing with a tracking closure / spy → `mock.patch.object(obj, "method", side_effect=fn)` — **do NOT** use `autospec=True` with `side_effect` since `autospec` overrides the body | `# justified: <reason>` |
| `PLC0415` | `# noqa: PLC0415` | **Classify first:** (a) defensive lazy (dep always available) → lift to top-level; (b) optional-dependency guard (e.g. `pynvml`) → wrap in `try/except ImportError` at module scope, not `TYPE_CHECKING`; (c) circular-import break → keep as lazy import, add `# justified: breaks import cycle with X` | `# justified: <reason>` |

**Known must-justify cases (identified during review):**
- `src/lyra/core/hub/middleware_stages.py:128` — `from .hub import RoutingKey` is a confirmed circular break; keep lazy with `# justified:`.

**Known safe-to-lift cases:**
- `src/lyra/core/runtime_config.py:95` — `pydantic_core` ships with `pydantic`, always importable; lift to top-level.
- `src/lyra/bootstrap/{stt,tts}_adapter_standalone.py` — `pynvml` is optional; use top-level `try/except ImportError` guard (not `TYPE_CHECKING`, since `nvmlInit()` is called at runtime).

## Success Criteria

Let `$paths` := the Target Paths list above. Each check runs as a shell one-liner returning exit 1 (no match) for the pattern-count checks.

- [ ] `rg -l 'type:\s*ignore\[arg-type\]' $paths | grep -v 'justified:' ; test $? -eq 1`
- [ ] `rg -l 'noqa:.*ARG002' $paths | grep -v 'justified:' ; test $? -eq 1`
- [ ] `rg -l 'type:\s*ignore\[assignment\]' $paths | grep -v 'justified:' ; test $? -eq 1`
- [ ] `rg -l 'type:\s*ignore\[method-assign\]' $paths | grep -v 'justified:' ; test $? -eq 1`
- [ ] `rg -l 'noqa:.*PLC0415' $paths | grep -v 'justified:' ; test $? -eq 1`
- [ ] Machine-verifiable silent-ignore audit: `rg '(type:\s*ignore|noqa:)' $paths | grep -vE 'justified:|^[^:]+:[0-9]+:\s*$' | wc -l` reports **≤ 5 across all 5 buckets combined** (ceiling for `# justified:` retentions — more = stop signal, deeper refactor needed)
- [ ] `make validate` passes (lint + typecheck + tests + file-length + folder-size + import-layers)
- [ ] Pytest coverage ≥ pre-PR baseline (no regression)
- [ ] PR contains 5 bucket-scoped commits (one per bucket) — bisectable
- [ ] No edits outside enumerated target paths
- [ ] No new runtime dependencies added to `pyproject.toml` ([project.dependencies], [tool.uv.sources] unchanged)

**Note:** the `# justified:` convention is a review discipline, not a toolchain-enforced gate (ruff/pyright do not require inline reasons). A follow-up issue may add a pre-commit hook that rejects bare `# type: ignore` / `# noqa` without `justified:`. Out of scope here.

## Edge Cases

| Edge | Handling |
|---|---|
| Removing an ignore reveals a real bug | Fix the bug in the same commit as the removal; note in commit body |
| Removing an ignore requires a non-trivial refactor | Leave the ignore with `# justified:` + open a follow-up issue; do not silently defer |
| A bucket grows beyond 3h of work | Stop, note state in commit, flag to user before continuing — likely structural, not quick-win |
| `patch.object` breaks existing test assertions | Revert to pre-slice commit, note in PR body, keep ignore with `# justified:` |
| Pyright surfaces new ignores as a side effect of narrowing | Accept if ≤ 2 per slice; > 2 → roll slice back, analyze |
| Import cycle blocks `PLC0415` lift | Use `TYPE_CHECKING` block + `# justified:` if still needed at runtime |
