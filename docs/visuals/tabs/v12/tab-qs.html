Quick Start
Get Lyra running in 5 steps — from zero to first message.
1
Prerequisites
Install these before cloning.
Python 3.12+
python --version
# Need 3.12+. pyenv or deadsnakes PPA.
Uses StrEnum, tomllib — older versions fail at import.
uv
pip install uv
# or: curl -Ls https://astral.sh/uv/install.sh | sh
Fast Python package manager. Replaces pip + virtualenv.
Claude Code CLI
claude --version
# claude.ai/download
Default LLM backend. Requires an active Claude subscription.
💡
Claude CLI is the default backend. Lyra spawns claude --input-format stream-json as a subprocess. You need a Claude subscription. For API key access instead, set backend = "anthropic-sdk" and provide ANTHROPIC_API_KEY in your .env.
2
Clone & Install
bash
git clone https://github.com/roxabi/lyra
cd lyra
uv sync

# Put lyra CLI on your PATH
source .venv/bin/activate
# Or permanently in ~/.bashrc:
# export PATH="$HOME/projects/lyra/.venv/bin:$PATH"

# Verify
lyra --version
3
Configure .env & config.toml

.env — secrets

bash.env
# Bot tokens go in auth.db — NOT here.
# Use: lyra bot add --platform telegram --bot-id lyra

# Voice STT (optional — omit to disable)
STT_MODEL_SIZE=small

# Knowledge vault (optional)
ROXABI_VAULT_DIR=~/.roxabi-vault

# Health / config secrets (optional)
LYRA_HEALTH_SECRET=...
LYRA_CONFIG_SECRET=...
🔐
Bot tokens are encrypted in auth.db — not in .env or config.toml. Run lyra bot add once after install.

config.toml — auth (gitignored)

bash
cp config.toml.example config.toml
nano config.toml
tomlconfig.toml
[auth.telegram]
default     = "blocked"
owner_users = [YOUR_TG_ID]

[auth.discord]
default     = "blocked"
owner_users = [YOUR_DC_ID]

[defaults]
cwd = "~/projects"
👤
Get Telegram ID from @userinfobot. Get Discord ID: Settings → Advanced → Developer Mode → right-click username.
4
Seed agents & assign bot
bash
# Seed DB from bundled TOML files (first-time setup)
lyra agent init

# Verify agents loaded
lyra agent list

# Store bot token (encrypted in auth.db)
lyra bot add --platform telegram --bot-id lyra

# Assign agent to bot (creates row in bot_agent_map)
lyra agent assign lyra --platform telegram --bot lyra
💾
DB-first: lyra agent init imports TOML files into ~/.lyra/auth.db. Runtime reads DB only. TOML files are seed sources — edit them, then re-run init --force to update.
5
Start Lyra
make lyra (supervisor)
# Recommended for production
make lyra          # start via supervisord
make lyra status   # verify running
make lyra logs     # tail stdout
make lyra errors   # tail stderr
Delegates to lyra-stack supervisord. Adapters restart independently.
lyra start (direct)
lyra start
output
INFO lyra.bootstrap.agent_factory: Agent loaded: name=lyra model=claude-sonnet-4-6
INFO lyra.bootstrap.multibot: Lyra started — adapters: telegram, discord
INFO lyra.adapters.discord: Discord bot ready: YourBot#1234
Good for dev. Runs in foreground, Ctrl+C to stop.
What makes Lyra different
Full Voice Loop
Send a voice message → Whisper transcribes → Claude responds → TTS replies with audio. Full voice conversations on Telegram and Discord.
Paste a Link, Save Knowledge
Just paste a URL. Lyra auto-saves it to your personal knowledge vault with title and tags. Search later with /search. Build your second brain via chat.
Code Assistant via Chat
Type /workspace lyra what's the last commit? — Lyra switches Claude's working directory to your project and answers. Full code workspace access from Telegram or Discord.

Troubleshooting
Python version too old
python --version
# Need 3.12+. Use pyenv or deadsnakes PPA.
Lyra uses StrEnum, tomllib. Older versions fail at import.
Claude CLI not authenticated
claude --version
claude   # interactive — works = authenticated
Visit claude.ai/download to install and authenticate.
Port 8443 already in use
LYRA_HEALTH_PORT=9443 lyra start
# Or in supervisor: environment=LYRA_HEALTH_PORT="9443"
Default health port is 8443. Override via env var.
Bot token invalid or missing
lyra bot list
lyra bot add --platform telegram --bot-id lyra
Tokens are encrypted in auth.db. Re-add if bot fails to start.
config.toml errors
lyra config validate
lyra config show
Common issues: missing owner_users, wrong TOML syntax.
uv sync fails
pip install --upgrade uv
uv sync --refresh
Ensure uv is up to date. --refresh clears the cache.
