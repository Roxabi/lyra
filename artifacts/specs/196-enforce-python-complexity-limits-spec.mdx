## Goal

Enforce standard Python complexity and size limits via ruff so that overly complex or oversized code is caught automatically before merge.

## Users

- **Primary:** Lyra developers — guardrails prevent complexity creep
- **Secondary:** Reviewers — automated checks reduce review burden

## Expected Behavior

1. Developer runs `uv run ruff check .` (or pre-commit fires)
2. Ruff reports violations for functions exceeding complexity 10, 50 statements, or 5 args
3. A pre-commit hook flags files exceeding 500 lines
4. CI catches any violations that slip past local hooks
5. Existing violations are either fixed (preferred) or annotated with justified `noqa`
