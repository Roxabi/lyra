Command types

Session commands

Built-in slash commands registered at startup. Work in every conversation.

Workspace commands

Per-agent slash commands defined in the agent's workspaces config. Override cwd for that message.

Plugin / Skill commands

Commands provided by loaded plugins. Available if the agent has the Skill tool enabled.

Session commands

Workspace commands

Each key in the agent's workspaces config becomes a /workspace <key> slash command that overrides the working directory for that message.

bashConfigure via DB edit
lyra agent edit my_agent
# Add to workspaces field:
# workspaces = { lyra = "/home/mickael/projects/lyra", plugins = "/home/mickael/projects/roxabi-plugins" }

With the above config, sending /workspace lyra list the recent commits will run with cwd=/home/mickael/projects/lyra and execute the rest of the message as the prompt.

cwd is machine-specific and lives in config.toml [defaults], not in the agent config. Workspaces are absolute paths and override cwd for the scope of a single message.
