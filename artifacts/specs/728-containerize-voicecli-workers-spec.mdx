## Context

Promoted from `artifacts/analyses/728-containerize-voicecli-workers-analysis.mdx`.
Supersedes #609 (closed). Blocked on #691 + #727 — both now CLOSED.

Architecture and design are resolved. The voiceCLI repo already ships:
- `Dockerfile` (CUDA 12.5.1, UID 1501, `voicecli nats-serve {stt|tts}` entrypoint)
- `deploy/quadlet/voicecli-{stt,tts}.container` + `voicecli-models.volume`
- `publish.yml` (Roxabi/.github reusable workflow; `ghcr.io/roxabi/voicecli:staging` published)

Two implementation gaps block cutover; this spec closes them and drives M₁ activation.

## Goal

`voicecli-stt` and `voicecli-tts` running as Podman Quadlet user services on M₁, with GPU
passthrough, persistent model cache, and `make voice-smoke` green — supervisord programs retired.

## Users

- **Primary:** ops/Mickael — installs, activates, and monitors the services on M₁
- **Secondary:** lyra engine — consumes voiceCLI workers via NATS; `make voice-smoke` is the integration signal

## Expected Behavior

A voiceCLI PR fixes two gaps in the Quadlet container files: adds the missing `NATS_NKEY_SEED_PATH`
environment variable (secret is already mounted; path just needs exporting) and adds `ReadOnly=true`
to match ADR-053/054 hardening. CI publishes an updated `ghcr.io/roxabi/voicecli:staging` image.

On M₁, prior to activation:
1. NATS auth.conf is verified to contain voicecli identities (added in #691; confirm in place).
2. Two Podman secrets are provisioned from existing nkey seed files.
3. The `voicecli-models` named volume is pre-seeded from the existing `~/.cache/huggingface` and
   `~/.cache/voicecli` directories — avoiding an 8GB cold download on first container start.

Quadlet units are installed (`~/.config/containers/systemd/`) and `systemctl --user daemon-reload`
is run. Both services start; GPU is confirmed visible inside containers via `nvidia-smi`; VRAM
headroom after both services steady-state is ≥ 1GB. `make voice-smoke` exits 0.

Once smoke is confirmed, the supervisord DEPRECATED stubs (`conf.d/voicecli_{stt,tts}.conf`) are
removed and `supervisorctl reread && update` is run. Docs are updated in both repos.

**Rollback (if smoke fails):** `systemctl --user stop voicecli-{stt,tts}` — supervisord stubs are
still present and startable. Stubs must not be deleted until smoke is confirmed green.

## Data Model & Consumers

### Quadlet unit configuration (required fields)

```
classDiagram
    class VoicecliContainer {
        +Image: ghcr.io/roxabi/voicecli:staging
        +Network: roxabi.network
        +Device: nvidia.com/gpu=all
        +ReadOnly: true
        +UserNS: keep-id uid=1501 gid=1501
        +NoNewPrivileges: true
        +DropCapability: all
        +NATS_URL: nats://lyra-nats:4222
        +NATS_NKEY_SEED_PATH: /home/voicecli/.config/voicecli/nkeys/seed.seed
        +Secret: voicecli-nats-{stt|tts}
        +Volume: voicecli-models → .cache/huggingface + .cache/voicecli
        +RestartSec: 10
    }

    class ModelsVolume {
        +VolumeName: voicecli-models
        +Mountpoints: .cache/huggingface, .cache/voicecli
    }

    class PodmanSecret {
        +Name: voicecli-nats-stt | voicecli-nats-tts
        +Source: ~/.voicecli/nkeys/voice-{stt|tts}.seed
        +MountTarget: /home/voicecli/.config/voicecli/nkeys/seed.seed
        +Mode: 0400
    }

    VoicecliContainer --> ModelsVolume : mounts rw
    VoicecliContainer --> PodmanSecret : secret mount
```

## Breadboard

### Component wiring

| Component | Interface | Wires to | Constraint |
|---|---|---|---|
| **voicecli-{stt,tts}.container** | `Exec=voicecli nats-serve {stt\|tts}` | lyra-nats:4222 via NATS | NATS_NKEY_SEED_PATH must point to mounted secret |
| **PodmanSecret voicecli-nats-{stt\|tts}** | `type=mount, mode=0400, uid=1501` | Container at `/home/voicecli/.config/voicecli/nkeys/seed.seed` | NATS_NKEY_SEED_PATH must export this path |
| **voicecli-models volume** | rw named volume | `.cache/huggingface` + `.cache/voicecli` in both containers | Pre-seeded before first start; ReadOnly=true does not affect volume mounts |
| **CDI GPU** | `Device=nvidia.com/gpu=all` | Container process (CUDA runtime) | CDI validated for RTX 3080 (Risk 5 closed 2026-04-24) |
| **lyra-nats** (roxabi.network) | NATS 4222 | Both voicecli containers; lyra-hub | Shared NATS — big-bang consolidation #611; no separate voicecli-nats needed |
| **supervisorctl** | conf.d stubs | REMOVED post-cutover | Must not remove before smoke green |

### Operation wiring

| Operation | Pre-condition | Action | Post-condition |
|---|---|---|---|
| Fix Quadlet gaps | voiceCLI PR open | Add `NATS_NKEY_SEED_PATH` + `ReadOnly=true` to both .container files | PR merged; CI publishes updated staging image |
| Provision secrets | PR merged; nkey seeds exist at `~/.voicecli/nkeys/` | `podman secret create voicecli-nats-{stt,tts} <seed>` | `podman secret ls` shows both names |
| Pre-seed volume | `~/.cache/huggingface` exists on M₁ | `podman run --rm -v ~/.cache/huggingface:/src/huggingface:ro -v ~/.cache/voicecli:/src/voicecli:ro -v voicecli-models:/dst docker.io/library/alpine sh -c "cp -a /src/huggingface /dst/ && cp -a /src/voicecli /dst/"` | Volume contains models; first container start uses cache |
| Install units | Quadlet files fixed; `~/.config/containers/systemd/` writable | `cp *.container *.volume ~/.config/containers/systemd/ && systemctl --user daemon-reload` | Units recognised by systemd |
| Activate | Units installed; secrets provisioned; volume seeded | `systemctl --user start voicecli-{stt,tts}.service` | Both services `active` |
| Verify GPU | Containers active | `nvidia-smi` from inside container (or check service logs) | GPU device visible |
| Smoke test | Both services active | `make voice-smoke` | Exit 0 |
| Cleanup | Smoke green | `rm conf.d/voicecli_{stt,tts}.conf && supervisorctl reread && update` | Stubs absent; supervisord no longer shows voicecli programs |

## Success Criteria

### Slice 1 — Quadlet gaps fixed

- [ ] `voicecli-stt.container` contains `Environment=NATS_NKEY_SEED_PATH=/home/voicecli/.config/voicecli/nkeys/seed.seed`
- [ ] `voicecli-tts.container` contains `Environment=NATS_NKEY_SEED_PATH=/home/voicecli/.config/voicecli/nkeys/seed.seed`
- [ ] Both container files contain `ReadOnly=true`
- [ ] voiceCLI publish CI completes successfully; `ghcr.io/roxabi/voicecli:staging` published

### Slice 2 — M₁ provisioned

- [ ] `grep -c voicecli ~/projects/lyra/deploy/nats/auth.conf` returns ≥ 2 (voice-stt and voice-tts identities present)
- [ ] `podman secret ls --filter name=voicecli-nats-stt` and `voicecli-nats-tts` both return entries
- [ ] `podman run --rm -v voicecli-models:/mnt docker.io/library/alpine ls /mnt/huggingface/hub 2>/dev/null | wc -l` returns ≥ 1 (models pre-seeded, not empty)

### Slice 3 — Containers active + smoke green

- [ ] `systemctl --user is-active voicecli-stt` returns `active`
- [ ] `systemctl --user is-active voicecli-tts` returns `active`
- [ ] `podman exec voicecli-stt test -f /home/voicecli/.config/voicecli/nkeys/seed.seed && echo ok` returns `ok` (secret mounted at correct path)
- [ ] `podman exec voicecli-tts test -f /home/voicecli/.config/voicecli/nkeys/seed.seed && echo ok` returns `ok`
- [ ] `podman exec voicecli-stt nvidia-smi` exits 0 (GPU visible inside container via CDI)
- [ ] `make voice-smoke` exits 0 against containerized workers (`voice-smoke` = round-trip STT+TTS NATS request returning non-empty transcript)
- [ ] `nvidia-smi --query-gpu=memory.free --format=csv,noheader` ≥ 1000 MiB after both services steady-state (measured after model warm-up, not during load)
- [ ] `systemctl --user show voicecli-stt --property=Restart` returns `Restart=on-failure`
