## Context

Discovered during a production incident: `lyra_stt` / `lyra_tts` adapters were not running,
hub logged `TTS adapter unreachable: NoRespondersError` on every single message with no backoff.
Also surfaced two env var inconsistencies in the voice stack.

## Goal

Normalize voice env vars to a consistent `LYRA_` prefix + add an in-process circuit breaker
to both NATS voice clients so the hub degrades gracefully when adapters are down.

## Users

- **Mickael (operator):** no more log spam; clear env var names to enable/disable voice per machine
- **Hub runtime:** zero wasted NATS round-trips when adapters are down

## Expected Behavior

### Env var normalization

Four new vars replace the old inconsistent set:

| Var | Purpose | Default |
|-----|---------|---------|
| `LYRA_STT_ENABLED` | Enable STT (`1` = on, `0` = off) | `0` |
| `LYRA_STT_MODEL` | Whisper model size | `large-v3-turbo` |
| `LYRA_TTS_ENABLED` | Enable TTS (`1` = on, `0` = off) | `0` |
| `LYRA_TTS_ENGINE` | voicecli engine name | voicecli default |

`LYRA_STT_ENABLED=1` activates STT; `LYRA_STT_MODEL` selects the model (optional, has default).
`LYRA_TTS_ENABLED=1` activates TTS; `LYRA_TTS_ENGINE` selects the engine (optional, has default).
TTS is **independent** of STT — either can be enabled without the other.
`LYRA_VOICE_RESPONSES` and the old `STT_MODEL_SIZE` / `TTS_ENGINE` names are removed;
both old names are supported as fallbacks with a deprecation warning for one release cycle.

### Circuit breaker

Both `NatsSttClient` and `NatsTtsClient` embed a `NatsCircuitBreaker`. After
`failure_threshold` (default: 3) consecutive failures the circuit opens and all calls return
`STTUnavailableError` / `TtsUnavailableError` immediately without touching NATS. After
`recovery_timeout` seconds (default: 60s) the circuit enters HALF-OPEN: the next call is
attempted. Success → CLOSED (reset counter). Failure → OPEN again (reset timer). Any success
resets the failure counter to zero.

### Startup probe

After hub boot, `probe_voice_services()` is called once (async, non-blocking to startup).
It attempts a NATS presence check on each configured voice subject. No responder found →
`WARNING` log: `"STT adapter not reachable at boot — voice input will be disabled until adapter starts"`.
This is best-effort: failure does not prevent hub startup.

### Makefile

`LYRA_PROGRAMS` in the Makefile expands to include `lyra_stt` when `LYRA_STT_MODEL` is set
in `.env`, and `lyra_tts` when `LYRA_TTS_ENGINE` is set. `make lyra` starts only what's
configured on the machine.

## Data Model & Consumers

```classDiagram
    class NatsCircuitBreaker {
        +failure_threshold: int
        +recovery_timeout: float
        -_failures: int
        -_open_until: float
        +is_open() bool
        +record_success() None
        +record_failure() None
        +reset() None
    }

    class NatsSttClient {
        -_cb: NatsCircuitBreaker
        -_timeout: float
        -_model: str
        +transcribe(path) TranscriptionResult
    }

    class NatsTtsClient {
        -_cb: NatsCircuitBreaker
        -_timeout: float
        +synthesize(text, ...) SynthesisResult
    }

    NatsSttClient --> NatsCircuitBreaker
    NatsTtsClient --> NatsCircuitBreaker
```

    CB --> NATS[NATS subjects\nlyra.voice.stt.request\nlyra.voice.tts.request]

## Breadboard

| Affordance | Handler | Data |
|-----------|---------|------|
| `LYRA_STT_ENABLED=1` | `init_nats_stt(nc)` → `NatsSttClient(nc, model=LYRA_STT_MODEL)` | env → model string |
| `LYRA_STT_ENABLED` absent/`0` | `init_nats_stt(nc)` → `None` | — |
| `LYRA_TTS_ENABLED=1` | `init_nats_tts(nc)` → `NatsTtsClient(nc)` | engine via `LYRA_TTS_ENGINE` in request |
| `LYRA_TTS_ENABLED` absent/`0` | `init_nats_tts(nc)` → `None` | — |
| hub boot complete | `probe_voice_services(nc, stt, tts)` | NATS presence check → log |
| `NatsSttClient.transcribe()` call | `_cb.is_open()` check → skip or attempt | circuit state |
| NATS call succeeds | `_cb.record_success()` | `_failures = 0` |
| NATS call fails | `_cb.record_failure()` | `_failures++`; open if ≥ threshold |
| circuit open | raise `STTUnavailableError("circuit open")` immediately | no NATS call |
| `recovery_timeout` elapsed | HALF-OPEN: attempt one call | `_open_until` check |
| `make lyra` | read `.env`, expand `LYRA_PROGRAMS` | `lyra_stt` and/or `lyra_tts` added |

## Success Criteria

- [ ] `LYRA_STT_ENABLED=1` in `.env` → hub creates `NatsSttClient` (model from `LYRA_STT_MODEL` or default); `0`/absent → `stt=None`
- [ ] `LYRA_TTS_ENABLED=1` in `.env` → hub creates `NatsTtsClient` (engine from `LYRA_TTS_ENGINE` or default); `0`/absent → `tts=None`
- [ ] `LYRA_STT_MODEL` set without `LYRA_STT_ENABLED=1` → STT still disabled
- [ ] `LYRA_TTS_ENABLED=1` + `LYRA_STT_ENABLED=0` → TTS created, STT None (independent)
- [ ] `LYRA_STT_ENABLED=1` + `LYRA_TTS_ENABLED=0` → STT created, TTS None
- [ ] Old `STT_MODEL_SIZE` still works with a deprecation warning logged
- [ ] Old `TTS_ENGINE` still works with a deprecation warning logged
- [ ] `LYRA_VOICE_RESPONSES` removed from `voice_overlay.py`
- [ ] After 3 consecutive STT NATS failures → circuit opens → next call raises `STTUnavailableError` immediately (no NATS call made)
- [ ] After 3 consecutive TTS NATS failures → circuit opens → next call raises `TtsUnavailableError` immediately
- [ ] After `recovery_timeout=60s` → circuit half-opens → one attempt made
- [ ] Successful half-open call → circuit closes, counter resets
- [ ] Failed half-open call → circuit re-opens, timer resets
- [ ] Hub startup with unreachable STT adapter → WARNING log, hub boots successfully
- [ ] Hub startup with unreachable TTS adapter → WARNING log, hub boots successfully
- [ ] `make lyra` with `LYRA_STT_ENABLED=1` in `.env` → `lyra_stt` included in started programs
- [ ] `make lyra` with `LYRA_TTS_ENABLED=1` in `.env` → `lyra_tts` included in started programs
- [ ] `make lyra` with neither set → only `lyra_hub lyra_telegram lyra_discord` started
- [ ] Tests for `NatsCircuitBreaker`: open/close/half-open transitions
- [ ] Tests for env var normalization in `voice_overlay`
