Tab 8 — Root Causes

Two bugs, two missing wires

Both issues stem from the same architectural choice: a hard fork between streaming and non-streaming paths, with no cross-feature support.

🐛 Bug 1 — Tool recap missing in streaming=False mode
Symptom: When streaming=False, no tool recap card is ever shown. Discord and Telegram both affected.
Causal chain:
simple_agent.py — takes non-streaming branch
if model_cfg.streaming and _stream_fn is not None: # streaming=False → this block is SKIPPED entirely ... return processor.process(stream_iter) # StreamProcessor never runs # Falls through to complete() result = await self._provider.complete(pool_id, text, ...)
cli_protocol.py — tool events silently consumed, never surfaced
# read_until_result() in cli_protocol.py if msg_type == "tool_result": pass # tool events DISCARDED — never returned to caller if msg_type == "result": return CliResult(result=result_text, ...) # only final text
StreamProcessor never instantiated → ToolSummaryRenderEvent never emitted
# StreamProcessor only created here — which is skipped: processor = StreamProcessor(config=self._tool_display_config) return processor.process(stream_iter) # → No ToolSummaryRenderEvent ever → adapters never show recap
Fix: Set streaming=True in the DB so SimpleAgent always takes the streaming branch. StreamProcessor runs, ToolSummaryRenderEvent flows to send_streaming().
🐛 Bug 2 — Intermediate ⏳ messages missing in streaming=True mode
Symptom: When streaming=True, multi-turn agentic responses never emit ⏳ progress messages. User sees nothing until the final answer.
Causal chain:
cli.py — on_intermediate hardcoded to None in .stream()
# src/lyra/llm/drivers/cli.py line ~85 async def stream(self, pool_id, text, model_cfg, system_prompt) -> AsyncIterator[LlmEvent]: return await self._pool.send_streaming( pool_id, text, model_cfg, system_prompt, on_intermediate=None # ← HARDCODED. Callback from pool_processor is LOST here. )
simple_agent.py — doesn't pass on_intermediate to provider.stream()
# src/lyra/core/simple_agent.py line ~248 stream_iter = await _stream_fn( pool.pool_id, text, model_cfg, pool._system_prompt or self.config.system_prompt, # on_intermediate NOT passed — even if show_intermediate=True in agent config )
StreamingIterator — no turn-boundary parsing
# Even if callback were wired, StreamingIterator doesn't parse # "assistant" turn boundaries the way read_until_result() does: # cli_protocol.py StreamingIterator: yields TextLlmEvent per text chunk, # never tracks assistant_turn_count, never fires on_intermediate.
Fix (two steps):
  1. Thread on_intermediate through: simple_agent → ClaudeCliDriver.stream() → send_streaming() → StreamingIterator
  2. Add turn-boundary detection to StreamingIterator: track "assistant" type events in NDJSON, fire callback when turn_count >= 2 (same logic as read_until_result())
