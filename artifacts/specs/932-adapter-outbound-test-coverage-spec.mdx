## Context

Quality audit P1-15. Corrected baseline (measured 2026-04-26, all relevant test files):

| Module | Stmts | Miss | Branch | Cover | Missing lines |
|--------|-------|------|--------|-------|---------------|
| `discord_outbound.py` | 125 | 37 | 30 | 68% | 67–83, 103, 126, 132, 144–149, 169–175, 211–212, 218–234 |
| `telegram_outbound.py` | 124 | 43 | 28 | 64% | 53–67, 147, 154–158, 176–182, 217–218, 221–232, 235–244, 250 |

44 existing tests across 5 files (including `test_discord_typing.py`, `test_telegram_typing.py`).
The audit's 12% figure only counted per-file tests; typing tests were excluded from the measurement.

Remaining uncovered code clusters in 3 areas (typing workers already partially covered):

1. **Streaming callback closures** — `build_streaming_callbacks()` inner functions on both platforms (largest gap)
2. **Tool formatting** — `_build_tool_embed` (discord L142–149), `_format_tool_summary` (telegram L152–158)
3. **Send edge paths** — `_discord_typing_worker` bail-out/retry (L67–83), `_typing_worker` bail-out (L53–67), intermediate outbound, thread-context send, send-with-view

## Goal

Raise both modules to ≥80% line coverage by adding targeted tests for the 3 remaining uncovered clusters.

## Users

- **Primary:** Lyra developers — confidence when touching outbound send/streaming code
- **Secondary:** End users — fewer silent regressions in message delivery

## Expected Behavior

New tests exercise each uncovered cluster. All tests mock platform clients (discord.py / aiogram) — no live API calls. Tests follow existing patterns in `tests/adapters/conftest.py`. `asyncio.sleep` must be patched in typing worker tests to avoid real delays.

## Test Plan

### Slice 1 — Streaming callbacks

**Discord `build_streaming_callbacks` (L152–256, missing: 169–175, 211–212, 218–234):**
- `_send_placeholder` with `should_reply=False` → `channel.send(placeholder_text)`
- `_edit_placeholder_text` → `ph.edit(content=display, embed=None)`
- `_edit_placeholder_tool` → builds embed via `_build_tool_embed`, `ph.edit(content="", embed=...)`
- `_send_message` with text > `DISCORD_MAX_LENGTH` (2+ chunks) → `send_with_retry` for non-last, direct send for last, returns last id
- `_send_message` with send failure on last chunk → logs exception, returns None
- Noop path: non-discord inbound → all callbacks are noops/raise ValueError

**Telegram `build_streaming_callbacks` (L161–269, missing: 176–182, 217–218, 221–232, 235–244, 250):**
- `_send_placeholder` with `reply_to` set → sends with `reply_to_message_id`
- `_send_placeholder` without `reply_to` → sends without it
- `_edit_placeholder_text` → renders text, calls `bot.edit_message_text`
- `_edit_placeholder_text` with edit failure → logs debug, no raise
- `_edit_placeholder_tool` → formats tool summary, edits message
- `_send_message` → renders chunks, sends each, returns last message_id
- `_send_fallback` with text → renders + sends; with empty text → uses placeholder_text
- Noop path: non-telegram inbound → noop callbacks/raise ValueError

### Slice 2 — Tool formatting + send edges

**Discord:**
- `_build_tool_embed` (L142–149): complete event → green embed with "Done" title; incomplete → blue embed with "Working" title
- `send()` with `outbound.intermediate=True` (L132) → `_start_typing` called after send
- `send()` in thread context (thread_id set, L105–115) → skips reply-to, uses `messageable.send`
- `send()` with view + thread context (L125–126) → `messageable.send(chunk, view=chunk_view)`
- `send()` with `_validate_inbound` returning None (L103) → early return
- `_discord_typing_worker` retry (L47–60): first resolve fails, second succeeds — patch `asyncio.sleep`
- `_discord_typing_worker` bail-out (L67–83): 3 consecutive `channel.typing()` errors → returns — patch `asyncio.sleep`

**Telegram:**
- `_format_tool_summary` (L152–158): complete event → "Done ✅"; incomplete → "Working…"
- `send()` with `outbound.intermediate=True` (L147) → `_start_typing` called
- `send()` with `message_id=None` → no `reply_to_message_id` in kwargs
- `_typing_worker` bail-out (L53–67): 3 consecutive failures → breaks — patch `asyncio.sleep`
