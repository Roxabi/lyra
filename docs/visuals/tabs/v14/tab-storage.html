Data & Storage
Directory tree, SQLite databases, logs, voice assets, and config files.
🗺️
Two root vaults: ~/.lyra/ for credentials & auth, ~/.roxabi-vault/ for memory & sessions. Everything else is in standard system locations.
Directory tree
~/.lyra/ ├── auth.db # bot credentials + agents (AgentStore) + grants + prefs ├── turns.db # raw conversation turns (L1 audit trail) ├── pairing.db # invite codes (auto-expire) ├── keyring.key # Fernet encryption key (chmod 600) ├── agents/ # user-level agent overrides (machine-specific, gitignored) │ └── <name>.toml # higher precedence than src/lyra/agents/ └── logs/ ├── YYYYMMDD_HHMMSS_lyra.log # rolling (10 MB × 5) └── monitor.log # monitoring daemon (5 MB × 3) ~/.roxabi-vault/ # override with $ROXABI_VAULT_DIR ├── [SQLite + FTS index] # memory vault (roxabi-vault manages) └── personas/ # persona definitions (referenced by agents) ~/.claude/projects/ └── <cwd-slug>/ └── <session_id>.jsonl # Claude session history (per pool) ~/projects/lyra/supervisor/logs/ ├── lyra_telegram.log # stdout (10 MB × 3) ├── lyra_telegram_error.log # stderr (5 MB × 3) ├── lyra_discord.log └── lyra_discord_error.log ~/projects/lyra-stack/logs/ ├── supervisord.log # process manager (10 MB × 3) ├── voicecli_stt.log └── voicecli_tts.log /tmp/ or $LYRA_AUDIO_TMP └── lyra_audio_XXXXXX.ogg # temp audio (deleted after send)

Security & encryption
🗝️
keyring.key
~/.lyra/keyring.key
Algorithm
Fernet (AES-128-CBC + HMAC-SHA256)
Permissions
chmod 600 — owner read/write only
Used for
Encrypting bot tokens stored in auth.db
WarningPermanent — losing this key = losing all credentials
🛡️
What is never stored in plaintext
Data that stays ephemeral or encrypted
TTS output
Writes temporary WAV + OGG files to ~/.lyra/tmp/ during synthesis, then cleans up after bytes are read
Bot tokens
Never logged — only Fernet-encrypted at rest
Auth cache
In-memory dict (write-through to SQLite)
Webhooks
HMAC signature verified on every inbound request
