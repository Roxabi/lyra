Native commands are adapter-level
⚡
Commands are handled at the adapter level — zero LLM tokens consumed. When a message starts with /, the CommandRouter dispatches it to the matching handler before the agent sees it. /help shows both Commands (built-in) and Extensions (loaded via CommandLoader) sections.
Message flow
flow
1. Adapter receives message → normalizes to InboundMessage(platform, bot_id, scope_id, text, …)
2. Enqueued on per-platform asyncio Queue (bounded=100, immediate ACK when full)
3. InboundBus staging queue → Hub.run_loop()
4. Hub: resolve_binding(platform, bot_id, scope_id) → agent="lyra_default"
5. Pool created/reused for this scope — one Pool = one isolated conversation
6a. text starts with "/" → CommandRouter.dispatch() → handler → response  (zero tokens!)
6b. normal text → agent.process(msg, pool) → LLM backend (claude subprocess or SDK)
7. OutboundDispatcher: verify routing → deliver reply to originating channel only
Scope isolation
RoutingKey format
# (platform, bot_id, scope_id) → (agent, pool_id)
telegram · lyra · chat:555   → pool telegram:lyra:chat:555
discord  · lyra · thread:888 → pool discord:lyra:thread:888
Two different bots talking to the same user in the same channel have completely isolated conversation history.

Token streaming
⟳
New in v12 (#321). When streaming = 1 on an agent row, responses stream token-by-token from the LLM backend to the adapter. Telegram: message is edited in-place as tokens arrive. Discord: first message chunk sent, then edited. Enable per-agent via lyra agent edit <name> → set streaming.
Runtime troubleshooting
Bot not responding
Check owner_users in config.toml — if your user ID is missing, the bot silently ignores messages. Verify with lyra config show.
Agent not found on startup
Run lyra agent init to seed the DB from TOML files. Check lyra agent list to confirm agents are loaded.
Memory / vault not working
Ensure ROXABI_VAULT_DIR is set in .env or the default ~/.roxabi-vault/ exists. Check lyra agent show <name> for memory_namespace.
Streaming shows garbled output
Verify streaming = 1 in the agent DB row. Discord and Telegram handle streaming differently — check adapter logs with make lyra logs.
