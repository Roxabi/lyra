## Problem

10 `# type: ignore[arg-type]` sites remain in test files where adapter kwargs are splatted through a `**kwargs` dict into constructors like `TelegramAdapter(**kwargs)`. Mechanical `cast()` substitution fails because pyright leaks type errors through the splat to every constructor arg, causing cascading errors across 8+ files.

These are the structurally-blocked tail of #828's quick-win cleanup, which removed 50/60 inline ignores (83%). The remaining 10 require a structural refactor rather than mechanical fix.

## Who

- **Primary:** Developers maintaining the codebase — type check failures slow development, reduce confidence in type safety
- **Secondary:** CI pipeline — pyright failures block merges

## Constraints

- Must not change test behavior (refactor only)
- Must work with pyright's type narrowing
- Post-refactor total `# justified:` retentions must remain ≤ 6
- Pattern is mechanical but touches 5+ test files

## Out of Scope

- The 6 justified retentions from #828 (V4 PLC0415 + V2 ARG002)
- Any changes to non-test files
- Changing the underlying adapter constructors

## Complexity

**Tier: F-lite** — Clear scope (10 sites, ~5 files), single domain (tests), no architecture changes. Mechanical refactor with judgement required on shared-helper API design.

**Signals:**
- 10 specific sites, well-identified
- ~5 test files affected
- Single domain: test type hints
- Three clear implementation options (TypedDict, Factory, Direct)
- No new patterns or dependencies
