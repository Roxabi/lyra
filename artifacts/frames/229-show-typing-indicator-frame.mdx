## Problem

Lyra currently shows no typing indicator on Telegram or Discord while the LLM is computing a response. Users see silence from the moment they send a message until the first reply token or placeholder appears — a noticeably poor UX. The same problem was already solved in 2ndBrain, which has a working reference implementation.

## Who

- **Primary:** Lyra users on Telegram and Discord — they experience the unresponsive gap between sending a message and receiving a reply.
- **Secondary:** None.

## Constraints

- **Telegram:** The typing indicator expires after ~5 seconds, so it must be refreshed on a 3-second loop via `bot.send_chat_action(chat_id, "typing")`. Must be implemented as an async context manager wrapping `send()` and `send_streaming()`.
- **Discord:** `discord.py` provides a built-in `async with messageable.typing():` context manager that handles refresh internally — no custom loop needed.
- **Reference:** `~/projects/2ndBrain/telegram_bot/scripts/telegram_utils.py` lines 390–431 contains a battle-tested pattern for the Telegram loop.
- **Scope:** Touch only `src/lyra/adapters/telegram.py` and `src/lyra/adapters/discord.py`.

## Out of Scope

- Other adapters (e.g., future ones).
- Changes to the Hub, MessagePipeline, or OutboundDispatcher.
- Configurable indicator type or interval (hardcode 3.0s default).

## Complexity

**Tier: F-lite** — Clear scope, two files, single domain (adapters), reference implementation already exists. No architectural change.
