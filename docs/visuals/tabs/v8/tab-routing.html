Routing flow

1
Platform adapter receives update
Telegram: long-poll or webhook. Discord: gateway websocket. Each adapter normalizes the platform-specific event into a common InboundMessage(platform, scope_id, bot_id, text, user_id, …).
2
Scope ID resolution
scope_id is the unique conversation identity. For Telegram: <platform>:<chat_id>. For DMs: <platform>:dm:<user_id>. This is the key used to look up (or create) a Pool.
3
Bot → Agent lookup
Hub queries the bots table for the bot_id from the message, retrieves the assigned agent_name. If no assignment found, the message is dropped with a warning.
4
Pool lookup or creation
Hub looks up pools[scope_id]. On first message from this scope, a new Pool is created with the resolved agent. Subsequent messages reuse the same Pool (same history, same circuit state).
5
Command router check
If message starts with /, it's routed to the command router first. Session commands (/new, /add, etc.) are handled without hitting the LLM. Workspace commands set cwd and fall through to the agent.
6
Circuit breaker check
Before calling the agent, Pool checks the circuit state. If the circuit is OPEN (too many recent failures), the message is rejected with a user-facing error. Circuit auto-probes after a cooldown period.
7
Agent execution + dispatch
Agent generates response, tool calls are executed inline, final response is pushed to OutboundDispatcher queue for the bot. Dispatcher delivers with retry logic.

Circuit breaker states

CLOSED

Normal operation. All messages processed. Failure counter increments on errors.

OPEN

Too many failures. Messages rejected immediately with error. After cooldown, transitions to HALF-OPEN.

HALF-OPEN

Probe mode. One message allowed through. Success → CLOSED. Failure → back to OPEN.

Debug routing issues

bash
# See what bot is assigned to which agent
sqlite3 ~/.lyra/auth.db "SELECT id, platform, agent_name FROM bots"

# Check active pools (from logs)
make lyra logs | grep -i pool

# Check circuit state
make lyra logs | grep -i circuit

# Increase log verbosity temporarily
LYRA_LOG_LEVEL=DEBUG make lyra reload
