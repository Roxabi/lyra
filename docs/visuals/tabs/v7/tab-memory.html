L1 — TurnStore
turns.db — conversation audit trail
Every user + assistant turn persisted to ~/.lyra/turns.db
Columns: pool_id, session_id, role, platform, user_id, content, message_id, reply_message_id, timestamp, metadata
Separate DB from auth.db — avoids write contention with memory vault
Indices on (session_id, timestamp) and (pool_id, timestamp) for efficient query
📝
L1 is partially built. TurnStore provides the raw audit trail. Full session state management (multi-turn state per pool with configurable TTL) is coming in a future update.
L0 — Working memory & compaction
flow — compact() trigger
sdk_history grows with each turn
    ↓ reaches 80% of 200k token threshold (COMPACT_THRESHOLD)
SessionManager.compact() fires:
  1. _summarize_session()  → LLM generates summary of conversation so far
  2. MemoryManager.upsert_session(..., status="partial")  → write partial snapshot to L3
  3. Replace sdk_history with: [summary_turn] + last 10 turns  (in-place)
    ↓
Conversation continues with compact history — seamless to user

Pool eviction / disconnect:
SessionManager.flush_session()  → full L3 write (status="final") + background extraction
  ├── concept extraction  (people, dates, places, code topics)
  └── preference extraction  (user prefs, style)

Memory types & freshness TTL
FRESHNESS_TTL_DAYS (from memory.py)
Entries older than their TTL are marked stale in recall results (prefixed with age). None = permanent (never stale).
TypeTTLDescription
concept:technology180 daysTech stack, frameworks, languages

concept:decisionpermanentArchitecture / design decisions
concept:fact60 daysFacts, dates, events

MemoryManager API
python
# Exposed by MemoryManager (wraps AsyncMemoryDB from roxabi-vault)

# Lifecycle
connect()              # open DB + apply schema compat migrations
close()                # close DB connection

# Identity
get_identity_anchor(namespace)          # → str | None
save_identity_anchor(namespace, text)   # upsert identity anchor in L3

# Session + contact
upsert_session(snap, summary, status="final")
  → flush session summary to L3 (status: "partial" during compact, "final" on eviction)
upsert_contact(user_id, medium, namespace)  # track user last-seen

# Recall
recall(user_id, ns, first_msg, token_budget)
  → freshness-aware cross-session context ([MEMORY] + [PREFERENCES] blocks)

# Extraction (called by SessionManager background tasks)
upsert_concept(snap, data)     # background extraction: concepts with category
upsert_preference(snap, data)  # background extraction: user preferences

# Note: compact() lives on SessionManager, not MemoryManager
