STT — Speech to text (optional)
✅
STT is optional. Omit STT_MODEL_SIZE from .env and STT is disabled — Lyra starts normally. Set it to enable. The .env.example ships small for low-VRAM setups; use large-v3-turbo for best accuracy.
bash.env
# Omit entirely to disable STT
STT_MODEL_SIZE=small        # use large-v3-turbo for best accuracy
STT_DEVICE=auto             # cpu | cuda | auto
STT_COMPUTE_TYPE=auto       # float16 | int8 | auto
📚
Personal vocab: Drop voicecli.vocab at ~/.voicecli/voicecli.vocab (one word/phrase per line). Loaded automatically — shared with the voicecli dictate daemon.

End-to-end voice pipeline
🎙
Full voice loop: Send a voice message → Whisper transcribes → modality="voice" set → LLM replies in natural spoken language → auto-TTS synthesizes → audio sent back. Works on Telegram (voice messages) and Discord (TTS stream to voice channel).
chatTelegram voice message
You:  🎤 [voice message: "What time is it?"]
  ↓ Whisper transcribes → modality="voice"
  ↓ LLM replies in spoken language
  ↓ voicecli synthesizes TTS audio
Bot:  🔊 "It's 3pm on Saturday..."

# /voice command — text input, voice output
You:  /voice What's the weather like today?
  ↓ hint injected: [Voice — reply in natural spoken language]
Bot:  🔊 "The weather today is sunny and warm..."

STT transcription echo & noise filtering
Transcription feedback
When STT is enabled and a user sends a voice message, Lyra echoes the transcription back before processing it through the agent. This gives immediate feedback that the voice was understood correctly.
chat
You:  🎤 [voice message]
Bot:  📝 "What time is the meeting?"   ← STT echo
  ↓ agent processes transcribed text
Bot:  🔊 "The meeting is at 3pm..."      ← TTS reply
Whisper noise tokens (auto-discarded)
Transcriptions matching these tokens (or empty strings) are treated as noise. Lyra replies with stt_noise and does not forward to the agent.
pythonstt/__init__.py
WHISPER_NOISE_TOKENS = {
  "[music]", "[applause]",
  "[laughter]", "[silence]",
  "[noise]"
}
Discord voice commands coming soon
Join / leave a voice channel
discordguild messages only (no DMs)
# Join voice channel the user is in (transient — auto-leave when done)
!join              → bot joins, leaves when TTS queue drains
/join              → alias for !join

# Persistent mode — stays in channel (requires Trusted)
!join stay         → PERSISTENT — stays until !leave (falls back to transient if not trusted)

# Leave voice channel (requires Trusted)
!leave             → "Left the voice channel." or "I'm not in a voice channel."

# Error replies
"Already in a voice channel."   — !join when already joined
"Join a voice channel first."   — !join when user not in any channel
"Voice is not available."       — VoiceDependencyError (voicecli missing)

Wired into on_message() before the mention/DM filter — detected even without a bot mention.
