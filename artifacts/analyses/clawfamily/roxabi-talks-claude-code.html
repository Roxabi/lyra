À surveiller

Ce qu'il faut savoir

⚠
Context window limitée
200K tokens, mais un grand repo les épuise vite. Stratégie : /compress, sessions courtes, fichiers ciblés.
⚠
Coût API — surveiller les tokens
Sonnet 3.7 : ~$3/M input, $15/M output. Une session intensive = $2-5. Budget à prévoir. Tip : cache prompt avec CLAUDE.md.
⚠
Sur-engineering
Claude code facilement trop complexe si tu ne cadres pas. Toujours préciser les contraintes (simplicité, perf, maintenabilité).
🔒
Sécurité — permissions
Ne jamais activer --dangerously-skip-permissions sur un repo de prod. Restreindre les allowed tools dans settings.json.

Honnêteté

Les limites réelles

01
Pas de mémoire native longue
Chaque session repart à zéro sauf CLAUDE.md et MEMORY.md. La continuité, tu la construis toi-même.
02
Pas de vision temps réel
Pas d'accès live aux logs, metrics, ou état runtime. Il analyse du code statique, pas un système vivant.
03
Latence sur gros repos
L'indexation et la navigation dans 100K+ lignes ralentissent. Monorepo ? Penser à des sessions ciblées par module.
