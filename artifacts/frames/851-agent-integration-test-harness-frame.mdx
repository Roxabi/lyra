## Problem

SimpleAgent has 0% test coverage — a P0 finding from the 2026-04-22 code quality audit. The root cause is not laziness but friction: testing an agent requires stitching together Hub + Pool + LlmProvider + CLI subprocess + STT/TTS mocks. This high-ceremony setup deters test writing. Raising coverage one agent at a time doesn't fix the underlying friction.

The audit identified 4 other P0 long-term fixes, but they ride on in-flight epics. This one is greenfield — no natural parent. High compounding value: a test harness unblocks future agent classes, lyra_cli (#628), and lyra_harness (#633).

## Who

- **Primary:** Lyra developers adding/modifying agents — need fast, reliable test feedback
- **Secondary:** Future consumers of lyra_harness tooling (external integrators)

## Constraints

- Must build on existing test seams: FakeClaudeCliDriver, FakeStt/FakeTts, json_agent_store, PoolContext protocol
- Cannot require real LLM API calls (determinism + cost)
- Must support both text and audio (STT/TTS) paths
- Hot-reload and session resume must be testable
- Harness API should feel ergonomic (async context manager pattern)

## Out of Scope

- Real STT/TTS service integration (unit test territory)
- Load/performance testing
- Multi-agent orchestration tests (future harness extension)
- Mocking the Hub's NATS transport (ADR-045) — tests use in-process Hub

## Complexity

**Tier: F-lite** — Clear scope, single domain (testing infrastructure), no architectural unknowns. Reuses existing test seams; primarily assembly and API design work.
