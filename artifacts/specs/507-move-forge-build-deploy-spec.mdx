## Context

Forge build/deploy tooling lives in `lyra-stack/forge/` but logically belongs with the forge skill in `roxabi-plugins`. The frame (approved 2026-04-03) scoped this as a file relocation with Makefile and credential wiring ‚Äî no logic changes.

## Goal

Make forge build and deploy self-contained within `roxabi-plugins`, removing the dependency on `lyra-stack` for build/deploy operations.

## Users

- **Primary:** Mickael ‚Äî runs `make forge build`, `make forge deploy` from CLI
- **Secondary:** Future forge consumers who don't have lyra-stack checked out

## Expected Behavior

After this change:
1. `cd ~/projects/roxabi-plugins && make forge-build` assembles `_dist/` from `~/.roxabi/forge/`
2. `cd ~/projects/roxabi-plugins && make forge-deploy` builds + deploys to Cloudflare Pages
3. `cd ~/projects/roxabi-plugins && make forge-deploy-prod` rsyncs to production
4. `cd ~/projects/lyra-stack && make forge build` still works but delegates to roxabi-plugins scripts (or is removed with a clear error pointing to the new location)
5. Cloudflare credentials are loaded from `roxabi-plugins/.env`
6. Google Drive sync (`push/pull/sync`) stays in lyra-stack (rclone is repo-level)
7. Supervisor `forge` program (serve.py live-reload) stays in lyra-stack ‚Äî it's a daemon, not a build tool

- [ ] `gen-image-manifests.py` exists at `roxabi-plugins/forge/scripts/gen-image-manifests.py` and runs successfully
