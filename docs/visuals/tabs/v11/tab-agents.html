Agent Configuration ADR-029
DB-first config. Agents live in ~/.lyra/auth.db (SQLite). TOML files (src/lyra/agents/, ~/.lyra/agents/) are seed sources only — import them with lyra agent init. Startup reads only the DB.
Hot-reload via polling. The runtime checks updated_at on each agent row. Editing via lyra agent edit bumps this timestamp — no file mtime needed, no restart required for persona/voice changes.
TOML Search Locations

Two directories are scanned, in precedence order:

User-level (higher precedence)
~/.lyra/agents/

Machine-specific overrides. Gitignored. Takes priority over bundled defaults.

System defaults
src/lyra/agents/

Bundled in the repo. Committed. Lowest precedence — only loaded when no user override exists.

TOML edits require re-import. After editing a TOML file, run lyra agent init --force and restart to reflect changes. Use lyra agent edit for live DB edits that take effect without restart.
Agent CLI Reference
CommandDescription
lyra agent initSeed DB from TOML files (first-time or after TOML edits). Safe — skips existing rows.
lyra agent init --forceOverwrite existing DB rows from TOML. Use after TOML changes.
lyra agent create <name>Scaffold a new agent TOML at the appropriate path.
lyra agent listList all agents in the DB with backend and model info.
lyra agent show <name>Full config for one agent including all JSON fields.
lyra agent edit <name>Edit agent interactively in the DB. Bumps updated_at → hot-reload. No TOML needed.
lyra agent validate <name>Validate agent config against schema without starting.
lyra agent assign <name> --platform <p> --bot <id>Assign agent to a bot. Creates row in bot_agent_map.
lyra agent unassign --platform <p> --bot <id>Remove assignment. Bot reverts to default agent.
lyra agent delete <name>Delete agent from DB. Refuses if a bot is still assigned.
Key Config Notes
cwd
Machine-specific

Always set in config.toml [defaults], never in the agent TOML. Varies per machine.

workspaces
/workspace <key>

Each key in workspaces_json becomes accessible via /workspace <key>, overriding cwd for the current pool.

Persona changes. In V11, persona lives in persona_json (DB column). Use lyra agent edit to change it. TOML-based persona is legacy — it's auto-migrated to persona_json on AgentStore.connect().
