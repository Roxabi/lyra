## Problem

Per ADR-047 and `lyra-nats-truth §14`, the machine-level supervisord at `~/projects/` is the single host for all Roxabi tenants (lyra, voiceCLI, imageCLI, forge, roxabi-intel, idna, …). Lyra must not ship its own supervisor host.

Prod (roxabituwer, M₁) completed the migration on 2026-04-22: `~/.config/systemd/user/lyra.service` now targets `~/projects/supervisord.conf` + `~/projects/scripts/start.sh --all`. The legacy instance under `deploy/supervisor/` is stopped, but the repo still ships its files — the repo state is the only thing out of sync.

Leftover host files (`supervisord.conf`, `start.sh`, `supervisorctl.sh`, `scripts/`) and deprecated tenant confs (`lyra_tts.conf`, `lyra_stt.conf` — now owned by voiceCLI satellites) create ownership confusion and block future multi-host deploys where lyra isn't co-located with voiceCLI (§14.4 open question).

## Who

- **Primary:** Mickael / ops — needs the repo to match the production layering (tenant, not host).
- **Secondary:** Future contributors / AI agents reading `deploy/` — must not see lyra as a supervisor host.

## Constraints

- Prod already migrated; no runtime behavior change expected from this PR.
- Tenant confs (`lyra_hub.conf`, `lyra_telegram.conf`, `lyra_discord.conf`) must remain in the repo as reference — they are copied into `~/projects/conf.d/` on each host.
- `lyra_tts.conf` + `lyra_stt.conf` are deprecated — superseded by voiceCLI satellites (subjects `lyra.voice.{tts,stt}.request`, queue groups `{tts,stt}-workers`).
- Any doc referencing `deploy/supervisor/supervisord.conf` or `deploy/supervisor/start.sh` must be updated to point at the machine-level `~/projects/supervisord.conf` + `~/projects/scripts/start.sh --all`.
- Systemd example in docs must target `~/projects/scripts/start.sh --all`, not `deploy/supervisor/start.sh`.

## Out of Scope

- Changes to the machine-level supervisord at `~/projects/` (lives outside this repo).
- Changes to voiceCLI satellites or the `lyra.voice.{tts,stt}` NATS contract.
- Rewriting ADR-047 or `lyra-nats-truth` — this PR implements them.
- Migration tooling — prod is already migrated.

## Complexity

**Tier: F-lite** — single devops domain, clear acceptance criteria, ~10 file-level changes (delete + move + README rewrite + repo-wide reference scrub).

Signals:

- No new architecture — implements existing ADR-047 decision.
- Clear acceptance list already enumerated in the issue.
- Bounded blast radius: only `deploy/` + docs that reference the old paths.
