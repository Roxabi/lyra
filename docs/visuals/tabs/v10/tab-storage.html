File locations

~/.lyra/auth.db
Primary SQLite database. Contains agents, bot mappings, sessions, runtime state. Single source of truth.
~/projects/lyra/config.toml
Machine-specific config. Bot tokens, API keys, cwd. Gitignored.
~/projects/lyra/config.toml.example
Template committed to git. No secrets.
src/lyra/agents/*.toml
Bundled seed files for agent defaults. Read only by lyra agent init.
~/.lyra/agents/*.toml
User-level seed overrides. Higher precedence than bundled seeds.
~/.roxabi-vault/personas/
Persona TOML files. Loaded at migration time; runtime uses persona_json in DB.
~/.lyra/memory/
Persistent memory files. Written by the agent, read at session start.
src/lyra/commands/
Extension command packages (formerly src/lyra/plugins/). Loaded by CommandLoader.
~/projects/lyra-stack/conf.d/
Symlinks to supervisor program configs. Managed by make register.
~/projects/lyra/local/machines.md
Machine IPs, disk layouts, SSH commands. Gitignored, not versioned.

auth.db — agents table

The agents table is the single runtime source of truth. AgentStore maintains a write-through in-memory cache keyed by name.

name backend model max_turns tools_json show_intermediate smart_routing_json plugins_json memory_namespace cwd tts_json stt_json skip_permissions permissions_json workspaces_json i18n_language commands_json streaming persona_json ★ voice_json ★ fallback_language ★ patterns_json ★ source created_at updated_at

★ Added in v10 (ADR-029). COALESCE on upsert: pass null to preserve existing value.

bashQuick DB inspection
sqlite3 ~/.lyra/auth.db ".tables"
sqlite3 ~/.lyra/auth.db "SELECT name, model, updated_at FROM agents"
sqlite3 ~/.lyra/auth.db "SELECT name, persona_json IS NOT NULL, voice_json IS NOT NULL FROM agents"

auth.db — bot_agent_map table v10 updated

Maps (platform, bot_id) → agent_name. Loaded into AgentStore's _bot_map and _bot_settings caches on startup.

ColumnTypeDescription
platformTEXTPlatform identifier: telegram | discord
bot_idTEXTPlatform-specific bot ID
agent_nameTEXTName of the assigned agent
settings_jsonTEXTPer-bot settings: watch_channels, auto_thread, etc. COALESCE on upsert — null preserves existing. Updated via set_bot_settings().
updated_atTEXTISO-8601 timestamp of last update

AgentStore API

MethodSync/AsyncDescription
get(name)syncO(1) cache lookup. Returns AgentRow or None.
get_all()syncAll cached AgentRows.
get_bot_agent(platform, bot_id)syncagent_name for bot mapping, or None.
get_all_bot_mappings()syncAll (platform, bot_id) → agent_name mappings.
get_bot_settings(platform, bot_id)syncParsed settings dict for bot (watch_channels, etc.), or {}.
upsert(row)asyncWrite to DB and update cache. Bumps updated_at. COALESCE for persona_json/voice_json/patterns_json.
delete(name)asyncRefuses if bot still assigned (ValueError).
set_bot_agent(platform, bot_id, agent_name, *, settings)asyncUpsert bot mapping. settings=None preserves existing settings_json.
set_bot_settings(platform, bot_id, settings)asyncUpdate settings_json only. Raises ValueError if no existing row.

Backup strategy

auth.db is the single source of truth for all agent configs. Back it up before running lyra agent init --force or destructive DB operations.
bash
# Quick backup before risky operations
cp ~/.lyra/auth.db ~/.lyra/auth.db.bak.$(date +%Y%m%d_%H%M%S)

# SQLite online backup
sqlite3 ~/.lyra/auth.db ".backup ~/.lyra/auth.db.bak"
