## Context

Source: [frame](../frames/886-retire-deploy-supervisor-frame.mdx). Prod migrated 2026-04-22 to the machine-level supervisord at `~/projects/`. The repo still ships host scaffolding under `deploy/supervisor/`, creating ownership confusion. ADR-047 + `lyra-nats-truth §14` require lyra to be a tenant only.

Current on-disk state (hyphen rename from commit 7c05850 already applied):

```
deploy/supervisor/
├── .gitignore
├── supervisord.conf          # host file — delete
├── start.sh                  # host file — delete
├── supervisorctl.sh          # host file — delete
├── scripts/                  # host helpers — delete (5 files)
│   ├── run.sh
│   ├── run_hub.sh
│   ├── run_adapter.sh
│   ├── start.sh
│   └── stop.sh
└── conf.d/                   # tenant confs — keep, relocate
    ├── README.md
    ├── lyra-hub.conf
    ├── lyra-telegram.conf
    └── lyra-discord.conf
```

Note: deprecated `lyra_tts.conf` / `lyra_stt.conf` named in the issue body are already absent — voiceCLI owns tts/stt satellites now.

## Goal

Remove host scaffolding from the repo; keep lyra's three tenant confs as reference at `deploy/conf.d/`, with a README that points operators at the machine-level supervisord.

## Users

- **Primary:** Mickael / ops deploying lyra onto a Roxabi host.
- **Secondary:** Contributors and AI agents reading `deploy/` to understand lyra's runtime.

## Expected Behavior

1. `deploy/supervisor/` no longer exists.
2. `deploy/conf.d/` contains the three tenant confs (`lyra-hub.conf`, `lyra-telegram.conf`, `lyra-discord.conf`) plus a `README.md` that:
   - Names lyra as a tenant (¬host).
   - Points at `~/projects/supervisord.conf` as the host.
   - Gives the exact copy-in step (`cp deploy/conf.d/*.conf ~/projects/conf.d/`).
   - References ADR-041 / ADR-047 / `lyra-nats-truth §14`.
3. Docs (`CONFIGURATION.md`, `DEPLOYMENT.md`, `GETTING-STARTED.md`, `PROD-MIGRATION-STRATEGY.md`, ADR-041, ADR-039, ADR-043, ADR-050, ADR-053, `voice-to-voice-analysis.md`) no longer reference `deploy/supervisor/supervisord.conf`, `deploy/supervisor/start.sh`, or `deploy/supervisor/scripts/`. Where a systemd example was shown, it now targets `~/projects/scripts/start.sh --all`.
4. `Makefile`, `deploy/gen-supervisor-conf.py`, `deploy/provision.sh`, `scripts/deploy.sh`, `deploy/lyra.service`, `deploy/agents.yml`, `deploy/nats/gen-nkeys.sh`, `tests/deploy/test_supervisor_scripts.sh`, `src/lyra/integrations/supervisor.py`, `src/lyra/monitoring/checks.py`, `.github/workflows/ci.yml` — each reference to `deploy/supervisor/` is updated, neutralised, or deleted without breaking the surrounding behavior.
5. `grep -rn "deploy/supervisor" --exclude-dir=.git --exclude-dir=artifacts` returns no results outside historical artifacts.
