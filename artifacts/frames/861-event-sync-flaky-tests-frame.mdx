## Problem

CI tests use `asyncio.sleep(N)` for coordination, creating two failure modes:
1. **Arbitrary delays** — sleep shorter than actual startup on slow runners → flaky failures
2. **Infinite blocking** — `sleep(9999)` as "wait forever until something interrupts" → code smell, unclear intent

Observable impact: CI runtime penalty, intermittent failures erode trust in test suite, slow dev velocity.

## Who

- **Primary:** Developers running CI — need reliable, fast test feedback
- **Secondary:** Maintainers — waste time investigating flaky failures that aren't real bugs

## Constraints

- **Effort:** ~4h (audit all `asyncio.sleep(N)` where N≥5, replace case-by-case)
- **Scope:** Test code only — no production code changes
- **Pattern:** Use `asyncio.Event`, `asyncio.wait_for(cond, timeout)`, fixture-provided ready signals

## Out of Scope

- Production code changes
- Refactoring unrelated test patterns
- Adding new test coverage (only fixing existing flaky tests)

## Complexity
