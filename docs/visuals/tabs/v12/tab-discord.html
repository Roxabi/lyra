Watch Channels #347
★
New in v12. Discord channels listed in watch_channels (inside settings_json on bot_agent_map) bypass the mention filter entirely. The bot processes all messages in those channels automatically — no @mention required.
Normal channel
Mention required

Bot only responds when directly @mentioned. All other messages are silently ignored. Standard Discord bot behavior.

Watch channel
Always active

Every message triggers the agent. No mention needed. Threads auto-created per-conversation. Links auto-fetched and summarized.

Configuration

Watch channels are stored in settings_json on the bot_agent_map row for a specific bot.

jsonbot_agent_map.settings_json
{
  "watch_channels": [
    "1234567890",     // Discord channel ID (string)
    "9876543210"
  ]
}
ℹ️
Note: Only watch_channels belongs in settings_json. The auto_thread and process_links behaviors are configured in config.toml [[discord.bots]], not in settings_json.
bashAdding a watch channel via CLI
# Edit settings_json interactively for a specific bot's agent mapping
lyra agent edit <agent_name>

# Or update the bot_agent_map row directly via sqlite3
sqlite3 ~/.lyra/auth.db \
  "UPDATE bot_agent_map SET settings_json='{\"watch_channels\":[\"CHANNEL_ID\"]}' WHERE bot_id='mybot'"
⚠️
Direct writes bypass the cache. The AgentStore uses a write-through in-memory cache. After direct sqlite3 edits, restart the daemon (make lyra reload) to refresh the cache.

Thread Behavior
Auto-create
First message in a watch channel creates a Discord thread automatically. Subsequent messages from the same user in the same channel resume the existing thread rather than flooding the main channel.
Thread naming
Thread title is derived from the first message content (truncated to Discord limits). Bot replies inside the thread to maintain context isolation from the main channel feed.
Discord-Specific Behaviors
ℹ
Native Discord slash commands are registered separately from Lyra bot commands. Use /help inside Discord to see the full list of available commands for the assigned agent.
