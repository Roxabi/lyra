DB Schema
SQLite tables backing the agent runtime. Write-through cache, WAL mode, idempotent migrations.
💾
Write-through in-memory cache (dict[name, AgentRow]) backed by SQLite in WAL mode. AgentStore.connect() runs all migrations at startup — additive only, safe to re-run. Hot-reload polls updated_at on each agent row.

Migration strategy
🛡️
Additive only. All schema changes use ALTER TABLE ADD COLUMN. Each statement is wrapped in a try/except OperationalError — already-existing columns are silently skipped. Migrations are idempotent and run on every startup. No destructive changes ever.
pythonAgentStore.connect()
# Each migration runs at startup — safe to re-run, OperationalError = already exists
for stmt in _MIGRATE_AGENTS:
    try:
        conn.execute(stmt)
    except OperationalError:
        pass  # column already exists → skip silently

# After schema migration: populate new #343 columns from legacy values
_populate_343(conn)  # persona → persona_json, tts_json+stt_json → voice_json
Safe to re-run
Every migration is idempotent. Running lyra start on an already-migrated DB is always safe.
WAL mode
Write-Ahead Logging allows concurrent reads while writes are in progress. Better throughput for multi-pool agents.
Write-through cache
In-memory dict[name, AgentRow] is the read path. Writes go to SQLite first, then update the cache. Hot-reload syncs on updated_at change.
