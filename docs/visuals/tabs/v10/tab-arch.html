Component reference

InboundBus

One async queue per platform. Adapters push InboundMessage objects here. Hub consumes. Provides backpressure and decouples adapter I/O from agent execution.

src/lyra/core/inbound_bus.py

Hub

Central router. Receives all messages from InboundBus, resolves scope_id, looks up or creates a Pool, dispatches the message.

src/lyra/core/hub.py

Pool

A running conversation context: holds message history, circuit breaker state, and a reference to the current AgentBase instance. One Pool per scope_id. Created lazily.

src/lyra/core/pool.py

AgentBase

Stateless singleton. Holds Agent config dataclass. On every message: hot-reload check via _maybe_reload_config() + _maybe_reload_plugins(), then calls LLM API with full history. Supports token-level streaming.

src/lyra/core/agent.py

AgentStore

Write-through in-memory cache of all AgentRow DB records. get(name) is O(1). upsert() writes to both DB and cache atomically, bumping updated_at. Caches both agent rows and bot settings.

src/lyra/core/agent_store.py

CommandLoader

Loads extension commands from src/lyra/commands/ (formerly PluginLoader / src/lyra/plugins/). Managed by CommandReloadManager — hot-reloads separately from agent config via _maybe_reload_plugins().

src/lyra/core/command_loader.py

OutboundDispatcher

Per-bot async queues with exponential-backoff retry. Circuit breaker opens after sustained failures, closes on probe success. Notifies user when circuit opens.

src/lyra/core/hub_outbound.py

persona.py

Two composition paths: compose_system_prompt(PersonaConfig) for file-based personas, compose_system_prompt_from_json(dict) for inline DB persona_json. Both produce identical prose system prompt output.

src/lyra/core/persona.py

Key design constraints

AgentBase is a stateless singleton. All per-conversation state lives in Pool. One agent config serves all concurrent conversations without locking.
Hot-reload is lazy, not proactive. Config and command reloads are triggered by inbound messages, not a background timer. Split into two phases: config and plugins.
COALESCE upsert semantics. persona_json, voice_json, and patterns_json use COALESCE — passing null preserves the existing DB value. Explicit dict is required to overwrite.
Pool → Hub back-reference. Some Pool code holds a reference back to Hub, creating a circular dependency. Tracked as a high-priority audit finding.
Unbounded Pool growth. Pools are created per scope_id with no eviction policy. Long-running instances accumulate idle pools. A TTL-based cleanup is a pending improvement.
Audio loop blocking I/O. audio_loop has blocking I/O in async context — latency spikes under concurrent voice load. Fix tracked as high-priority.
