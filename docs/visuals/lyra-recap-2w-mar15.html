03Decision Log

AgentStore in SQLite (ADR-024)
TOML files become seed sources only. Runtime reads from SQLite DB. In-memory cache warmed at connect() — no per-message file I/O. CLI overhaul: init, list, show, edit, validate, assign, delete.
Why: file I/O per message was slow, agent config needed to be mutable at runtime (edit without TOML + restart), and the auth.db was already SQLite-based — natural colocation.
Idle-timeout replaces global deadline in Pool
Per-read idle timeouts with 3x retry + process liveness detection instead of a single deadline. CliPool handles timeouts directly; outer Pool timeout disabled (None).
Why: the global deadline timed out during long but active Claude runs (tool use chains). The new approach resets the timer on each output chunk, only flagging truly idle processes.
Event bus: singleton pattern (ADR-021)
EventBus as a module-level singleton rather than dependency injection. Hub, adapters, and agents all import and use the same instance.
Why: simpler wiring for a solo-developer project. DI would add boilerplate with no testing benefit since the event bus is thin and stateless.
TurnStore in separate DB (turns.db)
Raw turn logging writes to ~/.lyra/turns.db, separate from roxabi-vault's semantic memory DB.
Why: avoid write contention. Turn logging is fire-and-forget via asyncio.create_task — mixing with vault queries would risk WAL checkpointing delays.
Split adapters into independent supervisor processes
Telegram and Discord run as separate supervisord programs. Independent restart, logging, crash isolation. Managed via make telegram / make discord.
Why: a crash in discord.py voice code was taking down Telegram. Independent processes mean one channel failure doesn't affect others.
Per-agent TTS/STT config via JSON columns
Added tts_json and stt_json columns to the agents table. Serialized from TOML [tts]/[stt] sections, deserialized into typed config objects.
Why: different agents need different voice configs (Lyra uses one voice, Aryl another). Storing in the same row as the agent avoids a join and keeps the data model flat.

05Mental Model Essentials

Agent = stateless singleton
An agent is immutable config (prompt, permissions, namespace). All mutable state lives in the Pool. No race conditions because agents never write to self.*. One agent instance serves many pools.
TOML → DB → Runtime
Agent TOML files are seed sources only. Must run lyra agent init to import into SQLite DB. Runtime reads from DB cache. TOML edits require init --force + restart.
RoutingKey = (platform, bot_id, scope_id)
Not user_id. scope_id is extracted by adapters: chat:NNN, thread:NNN, channel:NNN. Each unique routing key gets its own isolated Pool.
staging is default, not main
PRs target staging. Promotion to main via /promote. Main is the production branch deployed to Machine 1.
3 SQLite databases
auth.db (agents, credentials, auth), turns.db (L1 raw turn logging), vault DB (L3 semantic memory). Separate to avoid write contention.
Memory layers: 0, 1, 3 shipped
L0 = active context (compaction at 80% of 200k tokens). L1 = TurnStore (fire-and-forget writes). L3 = semantic (FTS5 + fastembed + sqlite-vec). L2 (episodic) and L4 (procedural) deferred.
Two machines, one always on
Machine 1 (RTX 3080, 24/7): hub, adapters, TTS, embeddings. Machine 2 (RTX 5070Ti, on-demand): heavy LLM. Connected via httpx HTTP/2 on local network.
cwd is machine-specific
[model].cwd in config.toml, NOT in agent TOML. [workspaces] in agent TOML registers /keyname slash commands that override cwd per pool. Switching workspace kills + respawns the subprocess.
Per-channel queue isolation
Each adapter has its own bounded inbound/outbound queue pair. InboundBus staging queue (max 100) feeds the Hub. Backpressure: queue full → immediate ACK + blocking put.
Library-first paradigm
voiceCLI, imageCLI are Python libraries imported directly (from voicecli import generate_async). No subprocess, no HTTP server for inter-project calls. CLI is a thin shell over the library.
