Layer Stack

Separation of concerns
Layer 0 — Transport

Adapters

Telegram, Discord, CLI. Emit InboundMessage carrying user_id and platform. Unchanged by this issue.

Layer 1 — Orchestration

Hub + Pool

Hub owns pool lifecycle, flush scheduling, task registry. Pool gains identity fields (session_id, user_id, medium). Pool.append() is the new internal mutator.

Layer 2 — Agent Logic

AgentBase

Gains 3 async methods: build_system_prompt(), compact(), flush_session(). Receives MemoryManager via DI. Opt-in (None is valid).

Layer 3 — Infra Adapter

MemoryManager

Pure infra — accepts only value types (str, int, SessionSnapshot). Zero domain imports. Wraps AsyncMemoryDB. Fully mockable in tests.

Layer 4 — Storage

roxabi-vault

Already installed. AsyncMemoryDB wraps SQLite + WAL + FTS5 + optional embeddings. Schema migration is idempotent on connect.

Value Boundary

SessionSnapshot

Frozen dataclass. The only object that crosses the agent↔memory boundary. Prevents MemoryManager from importing Pool.

Session End — What Gets Saved

flush_session() outputs
Immediate (awaited in flush_session)
  • session LLM-generated summary → upsert_session()
  • contact Updated last_seen → upsert_contact()
Background (asyncio tasks — non-blocking)
  • concept Named entities + relations extracted by LLM
  • preference Explicit user preferences extracted by LLM
  • Both registered in Hub._memory_tasks, awaited on shutdown

# NOT a replacement for submit() — submit() is the external entry point
def append(self, msg: InboundMessage) -> None:
    if not self.user_id:
        self.user_id = msg.user_id
        self.medium = msg.platform
    self.message_count += 1
    if self._turn_logger is not None:
        asyncio.create_task(self._turn_logger(self.session_id, msg))

S3 — AgentBase.build_system_prompt()
src/lyra/core/agent.py — AgentBase
async def build_system_prompt(self, pool: Pool) -> str:
    if pool._system_prompt:
        return pool._system_prompt  # cached — no L3 re-read per turn

    if self._memory is None:
        return self.config.system_prompt

    ns = self.config.memory_namespace
    anchor = await self._memory.get_identity_anchor(ns)
    if anchor is None:
        anchor = self.config.system_prompt  # first boot — seed
        await self._memory.save_identity_anchor(ns, anchor)

    memory_block = await self._memory.recall(
        pool.user_id, ns,
        first_msg=pool.history[0].text if pool.history else "",
    )
    parts = [anchor, memory_block]
    pool._system_prompt = "\n\n".join(filter(None, parts))
    return pool._system_prompt
S4 — Hub._evict_stale_pools() extension
src/lyra/core/hub.py — _evict_stale_pools()
def _evict_stale_pools(self) -> None:  # stays synchronous!
    now = time.monotonic()
    if (now - self._last_eviction_check) < self._pool_ttl / 10:
        return
    self._last_eviction_check = now

    stale = [pid for pid, pool in self.pools.items()
             if pool.is_idle and (now - pool.last_active) > self._pool_ttl]

    for pid in stale:
        pool = self.pools.pop(pid)
        if not pool.user_id:
            continue  # zero-message session — no-op
        agent = self.get_agent(pool.agent_name)
        if agent is not None:
            task = asyncio.create_task(  # ← NOT await (sync context)
                agent.flush_session(pool, "idle")
            )
            self._memory_tasks.add(task)
            task.add_done_callback(self._memory_tasks.discard)
S5 — Compaction trigger in _process_one()
src/lyra/core/agent.py + pool.py
# Constants (AgentBase)
MODEL_CONTEXT_TOKENS = 200_000  # claude-sonnet default
COMPACT_THRESHOLD = int(0.8 * MODEL_CONTEXT_TOKENS)  # = 160_000
COMPACT_TAIL = 10  # turns to keep after compaction

# Trigger check at end of _process_one():
token_estimate = sum(
    len(t.get("content", "")) // 4
    for t in pool.sdk_history
)
if token_estimate > COMPACT_THRESHOLD:
    await self.compact(pool)

# compact() body:
async def compact(self, pool: Pool) -> None:
    summary = await self._llm_summarize_incremental(pool)
    pool.sdk_history = deque(
        [{"role": "system", "content": summary}]
        + list(pool.sdk_history)[-COMPACT_TAIL:]
    )
    # Update L3 session record with latest summary
    if self._memory and pool.user_id:
        snap = pool.snapshot(self.config.memory_namespace)
        await self._memory.upsert_session(snap, summary)

Isolation Model

How user_id isolation is enforced
Write path

Every save_entry() and upsert_session() call passes namespace=agent_namespace AND stores user_id inside metadata JSON. Two-level isolation: by agent namespace and by user.

Read path

All retrieval in MemoryManager filters by both namespace and json_extract(metadata, '$.user_id'). Single enforcement point — no scattering across callers.

Security invariant Cross-user data leakage is impossible without explicitly bypassing MemoryManager. All L3 access goes through it — direct AsyncMemoryDB calls are never made from agent or pool code.
