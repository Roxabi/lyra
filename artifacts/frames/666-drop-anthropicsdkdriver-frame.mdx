## Problem

`AnthropicSdkDriver` (`src/lyra/llm/drivers/sdk.py`) was the original LLM
driver calling the Anthropic Messages API via the `anthropic` Python SDK.
It has since been superseded — all production traffic runs through
`ClaudeCliDriver` (CLI-only). The user has confirmed Lyra is now CLI-only
and wants the SDK path removed entirely.

Keeping the SDK driver costs:
- an unused `anthropic` runtime dependency
- dual backend branches in `agent_factory.py` (`anthropic-sdk` vs
  `claude-cli`) and related smart-routing / circuit-breaker plumbing
- an `AnthropicAgent` class (`src/lyra/agents/anthropic_agent.py`) with
  its own streaming/history logic that mirrors the CLI agent
- documentation drift across ADRs, CLAUDE.md files, and architecture docs

Removing it simplifies the LLM layer to a single CLI-based backend and
eliminates dead code + one PyPI dependency.

## Who

- **Primary:** Lyra maintainer (Mickael) — owns the LLM layer and agent
  lifecycle.
- **Secondary:** any agent TOML seed currently declaring
  `backend = "anthropic-sdk"` — must be migrated to `claude-cli` before
  removal (or the seeds are already unused).

## Constraints

- `#665` (LiteLLMDriver) is **not** a blocker for this change despite
  the issue body — user explicitly chose CLI-only, so LiteLLM is out of
  the picture for this cleanup.
- Must remove, not deprecate: no shims, no compat aliases.
- Remove the `anthropic` dependency from `pyproject.toml` and `uv.lock`.
- Update all CLAUDE.md + ADR references so docs reflect the new reality
  (mark historical ADRs as superseded, don't rewrite history).
- Tests referencing the SDK driver / agent are removed or migrated.
- Single domain: LLM layer (`src/lyra/llm/`, `src/lyra/agents/`,
  `src/lyra/bootstrap/factory/`) + tests + docs.

## Out of Scope

- `LiteLLMDriver` (#665) — separate effort, not blocked on nor blocking
  this one.
- Refactoring `ClaudeCliDriver` or the surviving CLI agent.
- Reworking the provider protocol / driver registry (ADR-016).
- Touching `NatsLlmDriver` or the NATS transport.
- AgentPool API redesign (only `sdk_history` accessors relevant to the
  SDK path are removed; the pool itself stays).

## Complexity

**Tier: F-lite** — clear scope, single domain, no unknowns.

- Risk surface limited to confirming no live agent seeds still declare
