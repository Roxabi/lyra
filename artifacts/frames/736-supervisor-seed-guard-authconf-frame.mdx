## Problem

Two silent drift sources made NATS auth problems invisible during the #689 Machine-1 voicecli cutover:

1. **Supervisor scripts don't validate seed readability.** `run_adapter.sh` and `run_hub.sh` source `.env` then `exec` the CLI unconditionally. If `NATS_NKEY_SEED_PATH` points at a missing/unreadable file, the Python NATS client raises `FileNotFoundError` mid-startup — but supervisor keeps reporting `RUNNING` until the process eventually crashes. Auth failures surface as opaque "connectivity" issues.
2. **`deploy/setup.sh` doesn't re-render `auth.conf` on re-provision.** Step 7 calls `gen-nkeys.sh --fix-perms`, which only touches file permissions. A fresh Machine 1 re-provision today would not render `auth.conf` to match the current IDENTITIES matrix — reproducing exactly the drift ADR-046 catalogues.

PR #735 fixed the specific legacy `.env` override that shadowed per-program seeds, but the structural file-existence guard and provisioning idempotency gap remain.

## Who

- **Primary:** Mickael operating Machine 1 during cutovers and re-provisions — currently blind to missing-seed failures until Python crashes.
- **Secondary:** Future on-call/operator recovering from a clean re-provision; supervisor's `autorestart=true` turns silent failures into visible `FATAL` state once the guard exits non-zero.

## Constraints

- Changes must be idempotent: `--regen-authconf --yes` must be safe to run on every provisioning pass.
- `--fix-perms`-only invocation must remain available for chmod-only runs (don't collapse the two modes).
- No regressions to existing `tests/nats/test_gen_nkeys_acls.sh` coverage.
- One-file edits in a single domain (deploy/supervisor hygiene) — F-lite scope.

## Out of Scope

- Revisiting NATS auth architecture or the IDENTITIES matrix (covered by epic #561).
- Changing how `.env` is loaded or scoped (PR #735 already addressed the override bug).
- Adding full-stack integration tests for NATS auth (guard test is sufficient).
