## Context

Commit C3 (#456) moved trust resolution to the Hub side, introducing `hub.register_authenticator()` as the canonical auth registration pattern. The old `auth=` constructor parameter on `TelegramAdapter` and `DiscordAdapter` was kept as a backwards-compat shim with a DeprecationWarning.

This backwards-compat is no longer needed. The deprecated parameter and its warning code should be removed to clean up the codebase and eliminate deprecation noise in test output.

Additionally, upstream-only deprecation warnings (`audioop`, `websockets.legacy`) should be suppressed in pytest config since Lyra cannot fix these — they're library-level issues.

**Promoted from:** [Frame: Remove deprecated auth= constructor parameter](../frames/847-remove-deprecated-auth-param-frame.mdx)

## Goal

Remove the deprecated `auth=` constructor parameter from TelegramAdapter and DiscordAdapter, migrate all tests to the `hub.register_authenticator()` pattern, and suppress upstream-only deprecation warnings in pytest config.

## Users & Use Cases

- **Developers:** Cleaner codebase, no deprecation warnings from Lyra code
- **Future maintainers:** No mixed signals about canonical auth pattern (`hub.register_authenticator()`)
- **CI/CD pipeline:** Cleaner test output, no spurious warnings

## Expected Behavior

### Happy path

1. Developer instantiates `TelegramAdapter` or `DiscordAdapter` without `auth` parameter
2. Auth is registered via `hub.register_authenticator(platform, bot_id, auth)` before adapter starts
3. Adapter retrieves auth from hub when needed
4. Tests run without any DeprecationWarning from Lyra code
5. Upstream warnings (audioop, websockets) are suppressed in pytest output

### Edge cases

| Case | Handling |
|------|----------|
| Old code still using `auth=` param | Compile-time error — param removed |
| Test using `auth=_ALLOW_ALL` | Migrate to hub.register_authenticator pattern |
| Test directly sets `adapter._auth` | Remove `_auth` assignment; inject auth via hub mock if needed |
| Auth not registered before adapter starts | Existing behavior — deny-all fallback |

## Constraints

- **Files affected:** 20 files (2 adapters + 18 test files)
- **Mechanical changes:** Auth flow unchanged, only parameter removal and test migration
- **Low risk:** No behavioral changes, only API cleanup
- **Upstream warnings:** `audioop` (discord.py, Python 3.13), `websockets.legacy` (uvicorn/discord.py) — suppress, not fix

## Non-goals

- Changing the auth flow logic (Hub-side trust resolution remains unchanged)
- Fixing upstream deprecation warnings in discord.py or uvicorn
- Any NATS or distributed auth changes

## Technical Decisions

| Decision | Rationale |
|----------|-----------|
| Remove param entirely (not deprecation cycle) | Internal API, no external consumers |
| Use `hub.register_authenticator()` in tests | Matches production pattern, canonical approach |
| Suppress upstream warnings in pytest config | Cannot fix library code, reduces noise |
