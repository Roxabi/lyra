Agents
Create, edit, and manage agents. DB-first — TOML files are seed sources only.
💾
AgentStore — SQLite, DB-first. Agent config lives in ~/.lyra/auth.db. TOML files (~/.lyra/agents/ or src/lyra/agents/) are seed sources only — import them with lyra agent init. Startup reads the DB only. Hot-reload polls updated_at on each agent row — no restart needed for persona/voice changes.
Create / update an agent
1
Scaffold a new agent TOML
lyra agent create my_agent   # interactive wizard → writes TOML
📁
User overrides: Place a file at ~/.lyra/agents/my_agent.toml to override the system default without touching the repo. User-level files take precedence during seed.
2
Edit the TOML — v12 fields
tomlsrc/lyra/agents/my_agent.toml
[agent]
name             = "my_agent"
memory_namespace = "my_agent"
show_intermediate= true
streaming        = 1          # token streaming (#321)

# Inline persona — replaces legacy persona string (#343 · ADR-029)
[agent.persona_json]
name         = "Lyra"
role         = "Personal AI Assistant"
instructions = "You are Lyra, a precise and helpful assistant..."
language     = "en"
style        = "concise"

fallback_language = "en"  # used when locale detection fails (#343)

[tts]
engine   = "qwen-fast"     # qwen | qwen-fast | chatterbox
voice    = "Ono_Anna"
language = "French"
chunked  = true

[stt]
language_detection_threshold = 0.95
language_fallback            = "en"

# Trigger patterns for smart routing (#343)
[agent.patterns_json]
code    = ["write.*code", "implement", "refactor"]
search  = ["find", "search for", "look up"]

[model]
backend  = "claude-cli"
model    = "claude-sonnet-4-6"
max_turns= 10
tools    = ["Read", "Grep", "Glob", "WebFetch", "WebSearch"]
skip_permissions = true

[agent.smart_routing]       # requires backend = "anthropic-sdk"
enabled = false

[plugins]
enabled = ["echo", "search"]

[workspaces]
lyra     = "~/projects/lyra"
projects = "~/projects"

# Commands loaded by CommandLoader (#345)
[commands]
enabled = ["help", "clear", "workspace"]

# Passthroughs: commands forwarded direct to LLM (no special handling)
[passthroughs]
enabled = ["ask", "q"]
📦
Legacy fields: persona (text string) still works — auto-migrated to persona_json on AgentStore.connect(). Use [tts] / [stt] sections (mapped to tts_json / stt_json) for voice config.
3
Import into DB
lyra agent init          # seed DB from TOML files (skip if exists)
lyra agent init --force  # overwrite existing DB rows with TOML values
🔁
COALESCE upsert: On init --force, persona_json, patterns_json, and passthroughs_json use COALESCE(new, existing) — a NULL value in the TOML preserves the existing DB value rather than overwriting it.
4
Assign agent to bot
lyra agent assign my_agent --platform telegram --bot my_bot
# Creates row in bot_agent_map table

lyra start   # or: make lyra
🔗
Alternatively, wire the agent in config.toml via [[telegram.bots]] bot_id = "my_bot" agent = "my_agent". Both approaches write to bot_agent_map.
Agent config pipeline
~/.lyra/agents/<name>.toml
▸
src/lyra/agents/<name>.toml
▸ lyra agent init ▸
auth.db (AgentStore)
← runtime reads here
User TOML wins over system TOML during seed. After import, DB is authoritative. Hot-reload polls updated_at — two-stage: _maybe_reload_config() + _maybe_reload_plugins().
CLI commands
CommandDescription
lyra agent initSeed DB from TOML files (safe — skips existing rows)
lyra agent init --forceOverwrite existing DB rows from TOML. COALESCE for JSON fields.
lyra agent create <name>Interactive wizard → scaffold TOML at appropriate path
lyra agent listList all agents in DB with backend, model, source, bot assignments
lyra agent show <name>Full config dump from DB — all JSON fields expanded
lyra agent edit <name>Edit interactively in DB. Bumps updated_at → triggers hot-reload.
lyra agent validate <name>Validate config against schema (backend, JSON arrays, smart routing)
lyra agent assign <name> --platform <p> --bot <id>Assign agent to bot. Creates row in bot_agent_map.
lyra agent unassign --platform <p> --bot <id>Remove assignment. Bot reverts to default agent.
lyra agent delete <name>Delete from DB. Refuses if any bot is still assigned.
Editable fields
Via lyra agent edit
backend — claude-cli | anthropic-sdk
model — e.g. claude-sonnet-4-6
max_turns — integer
show_intermediate — true/false
cwd — working directory
memory_namespace — memory isolation key
i18n_language — response language (default: en)
streaming — 0/1 token-level streaming (#321)
persona_json — inline persona (name, role, instructions…)
tts_json / stt_json — TTS and STT config (#343)
fallback_language — locale fallback (#343)
Via TOML + init --force
tools — tool allowlist
commands_json — CommandLoader config (#345)
passthroughs_json — commands forwarded to LLM
plugins_json — enabled extensions list
smart_routing_json — routing config
workspaces_json — workspace → cwd mappings
patterns_json — trigger patterns (#343)
skip_permissions — bypass CLI permission prompts
permissions_json — permission scope list
📌
cwd is machine-specific — always set it in config.toml [defaults], not in the agent TOML. Workspaces each become accessible via /workspace <key>.
Multi-bot — N agents × N bots
Two bots with different personas
tomlconfig.toml
# Bot 1: Lyra — general assistant
# lyra bot add --platform telegram --bot-id lyra
[[telegram.bots]]
bot_id = "lyra"
agent  = "lyra_default"

# Bot 2: Aryl — different model/persona
# lyra bot add --platform telegram --bot-id aryl
[[telegram.bots]]
bot_id = "aryl"
agent  = "aryl_default"

[[auth.telegram_bots]]
bot_id      = "lyra"
default     = "blocked"
owner_users = [123456789]

[[auth.telegram_bots]]
bot_id  = "aryl"
default = "trusted"   # public bot — anyone can chat
Auth trust levels
LevelAccessConfig key
ownerFull access — all commands including admin-only. Auto-admin.owner_users = [ID]
trustedCan chat with bot. Cannot run admin commands.trusted_users / trusted_roles
publicEveryone can interact (no allow-list check).default = "trusted"
blockedMessages silently ignored.default = "blocked"
LLM Backends
claude-cli default
Spawns claude as a subprocess. Requires Claude Code CLI + active subscription. Supports streaming, tools, max_turns.
anthropic-sdk
Direct Anthropic API via SDK. Set ANTHROPIC_API_KEY in .env. Enables smart routing (haiku/sonnet/opus per complexity).
ollama
Local LLM via Ollama HTTP API. Set backend = "ollama" and model = "llama3" (or any pulled model). Your data never leaves the machine.
