## Context

Promoted from approved frame (`artifacts/frames/9-memory-layer-phase-1-frame.mdx`).
Architecture decision: ADR-011 (`docs/architecture/adr/011-memory-layer-package-structure-vault-vs-standalone.mdx`).
Reference exploration: `artifacts/explorations/02-memory.md`.

Two failure modes today: **overflow amnesia** (pool.history truncated mid-session) and **session amnesia** (full context lost on conversation close/idle). Both are solved by this spec.

Current code anchors:
- `src/lyra/core/agent.py` — `Agent` dataclass (`system_prompt: str` static), `AgentBase` ABC
- `src/lyra/core/pool.py` — `Pool` dataclass (`history: list[Message]`, TODO comment on compaction)

## Goal

Ship `roxabi-memory` — a standalone Python package that gives Lyra agents persistent semantic memory via hybrid search (BM25 + embeddings), eliminating overflow and session amnesia with zero data loss.

## Users

- **Primary:** Lyra agents (`AgentBase` subclasses) — consume `AsyncMemoryDB`, gain session coherence
- **Secondary:** Claude Code vault skills — consume `MemoryDB` (sync), must continue working unchanged
- **End user:** Mickael — benefits from an agent that remembers across sessions, channels, and context resets

## Expected Behavior

### Package installation

`roxabi-memory` is a standalone Python package with two install targets:
- `pip install roxabi-memory` — sync interface only (vault skills)
- `pip install roxabi-memory[embeddings]` — adds fastembed + sqlite-vec (Lyra)

On first `MemoryDB.connect()` or `AsyncMemoryDB.connect()`, the schema migration runner checks `PRAGMA user_version`. If version < 2, it applies `ALTER TABLE` statements to add the four new columns (`namespace`, `embedding`, `source_turns`, `event_date`) without touching existing data.

### Agent boot (identity anchor)

When `get_or_create_pool()` is called in the Hub, `AgentBase.build_system_prompt(pool)` is called instead of reading `agent.config.system_prompt` directly. It fetches the `IDENTITY_ANCHOR` record from L3 (namespace = `agent.memory_namespace`). On first boot (no record exists), it seeds `IDENTITY_ANCHOR` from the agent's TOML `[prompt] system` field and stores it in L3. Subsequent boots retrieve the stored record. This allows Lyra to evolve its identity over time (future).

### Retrieval (pre-response injection)

Before each `AgentBase.process()` call, the base class calls `self.memory.search(msg.content, namespace=self.config.memory_namespace, limit=5)`. Results are prepended to the system prompt as a `[Memory]` block. The agent sees relevant past context without replaying full history. Agents also see `namespace='vault'` entries (shared knowledge from vault skills and 2ndBrain).

### L0 compaction (overflow)

`Pool.append(msg)` checks token count after each message. When `pool.history` exceeds the compaction threshold (configurable, default 80% of model context limit), `AgentBase.compact()` is triggered:

1. Takes `pool.history[:-TAIL]` (old turns, keeping last `TAIL` turns)
2. Calls LLM to summarize old turns (includes existing session summary for incremental update)
3. Upserts the session record in L3 (`type='session'`, keyed by `pool.session_id`)
4. Replaces `pool.history` with `[SystemMessage(summary)] + pool.history[-TAIL:]`

### Session end / idle flush

Two triggers call `AgentBase.flush_session()`:
1. **Explicit close** — adapter signals session end (Telegram disconnect, CLI exit)
2. **Idle > 10h** — `Pool._last_activity` timestamp checked by a per-pool `asyncio` timer

`flush_session()` writes the final summary to L3 and sets `session_end` on the session record. The session record is searchable via FTS5 — future sessions can retrieve "what did we discuss last time?"

### Data migration

The `vault:vault-migrate` skill is updated to use `roxabi-memory`'s sync interface. Migrated 2ndBrain entries get `namespace='vault'` and have embeddings generated via fastembed on import. Migration is non-destructive (dedup by source URL/ID) and idempotent.

## Breadboard

### Package API surface

| Affordance | Handler | Data |
|------------|---------|------|
| `MemoryDB.connect(path)` | Migration runner (user_version pragma) | `entries` table v2, `entries_fts` |
| `MemoryDB.save_entry(content, type, namespace, ...)` | INSERT INTO entries | entry row |
| `MemoryDB.search_fts(query, namespace)` | FTS5 BM25 query | ranked entry rows |
| `AsyncMemoryDB.connect(path)` | async migration runner (aiosqlite) | same schema |
| `AsyncMemoryDB.search(query, namespace, limit)` | `hybrid_search()` | ranked entry rows |
| `AsyncMemoryDB.upsert_session(session_id, ...)` | INSERT OR REPLACE entries | session record |
| `hybrid_search(query, namespace)` | BM25 score + cosine score → ranked merge | top-K entries |
| `embed(text)` → vector | fastembed ONNX (run_in_executor) | float32 blob |
| `NamespacedReader.search(query)` | AsyncMemoryDB.search + namespace filter | scoped rows |
| `NamespacedWriter.save(content, type, ...)` | AsyncMemoryDB.save_entry + namespace | entry row |

### AgentBase extensions

| Affordance | Handler | Data |
|------------|---------|------|
| `AgentBase.build_system_prompt(pool)` | fetch IDENTITY_ANCHOR from L3; seed if missing | `IDENTITY_ANCHOR` entry |
| `AgentBase.compact()` | summarize old turns → upsert session record → truncate history | session record in L3 |
| `AgentBase.flush_session(reason)` | final summary → set session_end | session record in L3 |
| `Pool.append(msg)` | check token count → trigger compact() if threshold exceeded | `pool.history`, `pool.message_count` |

### Session record (stored as `type='session'` in entries)

| Field | Type | Notes |
|-------|------|-------|
| `id` | TEXT PK | UUID = session_id |
| `type` | TEXT | `'session'` |
| `namespace` | TEXT | = agent_namespace |
| `content` | TEXT | LLM-generated summary (updated incrementally) |
| `title` | TEXT | Auto-generated 1-line title from first summary |
| `metadata` | JSON | `{user_id, agent_namespace, medium, session_start, session_end, message_count, source_turns}` |
| `source_turns` | TEXT | JSON array of compacted turn refs |
| `event_date` | TEXT | = session_start (for timeline queries) |

### Schema v2 additions (ALTER TABLE, non-breaking)

```sql
ALTER TABLE entries ADD COLUMN namespace TEXT NOT NULL DEFAULT 'vault';
ALTER TABLE entries ADD COLUMN embedding BLOB;           -- nullable float32 vector
ALTER TABLE entries ADD COLUMN source_turns TEXT;        -- nullable JSON array
ALTER TABLE entries ADD COLUMN event_date TEXT;          -- nullable ISO datetime
```

Migration runner: `PRAGMA user_version = 2` after applying. Idempotent — checks version before running.

### Pool additions

```python
@dataclass
class Pool:
    pool_id: str
    agent_name: str
    session_id: str           # UUID, generated at pool creation
    user_id: str              # canonical user identity
    medium: str               # "telegram" | "discord" | "cli" | ...
    session_start: datetime   # first message timestamp
    history: list[Message] = field(default_factory=list)
    message_count: int = 0
    _last_activity: datetime = field(...)
    _lock: asyncio.Lock = field(...)
```

## Success Criteria

### Package

- [ ] `roxabi-memory` installable as `pip install roxabi-memory` (sync only) and `pip install roxabi-memory[embeddings]` (adds fastembed + sqlite-vec)
- [ ] Schema v2 migration runs on existing `vault.db` without data loss — all four columns added, existing rows get `namespace='vault'` default
- [ ] `user_version` pragma checked before migration; migration is idempotent (safe to run twice)

### Sync interface (vault skills)

- [ ] `MemoryDB` sync interface passes all existing vault skill smoke tests — behavior unchanged
- [ ] `roxabi-vault` plugin refactored to thin adapter over `MemoryDB`

### Async interface (Lyra)

- [ ] `AsyncMemoryDB` `search()` and `save_entry()` don't block the asyncio event loop (verified with `asyncio.get_event_loop().is_running()` check + `run_in_executor` for ONNX inference)
- [ ] `NamespacedReader.search(query)` returns entries from `agent.memory_namespace` AND `namespace='vault'`; no other namespaces visible

### Search

- [ ] FTS5/BM25 keyword search returns ranked results for exact terms, proper nouns, and dates
- [ ] `hybrid_search()` returns merged ranking of BM25 scores + cosine similarity scores
- [ ] Rows with `embedding IS NULL` handled gracefully — included in BM25 results, excluded from cosine ranking (no crash)

### Agent boot

- [ ] `IDENTITY_ANCHOR` seeded from TOML `[prompt] system` on first boot; stored in L3 with `namespace=agent.memory_namespace`
- [ ] Subsequent boots retrieve `IDENTITY_ANCHOR` from L3 — TOML field ignored if record exists
- [ ] `AgentBase.build_system_prompt(pool)` replaces direct read of `agent.config.system_prompt` in `process()`

### L0 compaction

- [ ] `Pool.append(msg)` increments `message_count` and updates `_last_activity`
- [ ] `AgentBase.compact()` triggered when `pool.history` token count exceeds threshold (default: 80% of model context)
- [ ] Compaction produces a summary that includes any existing session summary (incremental — not a fresh summary from scratch)
- [ ] Session record in L3 upserted by `session_id` — one record per session, not one per compaction
- [ ] After compaction: `pool.history = [SystemMessage(summary)] + pool.history[-TAIL:]`

### Session lifecycle

- [ ] `AgentBase.flush_session(reason='end')` called on explicit adapter disconnect — writes final summary, sets `session_end`
- [ ] `AgentBase.flush_session(reason='idle')` called after 10h of inactivity — same behavior
- [ ] Session record carries: `user_id`, `agent_namespace`, `medium`, `session_start`, `session_end`, `message_count`, `source_turns`, `title`
- [ ] Session records are searchable via FTS5 (summary text indexed)

### Data migration

- [ ] vault-migrate skill updated: migrated entries receive `namespace='vault'`
- [ ] Embeddings generated for migrated entries via fastembed ONNX at import time
- [ ] Migration is non-destructive: existing entries not overwritten (dedup by source URL/ID)
- [ ] Migration is idempotent: running twice produces the same result
