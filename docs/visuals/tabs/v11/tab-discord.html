Watch Channels #347
New in V11. Discord channels listed in watch_channels (inside settings_json on bot_agent_map) bypass the mention filter. The bot processes all messages in those channels automatically.
Normal channel
Mention required

Bot only responds when directly @mentioned. All other messages are ignored.

Watch channel
Always active

Every message triggers the agent. No mention needed. Threads auto-created per-conversation. Links auto-processed.

Configuration

Watch channels are stored in settings_json on the bot_agent_map row for a specific bot.

jsonbot_agent_map.settings_json
{
  "watch_channels": [
    "1234567890",     // Discord channel ID (string)
    "9876543210"
  ],
  "auto_thread": true,  // create thread per-conversation
  "process_links": true // auto-fetch + summarize URLs
}
bashAdding a watch channel via CLI
# Edit settings_json for a specific bot
lyra agent edit <agent_name>
# Then update the bot_agent_map row directly in auth.db,
# or use lyra agent assign with settings
Message Processing Flow
1Message arrives in Discord channeladapter receives event
2Is channel in watch_channels?check settings_json
3aYes → bypass mention filteralways process
3bNo → mention requiredskip if not @mentioned
4Contains link? → auto-fetch contentprocess_links: true
5Create/resume threadauto_thread: true
6Route to agent poolnormal backend dispatch
Thread Behavior
Auto-create
First message in a watch channel creates a Discord thread. Subsequent messages from the same user in the same channel resume the thread.
Thread naming
Thread title derived from the first message (truncated). Bot replies inside the thread to maintain context separation from the main channel.
Discord-Specific Commands
Native Discord slash commands are registered separately from Lyra bot commands. Use /help inside Discord to see the full list of available commands for the assigned agent.
