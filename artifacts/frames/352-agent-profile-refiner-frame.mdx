## Problem

Agent configuration (persona, voice, passthroughs, patterns, model) is currently
editable only via `lyra agent edit`, a field-by-field terminal prompt with no
conversational guidance. There is no way to refine an agent's profile from within
the bot itself, and no design-time tool to reason about what changes to make.

As agents accumulate settings across multiple DB columns, the edit experience
becomes increasingly mechanical and context-free — you must know what to change
before you can change it. There is no surface that reads the full profile,
explains what each setting does, and guides you toward a better configuration.

## Who

- **Primary:** Mickael (operator) — configuring and tuning Lyra and Aryl agents
  from the CLI, from within Telegram/Discord, or from Claude Code during development
- **Secondary:** Future operators managing multiple agents on different machines

## Constraints

- Three surfaces (CLI, bot, skill) must share a single refinement core to avoid
  logic drift
- Bot surface cannot write to the DB directly — requires a new `lyra_agent_patch`
  tool callable by Claude via the claude-cli backend
- Skill surface is design-time only (no DB access from Claude Code) — outputs
  a config suggestion the operator applies manually via CLI
- Must use existing `AgentStore.upsert()` and `AgentRow` patterns — no new DB schema
- Bot passthrough registers via the existing `register_passthrough()` mechanism
  (no new Python plugin file needed)

## Out of Scope

- Changing model backend or max_turns (infra settings, not profile)
- Multi-user / multi-operator agent management
- Automated or scheduled refinement (always operator-initiated)
- Refining agents not yet in DB (creation is a separate flow)

## Complexity

**Tier: F-lite** — Clear scope, single domain (agent management). Three surfaces
but all built on known patterns: AgentStore, register_passthrough, typer CLI,
roxabi-plugins skill scaffold. No new architecture required.

Signals observed:
- 5–8 files touched across lyra + roxabi-plugins
- Technical risk low: known patterns throughout
- Architectural impact: new `AgentRefiner` class, new `agent_tools.py` — contained
- 0 meaningful unknowns
- 2 domains: lyra core + roxabi-plugins (skill)
