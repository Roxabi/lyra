## Problem

Today each channel adapter (Telegram, Discord, etc.) passes platform-specific structures toward the hub. The hub implicitly assumes certain fields exist — `msg.text`, attachments, thread context — but there is no formal typed contract. This creates three concrete problems:

1. **Adapter authoring friction:** a new adapter author has no spec for what fields they must produce.
2. **Hub fragility:** hub and agent logic reads platform-specific fields directly, breaking when a new adapter omits or renames them.
3. **Test complexity:** unit-testing hub logic requires constructing real adapter objects rather than simple plain dataclasses.

This is part of **Phase 1b — Agent core** (#73) and ships before the harder OutboundMessage normalization (sibling issue), since input normalization is mostly additive and immediately improves hub testability.

## Who

- **Primary:** Hub and agent code — they currently receive untyped platform-specific dicts; after this change they only ever see `InboundMessage`.
- **Secondary:** Future adapter authors — `ChannelAdapter.normalize()` protocol gives them a clear contract to implement.

## Constraints

- `InboundMessage` must be a **frozen dataclass** (immutable, hashable).
- Defined in `core/message.py` (or equivalent canonical location).
- `scope_id` field sourced from #125 (already done).
- Both `TelegramAdapter` and `DiscordAdapter` must be updated to produce `InboundMessage`.
- No platform-specific fields may be accessed above the adapter layer after this change.
- `Attachment` dataclass required (type, url/bytes, mime_type, filename).

## Out of Scope

- OutboundMessage normalization (separate sibling issue).
- Voice/web gateway adapters (unblocked by this, not part of it).
- Locale/language detection logic (#42) — `locale` field is a passthrough hint only.
- Platform-specific rendering rules.

## Complexity

**Tier: F-lite** — clear scope, single domain (bus/adapter layer), no architectural unknowns. Work is mostly additive: define the dataclass, update two existing adapters, add normalization tests.
