Sending your first message

Open Telegram and message the bot associated with your lyra_default agent. Any text message starts a conversation. Lyra creates a Pool for your chat ID and assigns the configured agent.

In a group chat, Lyra only responds to messages that mention the bot's @username or reply to one of its messages — it doesn't eavesdrop on all messages by default. Discord watch channels are the exception — see the Commands tab.

Built-in session commands

These work in any conversation regardless of which agent is active.

CommandWhat it does
/newClear conversation history and start fresh
/add <text>Append text to the system prompt for this session
/explainAsk the agent to explain its last response in simpler terms
/summarizeGet a summary of the conversation so far
/statusShow current agent, model, tools, and circuit state
/helpList all available commands — split into Commands and Plugins sections
/add, /explain, and /summarize are registered by AnthropicAgent only. They're not available for SimpleAgent (claude-cli backend) agents.

Understanding the response lifecycle

  1. Message arrives at adapterTelegram or Discord adapter receives the webhook / poll update and wraps it in an InboundMessage. Native platform commands are handled here before the message reaches Hub.
  2. InboundBus queues itThe adapter pushes to the platform's async queue. Each platform has one queue; backpressure is handled here.
  3. Hub dispatches to PoolHub looks up (or creates) the Pool for this scope_id. Pool holds the conversation history and circuit state.
  4. AgentBase hot-reload checkOn every message, _maybe_reload_config() compares the cached updated_at string against AgentStore. If changed, config is rebuilt from the DB row — no restart needed. Then _maybe_reload_plugins() reloads extension commands if needed.
  5. Agent generates responseThe agent calls the LLM API, runs tools, streams tokens back to adapters. Multi-turn tool use is handled transparently.
  6. OutboundDispatcher deliversResponse is queued in the per-bot outbound queue. Retries on transient failures. Circuit breaker opens after sustained errors.

Troubleshooting a silent bot

bash
# Check if daemons are running
make lyra status

# Tail live logs (stdout)
make lyra logs

# Check for errors
make lyra errors

# Verify agent is in DB
lyra agent list
lyra agent show lyra_default
