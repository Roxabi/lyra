01 — The Two Prefixes
/
/command
Strict mode — hard command
known Dispatched to registered handler (builtin, session, plugin, or passthrough)
unknown Returns "Unknown command: /foo — Type /help". LLM never sees it.
!
!command
Soft mode — fallthrough allowed
known Dispatched identically to /command — same handler, same result
unknown Router returns None → hub falls through to LLM pool. The message text is forwarded verbatim.

03 — Routing Layers
Layer 0 — Pre-routing
Adapter-level (Discord only)
adapters/discord.py · lines 343–369
  • !join
  • !join stay
  • !leave

Handled before mention filter. Connects/disconnects from voice channel. Never reaches the hub router. Guild-only.

Layer 1 — Builtins
CommandRouter._DEFAULT_BUILTINS

  • /stop
  • /clear
  • /new
  • /config
  • /circuit
  • /routing
  • /folder

Inline handlers. /circuit, /routing, /config are admin-only.

Layer 2 — Workspaces
Dynamic from agent TOML
command_router.py · lines 124–130, 264–267
  • /lyra
  • /projects
  • /roxabi-vault
  • …custom

Defined in [workspaces] in agent TOML. Switch Claude's cwd per pool. Persists until another workspace cmd or /clear.

Layer 3 — Session Commands
Isolated LLM calls
session_commands.py
  • /add
  • /explain
  • /summarize
  • /search

Use pool_id="session:X". Never read or write the real pool history. Bare URLs auto-rewrite to /add.

Layer 4 — Plugins
plugin.toml + handlers.py
plugin_loader.py · lines 82–250
  • /echo
  • /svc
  • …custom

Declared in plugin.toml, enabled per-agent. Hot-reload without restart. No LLM needed (usually).

Layer 5 — Passthrough
Skip dispatch → go to LLM
agent.py · line 641, lines 797–824
  • /voice

Registered via register_passthrough("voice"). Rewrites message with modality="voice", then LLM reply → TTS → OGG/Opus native audio.

04 — Message Lifecycle
1
Adapter receives raw platform event
Telegram's on_message() or Discord's on_message() fires. For Discord: pre-check for !join / !leave before mention filter. Result: InboundMessage pushed to hub.inbound_bus.
2
Hub parses prefix
CommandParser.parse() checks for / or ! prefix. Produces CommandContext(prefix, name, args), attached to msg.command. Then router.prepare() rewrites bare URLs to /add.
3
CommandRouter.dispatch() — ordered lookup
Checks builtins → workspaces → session commands → plugins → passthrough, in that order. First match wins. Returns a Response or None.
4
LLM pool or error
If dispatch returned None (no handler, or !-prefix fallthrough): _submit_to_pool() → LLM agent. If / prefix and unrecognised: returns "Unknown command" error.
5
Response dispatch
Hub's dispatch_response() checks Response.speak. If true → TTS pipeline → OGG/Opus. Adapter sends as native voice message (Discord IS_VOICE_MESSAGE flag, Telegram send_voice).
05 — Platform Native Commands
⚠️
No native platform command menus
Lyra does not call Telegram's setMyCommands() or Discord's app_commands tree sync. All command handling is application-level text parsing — the same set works identically on both platforms, plugins hot-reload without resyncing Discord's command tree, and there is full control over help text and discovery.

To add Telegram autocomplete (cosmetic only), call bot.set_my_commands([BotCommand(...)]) at startup. The routing stays unchanged.
06 — Where Should a New Command Go?
Scenario Layer How to add it
Affects session or bot state (clear history, switch workspace) Hub Builtin Add to _DEFAULT_BUILTINS in command_router.py + implement handler method
Isolated LLM task that must not pollute chat history Session Command Add handler in session_commands.py, register in CommandRouter._session_commands
Feature that can be toggled per-agent, hot-reloadable Plugin Create plugins/myname/plugin.toml + handlers.py, enable in agent TOML
Needs LLM reasoning but user wants /-like entry syntax Passthrough Call router.register_passthrough("name") in agent.py, rewrite message before LLM
Discord voice channel control (guild-only, stateful, pre-mention) Adapter-level Add to _handle_voice_command() in adapters/discord.py before mention filter
User wants command syntax but fallthrough to LLM is OK ! Prefix Nothing to add — user just types !anything and the router falls through automatically
Everything else LLM Pool No command needed — plain text goes straight to the LLM agent
