Memory agent

Lyra supports a dedicated memory agent that can persist facts, preferences, and context across conversations. Memory is stored as markdown files in a structured directory, indexed by type.

user

Role, goals, expertise, preferences. Shapes how Lyra tailors responses.

feedback

Corrections and guidance given to the agent. Prevents repeated mistakes.

project

Ongoing work, decisions, deadlines, stakeholder context.

reference

Pointers to external systems: Linear boards, Grafana dashboards, Slack channels.

Memory file format

markdown~/.lyra/memory/user_role.md
---
name: user_role
description: User is a senior ML engineer focused on LLM infra
type: user
---

Works primarily on Python backend systems. Prefers concise responses.
Familiar with asyncio, SQLite, supervisord, FastAPI.

An index file (MEMORY.md) lists all memory files with one-line descriptions. This index is loaded into context on every conversation start.

When Lyra saves memory

  • When you explicitly say "remember that…" or "note that…"
  • When you correct the agent's behavior in a way that should apply to future conversations
  • When significant project context is shared (deadlines, decisions, constraints)
Memory is not used for in-conversation state. Use it for facts that should persist across session boundaries. Ephemeral task state belongs in the conversation history, not in memory files.
