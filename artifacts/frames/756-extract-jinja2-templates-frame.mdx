## Problem

`build.py` embeds ~580 lines of CSS (CSS_BASE), ~20 lines of JS (THEME_SCRIPT), and all HTML as f-string concatenation within `build_html()`. This makes it:

- Hard to edit/test styling independently
- Impossible to reuse components across pages
- Difficult to use CSS/HTML tooling (linters, minifiers)
- A pain to maintain as the visual design evolves

The inline approach works for a single-page prototype but scales poorly. Any CSS tweak requires editing Python strings, and there's no separation of concerns.

## Who

- **Primary:** Lyra maintainers editing the dep-graph visuals
- **Secondary:** Anyone extending the dep-graph to new views or reusing the card/milestone templates

## Constraints

- Output must match current golden HTML byte-for-byte (visual regression test)
- Keep `build.py` under 300 lines after extraction (per #753 exemption elimination goal)
- No new runtime dependencies beyond `jinja2` (already in stdlib for some Pythons, but we'll add via uv)
- Templates must remain loadable from the `dep_graph/` package (not external paths)

## Out of Scope

- CSS minification / optimization (separate pass)
- Theme variants beyond light/dark (current toggle is sufficient)
- New layout modes (this is pure extraction)
- Adding new visual features (strictly refactoring)

## Complexity

**Tier: F-lite** — Clear scope, single domain (dep-graph build), ~4-10 files affected, no architectural unknowns.
