## Problem

Slice S4 of #658. The voice infrastructure is migrating to NATS-based communication. The old voice satellites (`lyra_stt`, `lyra_tts`) running inside the Lyra process are being replaced by standalone voicecli workers that communicate via NATS. Once the voicecli NATS workers are healthy on Machine 1 (S3 cutover complete), the old Lyra-side voice satellite code becomes dead weight and must be deleted.

This cleanup removes:
1. Voice satellite bootstrap files and CLI commands
2. Voice service classes (`STTService`, `TTSService`) and their voicecli imports
3. Supervisor configs for voice satellites
4. Related tests
5. Doc references to deleted components

## Who

- **Primary:** Lyra maintainers — cleaner codebase, no stale voicecli coupling
- **Secondary:** DevOps — simplified deployment topology (no more voice satellites in Lyra process)

## Constraints

- **Prerequisite:** voicecli nats-serve must be healthy on Machine 1 (S3 cutover complete) — deleting satellites before then breaks prod
- **Files must remain importable:** Protocol classes, error types, config types, result types stay (used by NATS-based consumers)
- **Tests:** Delete satellite-specific tests; keep NATS-client tests
- **Doc sync:** Co-located with deletes to avoid stale references
- **Blocks #658 S5** (drop-dep + deploy.sh + CI)

## Out of Scope

- Removing `voicecli` from pyproject.toml dependencies (S5)
- Deploy scripts for voicecli workers (separate issue)
- Containerizing voicecli workers (#728, Lane B)
- NATS contract changes (already done in S3)
