Migration steps
StepAction
2.1 Install NATS server on Machine 1 (~20MB binary)
2.2 Add nats-py async client to Lyra deps
2.3 Create NatsBus — same interface as LocalBus
2.4–5 Adapters publish to lyra.inbound.{ch}, Hub subscribes
2.6 LLM worker on Machine 2 — lyra.task.llm queue group
2.7–8 Health checks + fallback to cloud if M2 offline
Key design decisions
→ NATS server on Machine 1 (24/7, single node)
→ No JetStream yet — plain pub/sub sufficient
→ asyncio.Queue kept as local fallback
→ Hub depends on Bus interface, not implementation
→ Config toggle: bus: local | nats
Why NATS
~20MB binary · <1ms latency · native queue groups · zero ops overhead · optional JetStream persistence
