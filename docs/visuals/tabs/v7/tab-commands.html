Session commands & bare URL detection
How session commands work
Each call makes an isolated LLM call — never reads or writes pool history
Uses claude-haiku-4-5 — fast and cheap
Pool ID: session:add, session:explain, etc.
Bare URL auto-rewrites to /add <url> — just paste a link
chat
# Explicit session commands
You:  /add https://example.com/article
Bot:  ✓ Saved: "Article Title" [tag1, tag2]

You:  /explain https://github.com/foo/bar
Bot:  This library does... [plain explanation]

You:  /search embeddings
Bot:  Found 3 entries: ...

# Bare URL — auto-rewrites to /add
You:  https://docs.anthropic.com/...
Bot:  ✓ Saved: "Anthropic Docs" [ai, api]
Workspace commands
tomlagents/lyra_default.toml
[workspaces]
lyra     = "~/projects/lyra"
projects = "~/projects"
vault    = "~/.roxabi-vault"
📂
The cwd override persists for the pool (conversation) lifetime. In Discord with auto_thread=true, each thread has its own workspace.
chat
You:  /lyra
Bot:  Switched to workspace lyra (~/projects/lyra)

You:  /lyra what's the last commit?
Bot:  Switched + forwarded to Claude with cwd set

You:  /projects list my repos
Bot:  Switched to projects (~/projects) + answered
