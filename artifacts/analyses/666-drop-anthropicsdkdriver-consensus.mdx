### Rationale

- **#3 (b):** The only "external" consumers are in-repo (roxabi-live
  dep-graph, monitoring scripts) and can be updated atomically. Dual-key
  aliasing adds protocol debt and a removal obligation with no
  confirmed external beneficiary.
- **#4 (a):** Under-counting assistant tokens risks context overflow in
  the 24/7 hub on roxabituwer. A startup warning does nothing at
  runtime. `TurnStore` already holds both roles.
- **#7 (a):** 280 LOC of code with no production callers is maintenance
  debt. `DeprecationWarning` on never-instantiated classes emits
  nothing. Keep `SmartRoutingConfig` + `SmartRoutingProtocol` as
  forward stubs so TOML parsing + future re-implementation stay cheap.
- **#9 (c):** Directly coupled to #7 — deleting the decorator removes
  the only concrete type that could populate `self._smart_routing`. A
  "not available in this build" stub implies a conditional feature
  gate we don't use.
- **#11 (a):** `spec=` enforces the attribute contract at
  test-definition time. Comments rot; typing doesn't.
- **#12 (a):** Parametrising is a one-liner; rename removes misleading
  backend-specific framing on a now-universal rule.
- **#13 (a):** Zero-cost one-line comment preserves the cross-agent
  STT exception contract (SimpleAgent catches vs. deleted
  AnthropicAgent re-raised). Next agent implementer needs that signal.

### Trade-offs

- #3: clean key space, but touches ~15 test fixtures + CB instance
  names (`CircuitBreaker("anthropic")`) — scope grows one commit.
- #4: needs a `TurnStore` accessor in `session_lifecycle`'s context —
  may require a thin helper.
- #7+#9: deleted dead code cannot be recovered without replaying
  history; but the Protocol stub keeps re-introduction cheap.

## Dissent

Devops preferred:
- **#3 (a)** — dual-key for one release, worried about missed
  downstream consumers. **Recorded.** Mitigation: the coordinator
  greps `deploy/`, `roxabi-live/`, and monitoring configs before
  merge — if any live consumer still reads `"anthropic"`, escalate
  back to dual-key.
- **#12 (b)** — parametrization out of scope for a driver-removal PR.
  **Recorded.** Mitigation: the change is ≤5 lines; ship it now.
- **#13 (b)** — doc-writer pass would re-introduce the comment later.
  **Recorded.** Mitigation: zero-cost one-liner now saves future churn.
