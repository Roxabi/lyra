Agents
Create, edit, and manage agents. Agents live in SQLite (~/.lyra/auth.db) — TOML files are seed sources only.
💾
AgentStore (v4): Agent config is now stored in SQLite (~/.lyra/auth.db). TOML files are seed sources — edit them, then run lyra agent init to import into DB. Runtime reads from DB only.
Create / update an agent
1
Scaffold a new agent TOML
lyra agent create my_agent   # interactive wizard → writes TOML
📁
User overrides: Place a file at ~/.lyra/agents/my_agent.toml to override the system default without modifying the repo. User-level files take precedence during seed.
2
Edit the TOML
tomlsrc/lyra/agents/my_agent.toml
[agent]
name              = "my_agent"
memory_namespace  = "my_agent"      # isolates memory from other agents
persona           = "lyra_default"  # system prompt from ~/.roxabi-vault
show_intermediate = true            # true → show ⏳ intermediate turns to user
permissions       = []

[model]
backend   = "claude-cli"            # "claude-cli" | "anthropic-sdk"
model     = "claude-sonnet-4-6"
max_turns = 10
tools     = ["Read", "Grep", "Glob", "WebFetch", "WebSearch"]
# Bash + Write omitted — prevents prompt injection → RCE
skip_permissions = true            # skip Claude CLI permission prompts

[agent.smart_routing]               # requires backend = "anthropic-sdk"
enabled = false

[agent.smart_routing.models]
trivial  = "claude-haiku-4-5-20251001"
simple   = "claude-haiku-4-5-20251001"
moderate = "claude-sonnet-4-6"
complex  = "claude-opus-4-6"

[plugins]
enabled = ["echo", "search"]        # search enables /search + /add + /explain + /summarize

[workspaces]                         # each key → / slash command
lyra     = "~/projects/lyra"
projects = "~/projects"

# Voice config (all optional — omit to inherit voicecli globals)
[tts]
engine   = "qwen-fast"   # qwen | qwen-fast | chatterbox | chatterbox-turbo
voice    = "Ono_Anna"
language = "French"      # overridden by user pref if "detected"
chunked  = true

[stt]
language_detection_threshold = 0.95
language_fallback            = "en"
3
Import into DB
lyra agent init          # seed DB from TOML files (skip if exists)
lyra agent init --force  # overwrite existing DB rows with TOML
💾
TOML → DB one-way: init reads TOML and writes to SQLite. After import, runtime reads from DB only. To update: edit TOML → lyra agent init --force, or use lyra agent edit to change fields directly in DB.
4
Wire the agent to a bot in config.toml
tomlconfig.toml
# First store token: lyra bot add --platform telegram --bot-id my_bot
[[telegram.bots]]
bot_id = "my_bot"
agent  = "my_agent"       # ← must match [agent].name in DB

[[auth.telegram_bots]]
bot_id      = "my_bot"
default     = "blocked"
owner_users = [YOUR_TG_ID]
lyra start

→

auth.db (AgentStore)

User TOML wins over system TOML during seed. After import, DB is authoritative.
CLI commands (DB-backed)
CommandDescription
lyra agent initSeed DB from TOML files (first-time — skips if agent exists)
lyra agent init --forceOverwrite existing DB rows with TOML values
lyra agent listList all agents in DB with status, source, and bot assignments
lyra agent show <name>Full config dump from DB (all fields)
lyra agent edit <name>Interactive edit in DB (see editable fields below)
lyra agent validate <name>Validate agent config (backend, JSON arrays, smart routing)
lyra agent createInteractive wizard → scaffold TOML (then init to import)
lyra agent delete <name>Delete from DB (refuses if any bot is still assigned)
lyra agent assign <name> --platform telegram --bot <id>Assign bot → agent mapping
lyra agent unassign --platform telegram --bot <id>Remove bot → agent mapping
Editable fields
Via lyra agent edit
backend — claude-cli | anthropic-sdk
model — e.g. claude-sonnet-4-6
max_turns — integer
persona — persona name
show_intermediate — true/false
cwd — working directory
memory_namespace — memory isolation
i18n_language — response language (default: en)
Via TOML + init --force
tools — tool allowlist
plugins — enabled plugins
smart_routing — routing config
[tts] — TTS engine, voice, language
[stt] — STT thresholds, fallback
skip_permissions — bypass CLI permission prompts
permissions — permission list
workspaces — workspace → cwd mappings
commands — custom slash commands
🔧
TTS/STT update flow: Edit [tts]/[stt] in agent TOML → run lyra agent init --force. The edit command does not yet expose TTS/STT fields.
Multi-bot — N agents × N bots
Run two bots with different personas
tomlconfig.toml
# Tokens in auth.db: lyra bot add --platform telegram --bot-id lyra
# Bot 1: Lyra (general assistant)
[[telegram.bots]]
bot_id = "lyra"
agent  = "lyra_default"

# lyra bot add --platform telegram --bot-id aryl
# Bot 2: Aryl (different model/persona)
[[telegram.bots]]
bot_id = "aryl"
agent  = "aryl_default"

[[auth.telegram_bots]]
bot_id      = "lyra"
default     = "blocked"
owner_users = [123456789]

[[auth.telegram_bots]]
bot_id   = "aryl"
default  = "trusted"   # public bot
Auth trust levels
LevelAccessConfig
ownerFull access — all commands including admin-only. Auto-admin.owner_users = [ID]
trustedCan chat with bot. Cannot run admin commands.trusted_users / trusted_roles
publicEveryone can interact.default = "trusted"
blockedMessages silently ignored.default = "blocked"
