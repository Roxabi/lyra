Setup
Start modes, platform-specific launch, and production daemon management.
Start modes
🚀 All adapters (default)
lyra start
# or
python -m lyra --adapter all
Starts all configured bots (Telegram + Discord).
✈️ Telegram only
python -m lyra --adapter telegram
Only starts Telegram adapter. Discord is skipped even if configured.
🎮 Discord only
python -m lyra --adapter discord
Only starts Discord adapter. Health port defaults to 8443.
Bot token setup
✈️ Telegram — via @BotFather
Open Telegram → search @BotFather
Send /newbot → choose name & username
Copy token → run lyra bot add --platform telegram --bot-id lyra
Username (no @) → TELEGRAM_BOT_USERNAME
Generate random string → TELEGRAM_WEBHOOK_SECRET
Your numeric ID → @userinfobot
🎮 Discord — via Developer Portal
discord.com/developers → New Application
Bot tab → Reset Token → run lyra bot add --platform discord --bot-id lyra
Enable Message Content Intent ← required!
OAuth2 → scope bot → Send Messages → invite
Your ID: Settings → Advanced → Developer Mode → right-click name
Supervisor per-adapter split
For production, adapters run as independent supervisor processes — each can restart independently.
supervisor/conf.d/lyra_telegram.conf
ini
[program:lyra_telegram]
command=run.sh --adapter telegram
environment=LYRA_HEALTH_PORT="8443"
autostart=true
autorestart=true
Health endpoint on port 8443.
supervisor/conf.d/lyra_discord.conf
ini
[program:lyra_discord]
command=run.sh --adapter discord
environment=LYRA_HEALTH_PORT="8444"
autostart=true
autorestart=true
Health endpoint on port 8444.
📝
run.sh sources .env before launching the Python process — secrets never live in supervisor conf files.
🚀 lyra-stack — recommended for production

✦ lyra-stack

A single supervisord instance manages all Roxabi daemons (lyra, voicecli_tts, voicecli_stt…). Each project repo owns its supervisor configs; lyra-stack/conf.d/ contains only symlinks. Recommended for Machine 1 (24/7 hub).
bash
# Clone alongside lyra
git clone https://github.com/Roxabi/lyra-stack ~/projects/lyra-stack

# One-time setup: register lyra symlinks + start supervisord
cd ~/projects/lyra
make register      # registers lyra conf symlinks into lyra-stack

cd ~/projects/lyra-stack
make start         # start supervisord (idempotent)
make ps            # status of all programs

# Lyra-specific shortcuts
make lyra          # start lyra
make lyra reload   # restart
make lyra logs     # tail stdout
make lyra errors   # tail stderr
make lyra status   # status

# Or from inside lyra repo:
cd ~/projects/lyra
make lyra          # delegates to lyra-stack
make deploy        # git pull → pytest → supervisorctl restart on Machine 1
  • Adapters run as independent processes — telegram can restart without touching discord
  • Machine 1 remote control: make remote status, make remote logs, make remote reload
  • make deploy runs git pull + pytest + restart on Machine 1 in one command
config.toml schema

Single-bot (simple)

tomlconfig.toml
[admin]
user_ids = []   # extra cross-platform admins

[auth.telegram]
default = "blocked"
owner_users = [YOUR_TG_ID]

[auth.discord]
default = "blocked"
owner_users = [YOUR_DC_ID]

[defaults]
cwd = "~/projects"
[defaults.workspaces]
lyra = "~/projects/lyra"

Multi-bot (explicit wiring)

tomlconfig.toml
# Tokens resolved at startup from CredentialStore (auth.db)
# Run once: lyra bot add --platform telegram --bot-id lyra
[[telegram.bots]]
bot_id = "lyra"
agent  = "lyra_default"

# Run once: lyra bot add --platform discord --bot-id lyra
[[discord.bots]]
bot_id      = "lyra"
auto_thread = true
agent       = "lyra_default"

[[auth.telegram_bots]]
bot_id      = "lyra"
default     = "blocked"
owner_users = [YOUR_TG_ID]

[[auth.discord_bots]]
bot_id      = "lyra"
default     = "blocked"
owner_users = [YOUR_DC_ID]
