Prefix Routing
/ prefix — bot commands
Intercepted by adapter

Parsed before the LLM. Executes local handler. Examples: /help, /reset, /model, /workspace.

! prefix — passthrough
Forwarded to LLM

Bypasses command parsing entirely. The full message including ! is sent to the backend as-is. Useful for prompts that start with /.

Routing Flow
1Incoming messageraw text from adapter
2Starts with /?→ command parser
3Known command?→ local handler → reply
4Unknown command→ "unknown command" or forward to LLM if passthroughs match
5No prefix / ! prefix→ agent pool → backend → response
Session commands (#341) attach to a session via reply-to message. /add, /explain, /summarize resume the correct message_index session when used as a reply.
Command Reference
CommandCategoryDescription

/explainSessionAsk the agent to explain the last response. Reply-based routing.
/summarizeSessionSummarize the conversation so far. Reply-based routing.

/workspace <key>SettingsOverride cwd with a named workspace from workspaces_json.
/voice [on|off]VoiceToggle voice I/O for this session.
/lang <code>SettingsSet response language (e.g. fr, ja). Falls back to fallback_language.
/statusInfoShow agent, model, backend, session, and pool status.
Inline Flow Example

How a /workspace command resolves:

/workspace lyra → lookup workspaces_json → cwd = ~/projects/lyra → pool updated → confirmed

How a session command routes via reply:

reply-to msg → message_index lookup → session ID resolved → /add / /explain / /summarize → correct pool
Passthroughs
Agent-level passthrough list. Commands in passthroughs_json are forwarded directly to the LLM without local processing — useful for agent-specific slash commands that should be interpreted by the model rather than the adapter.
jsonpassthroughs_json example
"passthroughs_json": ["/think", "/brainstorm", "/outline"]
