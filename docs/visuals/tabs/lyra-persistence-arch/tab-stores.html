⚡ Critical timing insight

  • Steps 1–2 are READ-ONLY — no store mutations before the user message is even submitted
  • Step 3 writes the user turn BEFORE the pool processes it (so it's logged even if processing fails)
  • Step 7 writes the assistant turn AFTER the platform confirms delivery (reply_message_id is required)
  • Step 8 is fire-and-forget — session memory never blocks the event loop
  • During streaming (step 6): stores are completely READ-ONLY from Lyra's perspective
