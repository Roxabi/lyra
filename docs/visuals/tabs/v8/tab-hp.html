Edit an agent and see it live

Thanks to ADR-029, config changes take effect on the next message — no restart needed.

1
Edit the agent
bash
lyra agent edit lyra_default
Change model, tools, max_turns, persona, or system prompt. Save and exit the editor.
2
AgentStore updated
upsert() writes the new row to the DB and updates the in-memory cache. updated_at is bumped to the current ISO-8601 timestamp.
3
Send a message to the bot
On the next inbound message, _maybe_reload() sees the changed updated_at and rebuilds the agent config from the new DB row.
4
New config is active
The response uses the updated model/tools/prompt. No daemon restart. The pool's conversation history is preserved.
Persona caveat: if you only edited a .persona.toml file on disk without saving via lyra agent edit, the daemon won't detect the change. Run lyra agent edit <name> and save any field (even unchanged) to bump updated_at.

Add a new agent

1
Create the agent in DB
bash
lyra agent create coder \
  --model claude-opus-4-6 \
  --system-prompt "You are an expert software engineer." \
  --tools Read,Edit,Write,Bash,Glob,Grep
2
Assign to a bot
bash
lyra agent assign coder --platform telegram --bot <bot_id>
Or edit config.toml to add a new bot entry with agent = "coder" and run make lyra reload.
3
Agent is live
No restart needed if you used the assign command. The agent is in the DB and any new Pool created for that bot will use it. Verify: lyra agent show coder

Deploy to production

1
Commit and push changes
bash
git add -p
git commit -m "feat(agents): add coder agent"
git push origin staging
2
Deploy to Machine 1
bash
make deploy   # git pull → pytest → supervisorctl restart on Machine 1
3
Seed DB on Machine 1 (if new TOML-based agents)
bash
ssh mickael@192.168.1.16 "cd ~/projects/lyra && lyra agent init"
Not needed if you created the agent via lyra agent create — that writes DB directly.
4
Verify production
bash
make remote status   # supervisor status on Machine 1
make remote logs     # tail production stdout

Debug a missing response

1
Check daemon is running
make lyra status — both lyra_telegram and lyra_discord should show RUNNING.
2
Check errors log
make lyra errors — look for tracebacks, auth failures, or DB errors.
3
Verify bot assignment
sqlite3 ~/.lyra/auth.db "SELECT * FROM bots" — confirm the bot has an agent_name.
4
Check circuit state
make lyra logs | grep circuit — if OPEN, the circuit is blocking messages. Wait for cooldown or restart the daemon.
5
Enable debug logging
LYRA_LOG_LEVEL=DEBUG make lyra reload — verbose output shows every routing decision.
