Structured Store

SQLite KV

  • Per-agent scoped key-value pairs
  • Metadata and config persistence
  • Fast lookup, no embedding overhead
Semantic Store

SQLite + Vector

  • Text memories with optional f32 embeddings
  • Phase 1: LIKE-based search fallback
  • Phase 2: cosine similarity
  • Fields: content, source, scope, confidence
  • Access tracking per memory
Knowledge Graph

Entity Triplets

  • Entity-relation-entity with confidence scores
  • SQLite-backed, queryable
  • Builds associative knowledge
Session Store

Per-Channel + Canonical

  • Cross-channel canonical sessions
  • Compaction at 100 messages (keep 50 + LLM summary)
  • JSONL mirror for durability
Context Injection

Prompt Cache Strategy

  • Canonical context injected as first user message
  • Not in system prompt — preserves prompt caching
  • Task queue: task_post(), task_claim(), task_complete() with priority ordering

Lyra Comparison

Key patterns to adopt or study for Lyra's own architecture

Adopt — High confidence, implement directly
  • Three-tier memory — Structured + Semantic + Knowledge Graph, all SQLite-backed with no external vector DB dependency
  • ChannelBridgeHandle abstraction — Trait-based channel decoupling prevents circular deps between kernel and adapters
  • EventBus with history buffer — Ring buffer enables late joiners to catch up without complex replay infrastructure
  • Per-agent channels — DashMap-based isolation provides scale without broadcast contention
  • HAND.toml manifest packaging — Declarative agent packaging with settings, requirements, and dashboard metrics
Study — Complex, needs deeper analysis before adoption
  • 16-layer security model — Comprehensive but heavy; evaluate which layers provide real value vs. complexity cost
  • Workflow engine with fan-out/collect — Powerful pattern but adds significant complexity to the runtime
  • Taint tracking + approval gates — Information flow control; study the granularity and performance impact
  • Canonical context injection — Using first user message instead of system prompt for prompt caching; test with Anthropic API specifically
Bottom Line
OpenFang validates Lyra's hub-and-spoke approach with a mature Rust implementation. The three-tier memory system and ChannelBridgeHandle abstraction are directly applicable. The 16-layer security and workflow engine are worth studying but should be adopted incrementally to avoid premature complexity.
