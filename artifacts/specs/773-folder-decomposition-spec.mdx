## Context

**Promoted from:** [Analysis: V4 — decompose oversized folders](../analyses/773-folder-decomposition-analysis.mdx)

Follow-up to #760 (file-size exemption elimination). V1–V3 merged in #772. V4 (folder decomposition) was deferred due to blast radius (~150+ import updates). This spec covers 3 folders: `core/` (flat files), `adapters/`, `bootstrap/`. `core/hub/` deferred to follow-up issue.

**Precedent:** #395 successfully decomposed `core/` into `stores/`, `hub/`, `pool/`, `commands/` using scripted approach with AST-aware import updates.

## Goal

3 folders (`core/` flat, `adapters/`, `bootstrap/`) contain ≤12 files per folder. `tools/folder_exemptions.txt` reduced to `core/hub/` only. Navigation is intuitive — module boundaries explicit in directory structure.

## Users & Use Cases

- **Primary:** Maintainers (Mickael) — navigate large folders quickly
- **Secondary:** AI coding agents — structural orientation without full directory scan

## Expected Behavior

### Happy path

1. Developer opens `src/lyra/core/` → sees 5 subdirs (agent/, cli/, memory/, messaging/, auth/) + flat utilities ≤12 files
2. Developer opens `src/lyra/adapters/` → sees 4 subdirs (discord/, telegram/, nats/, shared/)
3. Developer opens `src/lyra/bootstrap/` → sees 5 subdirs (standalone/, wiring/, lifecycle/, factory/, infra/)
4. All imports use new paths, verified by pyright + lint-imports

### Edge cases

- **Relative import depth:** Files moved from flat to subdir need `.` → `..` depth adjustment
- **Cross-subdir imports:** `from .agent` in cli/ → `from ..agent` (one level up)
- **Layer constraints:** `.importlinter` may need path updates for new subfolders

## Non-goals

- `core/hub/` decomposition (deferred to follow-up issue)

## Technical Decisions

1. **Existing subdirs unchanged:** `processors/`, `stores/`, `hub/`, `pool/`, `commands/` stay in place
2. **Orphan resolution:** Move `processor_registry`, `stream_processor` → `processors/`; move `builtin_commands`, `workspace_commands` → `commands/`
3. **Flat utilities accepted:** Up to 9 shared utility files remain flat in `core/` (≤12 goal met)
4. **Test co-location:** Tests updated in lockstep with source moves
